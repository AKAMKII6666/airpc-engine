## 3. 第一版关键共识

- Studio V2 不是旧 UI 的重皮，而是新 UI 项目。
- 首页是故事工程工作台，不是营销页。
- 故事编辑器是全屏自由画布，不是固定三栏表单。
- 不使用泳道作为默认视图，也不作为第一版整理工具。
- 角色通过左侧角色锚点和顶部角色归属线表达。
- CallCard 节点左侧接 parent，右侧接 exit，顶部接角色。
- 出口是触点，一个触点可以有动作列表。
- 资源库和角色库在编辑器中以浮窗形式辅助引用。
- 用户管理页用于维护测试用户档案，服务调试器，不参与正式剧情角色建模。
- 用户/角色头像由 Studio / Next.js 后端上传、裁剪、托管和打包；引擎只保存 avatarAssetId 等资源引用并做存在性校验。
- 调试器展示剧情电话如何推进状态，不以 raw JSON 为主体。
- 所有内部 ID 自动生成，主流程不要求用户手填。
- JSON 只在高级视图中出现，并使用格式化编辑器。
- UI 默认采用暗色调色板。
- 关键领域契约、状态转换和跨层副作用必须有意图注释，并由门禁脚本检查。
- Studio V2 从第一份代码开始执行文件规模、复杂度、依赖边界和注释门禁。
- Studio V2 代码目录只认 `app/`、`src/`、`typeFiles/`、`tests/`；页面实现落 `src/pageComponents/`，编排落 `src/bis/`，类型契约落 `typeFiles/`。

## 4. 第一版暂不做

- 角色锚点自定义排序。
- 剧情分组或章节区域。
- 泳道视图。
- 故事包地图。
- 多卡批量编辑。
- 调试器时间轴回放。
- 复杂模板市场。
- 浅色主题。

## 5. 给执行者的提醒

实现时应优先守住产品形态：

- 先搭正确的信息架构，再补局部功能。
- 先保证画布可用，再补属性表单。
- 先保证出口动作语义正确，再追求连线炫技。
- 先保证调试器能讲清楚一次电话的状态推进，再展示 raw snapshot。
- 不要把旧 UI 的字段暴露、JSON 优先、固定大侧栏、手填 ID 习惯带进 V2。
