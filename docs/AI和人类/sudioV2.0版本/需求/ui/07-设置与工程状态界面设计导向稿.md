# 设置与工程状态界面设计导向稿

> 版本定位：本文定义 Studio V2.0 设置页、工程状态页、校验报告入口的设计方向。  
> 设置页服务于工具本身，不能变成剧情编辑的替代入口。

## 1. 设置页定位

设置页只处理 Studio V2.0 的全局偏好和工程级配置。

它不负责：

- 编辑 CallCard。
- 修改出口动作。
- 手工维护 runtime 状态。
- 直接编辑故事包 JSON。

设置页负责：

- 外观偏好。
- 保存与导出偏好。
- Schema 与引擎兼容状态。
- 默认目录。
- 高级功能开关。
- 调试偏好。

## 2. 设置页布局

建议采用简单两栏布局：

```text
左侧：设置分类
右侧：当前分类内容
```

分类建议：

- 外观。
- 编辑器。
- 调试器。
- 导入导出。
- Schema 与引擎。
- 高级。

设置页不是高频页面，不需要画布式布局。

## 3. 外观设置

第一版外观设置可以很少：

- 主题：默认暗色。
- 画布点阵强度。
- 连线动画开关。
- 高亮强度。

由于 V2 已确定暗色调色板，第一版不需要做完整浅色主题。

## 4. 编辑器设置

编辑器偏好：

- 默认缩放。
- 是否显示小地图。
- 是否默认显示校验浮窗。
- 是否开启自动保存。
- 自动保存间隔。
- 新卡片默认类型。

注意：

- 不提供泳道相关设置。
- 不提供复杂布局模板。
- 不让用户手工设置 ID 生成规则。

## 5. 调试器设置

调试器偏好：

- 默认用户档案。
- 默认是否重置状态。
- 默认是否显示高级日志。
- 时间推进步长。
- 调试记录保留数量。

调试器设置只影响默认值，不改变引擎语义。

## 6. 导入导出设置

导入导出偏好：

- 默认导出类型。
- 是否导出编辑器布局元数据。
- 是否包含资源。
- 默认导出目录。
- 导入时是否复制资源。

危险设置需要明确说明，例如覆盖同名故事包。

## 7. Schema 与引擎状态

这是设置页里最重要的工程状态区域。

应展示：

- 当前 Studio schema 版本。
- 当前引擎 schema 版本。
- 兼容状态。
- 可用 effect 类型。
- 可用 CallCard 类型。
- 最近一次同步时间。

状态文案：

- 已兼容。
- 需要迁移。
- Studio 版本过旧。
- 引擎版本过旧。
- 存在未知字段。

不要把 schema diff 作为默认界面。

## 8. 校验报告入口

校验报告可以从多个地方打开：

- 首页工程状态。
- 故事编辑器校验浮窗。
- 导出前检查。
- 设置页工程状态。

校验报告应按问题分类：

- 阻断错误。
- 警告。
- 提示。

问题类型：

- 不可达卡片。
- 出口无动作。
- 章节结束点缺少下一章节配置。
- 资源缺失。
- 角色缺失。
- 延迟外呼配置不完整。
- effect 参数不完整。
- 循环或歧义入口。

每个问题都应提供：

- 问题说明。
- 影响范围。
- 跳转定位。
- 建议处理方式。

## 9. 高级设置

高级设置可以包含：

- 显示内部 ID。
- 打开 JSON 高级视图。
- 导出调试包。
- 清理本地缓存。
- 重建索引。

高级设置必须有明确警示，不应成为普通用户主要路径。

## 10. 工程状态页是否独立

第一版不需要单独做完整工程状态页。

建议：

- 首页显示简要工程状态。
- 设置页显示完整工程兼容状态。
- 校验报告作为可打开的浮窗或页面。

如果后续项目复杂，可以再拆成独立工程状态页。

## 11. 验收标准

第一版至少满足：

- 可以查看 Studio 与引擎 schema 兼容状态。
- 可以设置编辑器基础偏好。
- 可以设置调试器默认偏好。
- 可以设置导入导出偏好。
- 可以打开校验报告。
- 校验问题可以跳转到对应故事卡或资源。
- 高级 JSON 和内部 ID 默认隐藏。

## 12. 避免事项

- 不要把设置页做成所有字段的大杂烩。
- 不要在设置页编辑剧情内容。
- 不要把 schema 错误原样暴露给用户。
- 不要提供泳道设置。
- 不要提供会破坏工程的一键危险操作。
- 不要让高级功能影响普通创作路径。

## 13. 一句话总结

设置页负责 Studio 的工作偏好和工程健康，不负责替代编辑器；校验报告负责告诉用户哪里坏了，并带用户回到正确位置修。
