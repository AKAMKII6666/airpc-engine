# AI-RPG NPC 引擎 · 文档索引与总纲

本仓库（`airpc-engine`）目标是做成一套**完全独立的 AI-RPG NPC 引擎**：在 Studio 中走通后，可插入电话壳（或其它客户端）。话机只提供硬件、Realtime 通道与媒介执行；**剧情、角色、记忆、存档、世界规则以本引擎为准**。

旧话机（doubaoSister）**仅作功能参考**：要对齐「通话里能做什么」，用更干净的模型（CallCard / Exit / Effect / Free 管线）重做。

**对标 ≠ 迁移：**

- **不**把本引擎塞进 doubaoSister，也 **不**把旧仓模块 / store / handler 迁进本仓  
- **不**拷贝旧 JSON Schema / 配置文件当实现真源  
- 本仓 Studio 走通后，将另建**新话机软件底座**，专门嵌入本引擎（硬件 / Realtime / EffectSink）

CallCard 细文仍有效，但是引擎的**一柱**，不是全部。入口冲突时以本总纲 + `需求/` 领域文为准；工程落点以 [技术设计文档](../技术设计文档/00-技术设计索引.md) 为准。

## 3. 五柱模型

```text
内容层（跨用户共享模板）
  CharacterDef · CallCardDef · StoryPackage · WorldLore 模板

存档层（按用户隔离，运行时唯一真源）
  User → PlayerProfile
    · 各角色记忆 · 剧情进度 · CallCardBoard
    · Lore / Facts / knowledge · schedule · redialSlot
```

| 柱 | 职责 |
|----|------|
| **User / PlayerProfile** | 用户档案与全部进度存档 |
| **Character** | 角色定义（身份/人设/默认开场/过程话术）+ 设计期社交；运行态记忆在 Profile；Free 卡旁路存放 |
| **CallCard** | 单次通话合同；引荐/带话也用卡+effect，**不设 Relay 子系统** |
| **StoryPackage** | 可插拔剧情内容包（定义）；进度在 Profile |
| **World（瘦）** | Lore bootstrap 规则、Fact schema、WET/时钟；**具体数据在 Profile** |

## 4. 铁律

1. **定义在内容库；经历在 Profile。**
2. **每一通电话由一张 CallCard 驱动**；无剧情卡则 FreeCallCard。
3. **CallCard 是剧本合同，不是提示词**；提示词是 Composer 投影（含 `promptScenes` 与用户本地时间注入）。
4. **Story 出口 = condition + priority + effect plan**；静态 ∪ 动态候选挂机后进 ExitSelector。**Free 卡**走 `FreeCallPostPipeline`（MemoryCommit；有 candidate 则再 ExitSelector）；`packageId=__free__`；默认全开 free 工具。
5. **通话中不推进存档世界**（纯媒介除外且须记账）；工具只登记 `RuntimeExitCandidate`。挂机后允许跨角色 `patch_memory`；禁止 onConnect 写记忆。
6. **引荐与带话用 CallCard**，不设 Relay 子系统。
7. **社交设计期边在 CharacterDef**；已经历在 Profile。
8. **引擎纯 TS，只跑在 server / 壳**；禁止进 client bundle。
9. **Studio 为 B/S**：浏览器只请求与展示；Next 为门面；引擎是后端的后端。**Host 进程单例**（见 [19](../技术设计文档/19-引擎宿主与会话模型.md)）。
10. **模型通道可插拔**；Studio 文本 LLM **仅 server**。
11. **壳可替换**；媒介经 EffectSink；逻辑经 EffectExecutor。
12. **`toolId` 以 ToolRegistry 为真源**。
13. **Effect plan v1**：逐条、失败默认继续；`critical` 中止；无自动回滚。
14. **持久化：** Content + 薄 Profile = JSON；**Memory = SQLite**（MemoryPort）；旁路 `logs/*.jsonl`；CallSession 默认不进 SaveGame。**存得下 ≠ 查得起、注得起。**
15. **调试与编辑分离**：调试中不改卡；改卡须退出调试 → 保存 → 再调试。本通使用 `frozenCard`。

## 5. 术语（增量）

| 术语 | 含义 |
|------|------|
| User / PlayerProfile | 用户与存档 |
| CallSession | 单通通话会话态（内存；见 19） |
| EngineHost | 进程内引擎宿主（v1 单例） |
| UserGate | 可复用用户选择组件（Modal） |
| FreeCallPostPipeline | 自由通话挂机总函数 |
| EffectSink | 媒介副作用端口（壳/Studio mock） |
| RuntimeExitCandidate | 通话中登记的出口候选 |
| Content 工作区 | `data/storis-packages/`（conf + `.s-card.json`）等 |
| `FREE_PACKAGE_ID` | `"__free__"`；纯 Free 会话的 packageId 哨兵，禁止当真实包目录 |
| ComposeScene | 本通方向 + 用户本地时间（Composer 输入；见 50） |
| promptScenes | 卡上声明式开场/时段层；见 01 / 50 |
| RenderedPrompt | Composer 输出（含 matchedLayerIds） |
| effectiveDialable | `dialable \|\| unlocked`；见 11 / 02 |

## 6. 产品形态拍板

| 决策点 | 结论 |
|--------|------|
| 拓扑 | B/S；见 [18](../技术设计文档/18-部署拓扑与BS架构.md) |
| Studio 入口 | **故事列表** `/stories` → 画布 `/stories/[id]`；角色库；`/debugger` |
| 用户门禁 | 无密码；**UserGate** 组件；进调试 / 进故事时弹出 |
| UI 组件库 | **仅 MUI**（不用 shadcn/Radix/Ant 作主库） |
| 样式 | **CSS Modules + SCSS**；可用现代 CSS |
| 客户端编排 | **Zustand + bis + shell**（见 [21](../技术设计文档/21-Studio客户端分层.md)）；非 Profile 真源 |
| 画布 | `@xyflow/react`；layout 旁车文件 |
| 引擎 | `rpg-engine`；Host **单例** |
| Outcome v1 | Manual；可选用文本建议 |
| Free 挂机 | FreeCallPostPipeline（MemoryCommit；有候选再 Exit） |
| Free packageId | 哨兵 `__free__` |
| 提示词组装 | `promptScenes` + 强制注入用户本地时间（见 50） |
| 可拨 | `effectiveDialable`；解锁经 `set_character_unlocked` |
| 持久化 | Content/薄 Profile：JSON；**Memory：SQLite**（见 [12](./12-记忆模型.md)、[20](../技术设计文档/20-记忆存储与投影.md)） |
| 记忆注入 | 热层默认带；冷层 `search_memory`（FTS，见 20 §5）；禁止全量灌 prompt |
| 多存档槽 | **不做** |
| 旧迁移 | **不做**（对标能力，另建新壳） |

## 7. v1 非目标

- 改造、嵌入或迁移 doubaoSister（代码 / 数据 / 配置均不迁）
- 完整 World Tick / 云协作 / 自研 canvas / 真音频硬件（硬件在未来新壳）
- 通话中直接 Effect 或 client 引引擎写口
- 多存档槽、Web 多 Host 实例
- 调试器内改卡
