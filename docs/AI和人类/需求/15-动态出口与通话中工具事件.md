# 15. 动态出口与通话中工具事件

相关：[13](./13-工具与能力注册.md) · [02](./02-运行时协议.md) · [19](../技术设计文档/19-引擎宿主与会话模型.md) · [50](./50-对话与模型适配.md)

## 1. 铁律

```text
通话中工具 / FC 不得直接执行 EffectExecutor。
可以登记 RuntimeExitCandidate（写入 CallSession）。

挂机后 Story：
  Outcome → 静态 exits ∪ 动态候选 → ExitSelector → 仅选中出口的 effects。

挂机后 Free：
  FreeCallPostPipeline
    = MemoryCommit（有效聊天）
    + 若 exitCandidates 非空 → 同一套 ExitSelector（静态 exits 可空）
  不要求作者堆静态 exits；禁止从 transcript 隐式扫描写 Profile。
```

内置工具 id / 参数要点见 [13 §7](./13-工具与能力注册.md)（对标旧话机能力；实现用 Zod，不搬旧 schema）。

## 2. RuntimeExitCandidate

```ts
interface RuntimeExitCandidate {
  candidateId: string
  exitId?: string
  source: "tool" | "manual_debug" | "system"
  toolId?: string
  toolCallId?: string
  title: string
  exitKind?: ExitKind
  priority: number
  hints?: Record<string, unknown>
  effects?: Effect[]
  createdAt: string
}
```

存在 **CallSession.exitCandidates**（见 [19](../技术设计文档/19-引擎宿主与会话模型.md)）；**不**在 tool 回调写 Profile。

## 3. 主链补全

```text
tool.requested → ToolRuntime 校验
  → append RuntimeExitCandidate（不跑 effect）
  → session.ended
       ├─ Story → Outcome → ExitSelector(静态 ∪ 动态) → Effect (+ Sink)
       └─ Free  → FreeCallPostPipeline
                    → MemoryCommit
                    → 若有 candidates → ExitSelector → Effect (+ Sink)
```

## 4. 与 Outcome / 静态出口（定稿）

| 情况 | 行为 |
|------|------|
| 动态带 `exitId` | 用该静态出口的 **condition + effects**；candidate 可提高 priority / hints |
| 纯动态 | 必须自带 `effects`；隐式条件 = candidate 有效 AND Registry 声明的基础 flags（如 answered） |
| 同 priority | **静态声明顺序 > 动态** |
| 动态无 effects 且无 exitId | 校验/运行时报错 |
| Free 无 candidates | 跳过 ExitSelector（只 Commit + log） |

Outcome 回答「这通结果如何」；Exit 回答「选哪条推进」。二者配合，不是互相替代。Free 可无完整 Outcome，候选条件以 Registry 模板为准。

## 5. 调试器

- 可手动登记/删除 candidate；时间线标明 static / dynamic
- Free + 工具候选 → 挂机后须能看到 Effect 落盘
- 不在调试器改卡定义

## 6. 非目标

- 通话中直接挂卡 / 写 fact / 改记忆
- 工具任意拼未知 effect
- 将壳侧 `request_hangup` / `end_conversation` / `complete_referral_handoff` 当引擎 Effect
