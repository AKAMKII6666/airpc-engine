# 03. Studio 编辑器界面需求

Studio 编辑面：在 **B/S** 下由浏览器维护 UI；内容经 Next 门面进入引擎。**主入口是故事列表**，再进入单故事画布。

相关：[00](./00-文档索引.md) · [19-引擎宿主与会话模型](../技术设计文档/19-引擎宿主与会话模型.md) · [18](../技术设计文档/18-部署拓扑与BS架构.md) · [05](../技术设计文档/05-技术选型与工程边界.md)

**数据流**：浏览器编辑草稿 / 投影 → API 保存 → 引擎写 Content（JSON）。浏览器不直连磁盘业务文件、不持有 Profile 真源。

**UI**：基础组件库使用 **MUI**；画布使用 `@xyflow/react`。

## 1. 信息架构

| 台 | 路由 | 职责 |
|----|------|------|
| 故事列表 | `/stories` | 列出 StoryPackage；创建故事 = 新包 + 空画布；删除/重命名 |
| 故事画布 | `/stories/[packageId]` | 单包 CallCard 编辑（本文重点） |
| 角色 | `/characters` | CharacterDef、社交、dialable、FreeCallCard |
| 调试 | `/debugger` | 见 [04](./04-调试器界面需求.md) |

可选后续：世界台、资源库可挂在画布左侧或独立路由。

### 1.1 用户门禁（UserGate）

- **无密码**。组件 `UserGate`：列表 / 新增 / 选择 / 删除；可放进 MUI Dialog。
- **进入画布前**（点击某故事或「创建故事」）弹出 UserGate，选定用户后继续。
- 故事**编辑逻辑与用户无关**（改 Content）；门禁仅为建立可调试的 Profile 上下文。
- 顶栏可显示当前用户；未选时调试类 API 返回 `USER_REQUIRED`（见 [19](../技术设计文档/19-引擎宿主与会话模型.md)）。

## 2. 故事列表

```text
┌─────────────────────────────────────────┐
│  故事包列表          [+ 创建故事]          │
│  · 标题 / packageId / 校验状态           │
│  · 点击 → UserGate → 进入画布            │
└─────────────────────────────────────────┘
```

创建故事 = 新建目录 `data/storis-packages/<packageId>/`，写入空的 `story.conf.json`（`cards: []`）+ `cards/` + `canvas.layout.json`。

## 3. 通话卡画布布局

```text
┌──────────────┬───────────────────────────────┬────────────────┐
│ 左侧库        │           故事画布              │ 右侧属性面板    │
│ · 角色列表    │  角色泳道 + CallCard + 连线      │ 当前选中对象    │
│ · 卡模板      │                               │                │
│ · 资源库      │                               │                │
├──────────────┴───────────────────────────────┴────────────────┤
│ 底部：校验结果 / 问题列表 / 保存状态                              │
└──────────────────────────────────────────────────────────────┘
```

## 4. 画布核心元素

| 元素 | 视觉 | 职责 |
|------|------|------|
| 角色泳道 | 每 participant 一条 lane | 拖卡改 `ownerAgentId` |
| Start | 唯一入口 | 指向 `entryCardId` |
| CallCard 节点 | free / story / system 样式 | 徽章：entryMode、interactionMode |
| Exit 端口 / 边 | 连到目标卡 | **UX 镜像**；真源仍是 `exits[].effects` |
| Effect | 默认右侧面板 | 复杂 plan 可不展开节点 |
| End | terminal 汇聚 | 可选 |

坐标与泳道序写入 **`canvas.layout.json` 旁车**；引擎运行忽略 layout。见 [19 §10](../技术设计文档/19-引擎宿主与会话模型.md)。

### 4.1 电话媒介徽章

由出口 effects 推导：呼入/外呼、playback、重播/留言/回拨等。

## 5. 主交互（v1）

1. 创建/编辑 CallCard → 添加 Exit → 连线到下一卡（生成 `attach_call_card`，可改）。
2. 选中 Exit → 面板编辑 condition、priority、effects。
3. **引荐/带话**：连到目标角色卡并写好目标卡 brief；不要独立 Relay 节点。
4. FreeCallCard 在角色库维护；静态 exits 可空；挂机走 FreeCallPostPipeline（有工具候选再 Exit）。默认全开 free 工具。
5. 真源：`story.conf.json` + `cards/*.s-card.json`；画布为投影。改 `cardId` = 重命名 `.s-card.json` 并改 conf 索引与其它卡引用。

## 6. 左侧库与属性面板

- 角色列表可跳转角色台。
- 每角色 FreeCallCard 可打开编辑（上下文/工具策略等；出口可空）。
- `playback_only` 锁定 deny_all tools。
- **场景层 `promptScenes`**：属性面板可增删层（方向 / 时段 / opening / tone）；校验禁止 patch `objective`/`forbidden`。详见 [01](./01-CallCard数据模型.md)、[50](./50-对话与模型适配.md)。

## 7. 角色台

信息架构与字段见 [11](./11-角色与社交.md)：identity（年龄/性别）、persona.systemPrompt、defaultPromptScenes、callFlowPrompts、social、Free 卡跳转。

## 8. 校验与导入导出

- `validatePackage`（规则总表见 [08](./08-内容校验规则.md)）；error 阻断导出
- 内容包与 SaveGame 分开导出
- `schemaVersion` 不兼容 → 拒载 Dialog（见 [17](../技术设计文档/17-版本迁移与兼容策略.md)、[19](../技术设计文档/19-引擎宿主与会话模型.md)）

## 9. 编辑体验

撤销/重做、多选删除为 P0；复制粘贴、自动布局、搜索为 P1。

## 10. 与调试器关系

- 编辑器**不**在通话中热更新正在调试的 Session。
- 从画布「去调试」→ `/debugger`（再经 UserGate 若需要）→ 只读已保存内容。
- 调试中发现问题 → 退出调试 → 回本页改并保存 → 再调试。

## 11. 非目标

- 云协作、编辑器内强制 Realtime、Relay 专用节点
- 调试器内嵌改卡
