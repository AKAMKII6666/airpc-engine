# 50. 对话与模型适配 · 上下文合成

相关：[02](./02-运行时协议.md) · [18](../技术设计文档/18-部署拓扑与BS架构.md) · [12](./12-记忆模型.md) · [01](./01-CallCard数据模型.md) · [19](../技术设计文档/19-引擎宿主与会话模型.md)

## 1. 问题

电话壳使用 **全双工 Realtime**；Studio 调试用 **文本模型**（或 Manual）模拟同卡语义。  
引擎只认 Dialogue 端口与事件，**禁止**把厂商 Realtime/Chat SDK 写进核心状态机。

## 2. 部署口径（Studio）

```text
浏览器：聊天 UI（输入、流式展示、挂机按钮）—— 不持 API Key、不直连大模型
Next server：TextDialogueAdapter + LLM 调用（Key 在 server）
rpg-engine：会话事件、动态出口登记、挂机后 Outcome/Exit/Effect
```

浏览器侧的「执行」= **演 UI**；模型接入与领域推进都在 **server**。详见 [18](../技术设计文档/18-部署拓扑与BS架构.md)。

## 3. 两端口

### 3.1 DialogueAdapter

```ts
type DialogueChannel = "realtime_audio" | "text_turn" | "manual"

interface DialogueAdapter {
  readonly channel: DialogueChannel
  start(spec: DialogueSessionSpec): Promise<void>
  update?(patch: DialogueSessionPatch): Promise<void>
  sendUserText?(text: string): Promise<void>
  stop(reason: string): Promise<void>
  subscribe(handler: (ev: DialogueEvent) => void): () => void
}

interface DialogueSessionSpec {
  sessionId: string
  channel: DialogueChannel
  card: CallCardDefinition
  instanceId: string
  prompt: RenderedPrompt
  composeScene: ComposeScene
  /** 已按 toolPolicy ∩ Registry 过滤；壳挂机 FC 不在此列 */
  tools: ToolDefinition[]
  interactionMode: InteractionMode
  playback?: { assetId: string; transcript?: string }
}

interface DialogueSessionPatch {
  /** 过程话术注入等；v1 以追加为主 */
  appendSpeakable?: string
  appendPrivate?: string
  tools?: ToolDefinition[]
}
```

`DialogueSessionSpec` 由 beginCall → Composer 后交给 Adapter；过程话术经 `update` 注入（见 §5.4）。

**正交性：**

- 卡上 `interactionMode` = 剧情意图
- `DialogueChannel` = 音频双工 / 文本轮次 / 人工点选

可用 `text_turn` 跑 `interactionMode: realtime_dialogue` 的卡。

### 3.2 OutcomeEvaluator

```ts
interface OutcomeEvaluator {
  evaluate(input: {
    card: CallCardDefinition
    transcript: unknown
    toolTrace: unknown
    phoneFlags: Partial<Record<OutcomeFlag, boolean>>

```

| 实现 | 用途 | 运行位置 |
|------|------|----------|
| ManualOutcomeEvaluator | 调试器勾选（v1 默认） | server / 引擎 |

## 5. ContextComposer

### 5.1 类型定稿

```ts
interface ComposeScene {
  callDirection: "inbound" | "outbound"
  localTime: {
    isoWithOffset: string   // 用户墙钟，含 offset
    timeZone?: string       // 壳/浏览器上报；User 不存 timeZone 真源
    localHour: number       // 0–23；promptScenes 仅按 localHourRange 匹配
  }
  /** Free 默认可自然提及时段；Story 默认只校正用语 */
  timeMentionPolicy: "allow_casual" | "correct_only"
}

interface RenderedPrompt {
  systemHard: string[]
  openingSpeakable?: string
  openingPrivate?: string
  speakable: string
  private: string
  softContext: string[]
  matchedLayerIds: string[]
  debug?: { notes?: string[] }
}
```

`BeginCallOpts.localNowIso` / `timeZone` → Host 填 `composeScene`。缺省：请求到达时的 UTC + 上报时区；再缺 Studio v1 可用 `Asia/Shanghai`。**不以 User.location 当墙钟真源。**

### 5.2 组装管道（固定顺序，可测）

```text
compose({
  user, characterDef, memoryProjection, card,
  worldLore, worldFactsVisible, scene: ComposeScene
}) → RenderedPrompt
```

```text
1. base ← card.context（objective / forbidden / private / speakable / emotion…）
2. 本通 card.promptScenes（若有）按 match + priority 应用
3. 若本通未产出 opening*：回落 CharacterDef.defaultPromptScenes（同构 match）
4. 注入角色 `persona.systemPrompt`（hard）与可选 `persona.personalityCode`（hard，人格扮演约束）；`identity`（年龄性别等）进 soft
5. 注入「时间意识」块（每通必有，见 5.3）—— hard
6. 叠 Lore / Facts(可见) / Memory soft（投影须带记忆条目时间 / 相对「多久以前」；见 [12](./12-记忆模型.md)）
7. 输出 RenderedPrompt（含 matchedLayerIds）
```

只读，不写 Profile。  
优先级摘要：

```text
卡 hard（objective/forbidden）
  → 本通 promptScenes
  → 角色 defaultPromptScenes（卡未覆盖 opening 时）
  → persona.systemPrompt + persona.personalityCode + identity
  → 用户本地时间注入
  → Memory / Lore
```

### 5.3 时间意识注入（每通必做）

即使卡上无 `promptScenes`，也注入结构化时间事实（给模型校正用语；不要求念给用户）：

```text
[用户本地时间]
- 现在：<iso 本地>，本地小时=<0–23>
- 问候与节奏应符合当前本地时间；勿说错「早上好」等。
- 与剧情冲突时：objective / forbidden 优先，时间事实仍保留。
```

- Free：`timeMentionPolicy = allow_casual`  
- Story：默认 `correct_only`

### 5.4 通话中过程事件（过程话术）

与挂机 Exit 的 `hangup_early` 不同：下列事件**不改 Profile**，只触发角色 `callFlowPrompts` 或 Adapter 注入下一轮。

| DialogueEvent | 选用 |
|---------------|------|
| `silence_timeout` | `callFlowPrompts.longSilence` |
| `call_duration_threshold` | `callFlowPrompts.longCallNudge` |
| `pre_hangup_hint` | `callFlowPrompts.preHangupFarewell` |

**计时归属（定稿）：**

| 角色 | 职责 |
|------|------|
| **壳 / DialogueAdapter** | 按策略计时并上报上述事件（Realtime 与 text_turn 均由 Adapter 负责） |
| **引擎** | 不计时；按 `callFlowPrompts.*` 取变体 → 注入下一轮或请 Adapter 播一句 |
| **Studio Manual** | 调试器按钮「模拟静默 / 长通话 / 预挂机」发同一事件即可 |

默认阈值（角色 `callFlowPolicy` 可覆盖，见 [11](./11-角色与社交.md)）：

| 项 | 默认 |
|----|------|
| `silenceTimeoutMs` | `15000` |
| `longCallNudgeMs` | `600000`（10 min） |
| `preHangupHintBeforeEndMs` | 壳在用户点挂机前可选上报 |

### 5.5 与老话机效果对照

| 老机制 | 本设计 |
|--------|--------|
| 呼入/呼出不同开场 | 卡 `promptScenes` 或角色 `defaultPromptScenes` |
| 按时段切提示词 | `localHourRange` / `toneHint` / opening*（**禁止** `timeBuckets`） |
| 长静默 / 长通话话术 | Adapter 计时 + `callFlowPrompts` |
| 壳内 if 拼装 | Composer 固定管道 |
| AI 有时间观念 | `ComposeScene.localTime` 强制注入 |
## 6. 与主链衔接

```text
（门面）Resolver → Composer → adapter.start
  → events …（tool → RuntimeExitCandidate；过程话术事件）
  → session.ended
  → Story：OutcomeEvaluator → ExitSelector → Effect → save
  → Free：FreeCallPostPipeline（MemoryCommit；有候选则再 ExitSelector）→ save
```

Studio 文本调试典型 API：

```text
POST/SSE /api/debug/chat     # 用户一句 → server LLM → 引擎 → 流式回浏览器
POST     /api/debug/endCall  # 挂机推进
POST     /api/debug/simEvent # 模拟 silence_timeout 等
```

## 7. Studio 分档

| 档 | 行为 |
|----|------|
| v1 | Manual（电话事件 + beats）；可手动登记动态出口；可模拟过程事件 |
| v1.1 | server 侧 TextDialogueAdapter + 浏览器聊天 UI |
| Outcome | 默认 Manual；高级「文本建议 Outcome」可改后再确认（见 [19 §13](../技术设计文档/19-引擎宿主与会话模型.md)） |
| Free 挂机 | FreeCallPostPipeline（见 [19 §5](../技术设计文档/19-引擎宿主与会话模型.md)） |
| 壳 | RealtimeDialogueAdapter（负责过程话术计时上报） |

`playback_only`：不开放聊天，只模拟播放完成/跳过。

### 7.1 隐私

author 可见 `privateBrief`；playtest 默认隐藏；导出日志不落 private 原文（见 [04](./04-调试器界面需求.md)）。

## 8. 工程约束

- 引擎：端口类型、Composer、ToolRegistry、会话钩子；零 React
- LLM/Realtime **SDK 仅 server 或壳**；禁止 `'use client'` 调厂商 API
- 禁止 client 在 tool 回调里写 Profile

## 9. 验收要点

- Manual / Text 都能走到 ExitSelector；Text 时 Key 仅在 server
- client 不 import 引擎写口
- 工具只登记候选、不直接改 Profile
- Composer：`ComposeScene` / `RenderedPrompt` / `DialogueSessionSpec` schema 可测；方向/时段 layer 命中；每通注入用户本地时间；patch 不得覆盖 objective/forbidden
- 过程话术：Adapter 上报事件；引擎不计时；调试器可模拟三类事件
- 工具：Registry 含旧话机七支业务工具；挂机 FC 不进引擎
