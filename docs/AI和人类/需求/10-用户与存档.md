# 10. 用户与存档（PlayerProfile）

相关：[00-文档索引](./00-文档索引.md) · [12-记忆模型](./12-记忆模型.md) · [40-世界柱](./40-世界柱.md)

## 1. 为什么要管用户

引擎面向可插入多端的 RPG NPC 运行时。所有**经历态**必须按用户隔离：

- 各角色对该用户的记忆
- 剧情包进度与变量
- CallCardBoard（谁身上挂着什么卡）
- 该用户生成的世界 Lore、Facts、knowledge
- 调度中的回拨/意图队列

**内容定义**（角色模板、卡、剧情包）跨用户共享；**存档**不共享。

## 2. User（用户档案）

与 World 门面分离，单独管理。

```ts
interface User {
  userId: string
  nickname: string
  fullName?: string
  location?: {
    country: string
    province: string
    city: string
    district?: string
  }
  preferences?: Record<string, unknown>
  createdAt: string
  updatedAt: string
}
```

- 地理位置用于 **World Lore bootstrap**（见 [40](./40-世界柱.md)），不是 World 子系统里的「地址 store」。
- 称呼/姓名供 Composer 与角色对话使用；可被剧情/初识更新（经 Effect 或 MemoryCommit，须审计）。

## 3. PlayerProfile（存档）

每个 `userId` 一份存档。**v1 不做多存档槽**，API 不预留 `slotId`。

```ts
interface PlayerProfile {
  schemaVersion: 1
  userId: string
  user: User

  characters: Record<string, CharacterRuntime>
  stories: Record<string, StorySave>
  callCards: {
    board: CallCardBoard
  }
  telephony?: {
    redialSlot?: { agentId: string; cardId?: string }
  }
  world: {
    lore: WorldLoreDoc | null
    facts: WorldFact[]
    knowledge: Record<string, string[]>
  }
  schedule?: {
    clockMs: number
    intents: ScheduledIntent[]
  }
  /** 研究承诺（工具 create_research_commitment → Effect 写入） */
  research?: {
    commitments: ResearchCommitment[]
  }
  meta: {
    createdAt: string
    updatedAt: string
  }
}
```

### 3.0 ScheduledIntent（定稿）

```ts
type ScheduledIntent =
  | {
      kind: "once"
      intentId: string
      agentId: string
      cardId?: string
      topicHint?: string
      /** 相对 Profile.schedule.clockMs 的触发点（绝对逻辑时刻） */
      fireAtMs: number
      status: "pending" | "fired" | "cancelled"
    }
  | {
      kind: "recurring"
      intentId: string
      agentId: string
      cardId?: string
      topicHint?: string
      hour: number              // 0–23
      minute: number            // 0–59
      scheduleMode: "daily" | "weekly"
      weekdays?: number[]       // weekly：0=周日 … 6=周六
      status: "active" | "paused" | "cancelled"
    }
```

- `schedule_call_card` → 写入 `kind: "once"`（`fireAtMs = clockMs + minMs`）  
- `schedule_recurring_call` → 写入 `kind: "recurring"`；真定时由壳/调度 Sink 执行，引擎保存意图真源  

### 3.0a ResearchCommitment（定稿）

```ts
interface ResearchCommitment {
  id: string
  question: string
  notifyMode: "next_call" | "intent" | "urgent"
  status: "pending" | "in_progress" | "done" | "cancelled"
  sourceAgentId: string
  createdAt: string
}
```

- Effect `create_research_commitment` **只写** `Profile.research.commitments`（唯一落点）  
- **不**用 `patch_memory` 代替；可选另加 `patch_memory` 仅作角色口头备注，非研究真源  
- 联网执行在通话外 / 壳；完成后再改 `status` 并可选挂卡通知  

### 3.1 CharacterRuntime（摘记）

```ts
interface CharacterRuntime {
  agentId: string
  unlocked?: boolean
  // 记忆明细不在 Profile JSON；见 Memory SQLite（12 / 技术设计 20）
}
```

调试/API 若需「某角色记忆视图」，经 `MemoryPort` 查询组装，**不要**把万级 episodic 写回 `profile.save.json`。

### 3.2 StorySave 与 ActiveStoryLock

```ts
interface StorySave {
  packageId: string
  status: "inactive" | "active" | "completed" | "aborted"
  instanceId?: string
  variables: Record<string, unknown>
  completedCardIds?: string[]
  lock?: ActiveStoryLock | null
}

interface ActiveStoryLock {
  activeStoryInstanceId: string
  packageId: string
  lockLevel: "soft" | "hard"
  /** 锁定期仍允许剧情相关通话的角色；其它角色按 blockedPolicy */
  allowedAgentIds: string[]
  blockedPolicy:
    | "reject_call"           // 拒接 / 占线语义
    | "force_free_suppressed" // 允许拨号但抑制自由剧情挂卡
    | "allow_with_warning"    // 调试用
  reason: string
  startedAt: string
}
```

v1：同一 Profile **至多一个** hard lock。完整 soft/hard × blockedPolicy 决策表见 [19 §8.4](../技术设计文档/19-引擎宿主与会话模型.md)。

**不要**只用 `activeLock?: boolean`。

## 4. 铁律

1. **禁止**无 `userId` 的全局经历态单例。
2. 引擎 API 必须带 `userId` 或 Profile 上下文。
3. Studio：**无密码**；经 **UserGate** 选定用户（进故事 / 进调试）。编辑 Content 逻辑不依赖用户，但入口门禁建立 Profile 上下文。
4. 导出：**内容包** vs **SaveGame**；日志在 `data/logs/`，不整份塞进 SaveGame。
5. `redialSlot` 在 Profile，跨通话保留。

## 5. Studio 用户能力（经 UserGate，非独立必选页）

- 创建 / 选择 / 删除用户
- 编辑档案（姓名、称呼、地理位置）
- 重置存档（确认）；导入 / 导出 SaveGame

## 6. v1 范围

- 多用户本地：薄 Profile JSON + **Memory SQLite**；无账号密码 / 云同步
- Lore：建用户有 location 时 bootstrap；失败用 fallback（见 [19 §12](../技术设计文档/19-引擎宿主与会话模型.md)、[40](./40-世界柱.md)）
