# 40. 世界柱（瘦世界）

相关：[10-用户与存档](./10-用户与存档.md) · [11-角色与社交](./11-角色与社交.md) · [01-CallCard数据模型](./01-CallCard数据模型.md)

## 1. 世界管什么、不管什么

**管：**

| 子域 | 说明 |
|------|------|
| **Lore** | 世界背景设定；可按用户地理位置 bootstrap |
| **Facts** | 结构化世界事实 + 每角色 knowledge 索引（数据在 Profile） |
| **WET** | 事件追加、分发、幂等、可重放（逻辑） |
| **Clock / Schedule** | 逻辑时钟与延迟意图（回拨等） |

**不管（已挪走）：**

| 旧概念 | 新归属 |
|--------|--------|
| Social Graph | [11 角色一对多](./11-角色与社交.md) |
| User Address | [10 User](./10-用户与存档.md) |
| Relay 硬模块 | [CallCard 挂卡+上下文](./01-CallCard数据模型.md) |
| 专家引荐上帝模块 | 剧情包 + CallCard + social.canIntroduce |
| CallCardBoard | Profile.callCards |
| 角色长期记忆 | Profile.characters[].memory |

## 2. Lore（世界背景）

参考旧「stories bootstrap」产品点，在新引擎正规化：

```ts
interface WorldLoreDoc {
  version: 1
  source: "llm" | "fallback" | "manual"
  generatedAt: string
  location: User["location"]
  sharedPremise: string
  perspectives: Record<string, string[]>  // agentId → 1..n 短句
  characters?: Record<string, { displayName: string; blurb: string }>
}
```

### 2.1 Bootstrap

```text
输入：User.location + CharacterDef 列表 +（可选）模板
  → LoreBootstrapPort（文本模型或 fallback）
  → 写入 Profile.world.lore
```

**触发（定稿，见 [19 §12](../技术设计文档/19-引擎宿主与会话模型.md)）**：

1. 新建用户且有 `location` → 自动 bootstrap  
2. 改 `location` → 提示是否重新生成（默认不自动覆盖）  
3. 世界台 / 调试手动「重新生成」

失败：`lore = null` + 内容库 fallback；UI 黄条；写入 `data/logs/`；不阻塞建用户。

- 每用户一份。
- Composer 注入：`sharedPremise` + 当前角色 `perspectives`。

## 3. WorldFact 与 knowledge

数据在 `PlayerProfile.world`；引擎提供 schema 与 API。

```ts
interface WorldFact {
  factId: string
  type: string
  value: boolean | string | number | object
  visibility: "global" | "story" | "agent" | "temporary"
  scope?: { storyInstanceId?: string; agentId?: string }
  sourceEventId?: string
  confidence?: number
  expiresAt?: string | null
  updatedAt: string
  tags?: string[]
}
```

- `knowledge[agentId]`：该角色知道的 `factId` 列表。
- Composer：**按 knowledge 过滤**，禁止全知。
- Effect：`set_world_fact`、`update_npc_knowledge` 写 Profile。
- 剧情结束可将 story 可见事实提升为 global（effect / 包 exports）。

## 4. WET（逻辑事件）

- 业务事件可反映在调试时间线；**排障持久化**以 `data/logs/engine-*.jsonl` 为主（见 [19 §14](../技术设计文档/19-引擎宿主与会话模型.md)），不强制整份 WET 塞进 SaveGame。
- 电话壳上报 `call.*`；引擎写 `story.card.*`、`world.*`、`intent.*` 等。
- 稳定 handler key、幂等；与 Effect ledger 一致（见 [02](./02-运行时协议.md)、[16](./16-错误处理与幂等事务.md)）。

**不做旧话机迁移**。

## 5. Clock 与调度

- `Profile.schedule.clockMs` + `intents[]`（`ScheduledIntent`：`once` | `recurring`，见 [10](./10-用户与存档.md)）
- `schedule_call_card` → `kind: "once"`；触发点 = `clockMs + minMs`
- `schedule_recurring_call` → `kind: "recurring"`；真闹钟由壳执行
- 调试器可快进 once；完整日常 tick 非 v1 必达

## 6. 实现阶段（引擎内）

| 阶段 | 内容 |
|------|------|
| W0 | FactStore + knowledge + 事件日志 |
| W1 | Lore bootstrap 端口 + fallback + 投影 |
| W2 | Clock + scheduled intents |
| W3 | WET retry/replay 加固 |
| W4 | 可选日常 Tick |
