# 14. 资源与媒体资产

相关：[01-CallCard数据模型](./01-CallCard数据模型.md) · [03-编辑器界面需求](./03-编辑器界面需求.md) · [19](../技术设计文档/19-引擎宿主与会话模型.md)

## 1. 为何单独成文

卡上已有 `playbackClipId` / `systemPromptClipId`，但缺少资产库 schema 与校验。`playback_only` 缺文件会直接坏体验。

## 2. 真源（定稿）

**全局资产库**为唯一元数据与文件真源：

```text
data/assets/
  meta/<assetId>.json     # AssetMeta
  files/...               # 二进制 / 旁路文件（uri 相对此根或 workspace）
```

| 层 | 存什么 |
|----|--------|
| `data/assets/meta/` | `AssetMeta` 真源 |
| 卡 / effect | 只引用 `assetId` |
| `story.conf` / 包 | 可选 `assetRefs: string[]`（本包用到的 id，便于导出打包） |
| 包内 | **禁止**再维护一份完整 `assets: AssetMeta[]` 当第二真源 |

```ts
interface AssetMeta {
  assetId: string
  kind: "wav" | "tts" | "prompt_clip" | "image"
  /** 相对 data/assets/ 或 workspace 可解析路径 */
  uri: string
  displayName?: string
  durationMs?: number
  transcript?: string
  locale?: string
  hash?: string
  meta?: Record<string, unknown>
}
```

## 3. 与 CallCard

- `interactionMode: playback_only` → **必须**有可解析的 `playbackClipId`（校验 error）
- `hybrid` → 播放阶段同要求；对话阶段按 toolPolicy
- `play_system_prompt` effect → `clipId` 必须在全局资产库

## 4. 校验规则

| ruleId | 级别 | 说明 |
|--------|------|------|
| `ASSET_UNKNOWN` | error | 卡/effect / assetRefs 引用不存在的 assetId |
| `ASSET_URI_MISSING` | error | uri 指向缺失文件 |
| `ASSET_KIND_MISMATCH` | warning | playback 引用了 image 等 |
| `PLAYBACK_NO_ASSET` | error | playback_only 无 playbackClipId |
| `ASSET_PACKAGE_INLINE` | warning | 包内出现完整 assets[] 内联（应迁到全局库） |

## 5. Studio 资源库

- 独立资源台或画布左侧：增删改**全局**资产元数据；文件写入 `data/assets/files/`
- 包导出时可按 `assetRefs` / 卡引用收集子集打进 zip
- 预览：时长、transcript；删除前检查引用

## 6. 运行时

- 引擎只认元数据与引用完整性
- 真播放由壳 / Studio Adapter 执行
- 调试器可「模拟播放完成」而不真播

## 7. v1 范围

- 必做：wav + prompt_clip + 全局库引用校验
- image / 复杂 TTS 流水线可选
