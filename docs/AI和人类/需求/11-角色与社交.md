## 1. 二分

| 层 | 存放 | 内容 |
|----|------|------|
| **CharacterDef** | 内容库 | 身份、人设长文、默认开场矩阵、过程话术、能否可拨、Free 卡、设计期社交 |
| **CharacterRuntime** | PlayerProfile | 解锁态、记忆、对该用户的经历 |

不可把「可引荐强叔叔」（设定）和「已经向用户介绍过强叔叔」（经历）混在同一张运行时表里只存一份。

## 2. CharacterDef（架构字段）

磁盘：`data/characters/<agentId>.json`（角色内容单文件；Free 卡见 §4）。

```ts
interface CharacterDef {
  agentId: string
  displayName: string
  dialable: boolean              // false：可提及/剧情，用户通讯录不可拨

  /**
   * 仅设定/社交/Lore 提及：不可拨、无 Free 卡、不进通讯录。
   * true 时 freeCardId 可省略；dialable 必须为 false。
   */
  isNarrativeOnly?: boolean

  /** 身份档案（给 Composer / UI） */
  identity: {
    gender?: "female" | "male" | "non_binary" | "unspecified"
    age?: number
    ageNote?: string
    birthday?: string
  }

  persona: {
    systemPrompt: string
    personalityCode?: string
    profession?: string
    traits?: Record<string, string>
    speakingStyle?: string
    voiceNotes?: string
    voiceId?: string
    exampleLines?: string[]
  }

  /** 可通话角色必填；isNarrativeOnly 时可省略 */
  freeCardId?: string
  defaultToolPolicy?: ToolPolicy

  /**
   * 角色级默认开场 / 时段风格
   * 闲聊与「本通卡未写 opening」时使用；结构与卡上 promptScenes 对齐
   */
  defaultPromptScenes?: PromptSceneLayer[]

  /**
   * 通话过程话术（非剧情出口）
   * 由壳/Adapter 上报静默、超长通话等事件后选用
   */
  callFlowPrompts?: {
    longSilence?: PromptVariant[]
    longCallNudge?: PromptVariant[]
    preHangupFarewell?: PromptVariant[]
  }

  /** Adapter 计时阈值；缺省用引擎/壳默认（见 50） */
  callFlowPolicy?: {
    silenceTimeoutMs?: number
    longCallNudgeMs?: number
    preHangupHintBeforeEndMs?: number
  }

  social: SocialEdge[]
  meta?: {
    phoneNumber?: string
    avatarAssetId?: string
    tags?: string[]
  }
}

interface SocialEdge {
  targetAgentId: string
  canKnow: boolean
  canMention: boolean
  canIntroduce: boolean          // 允许引荐（替代旧 canRelayTo）
}

/** 过程话术变体；可再带时段 */
interface PromptVariant {
  variantId: string
  timeBuckets?: TimeBucket[]     // 不写 = 全天
  text: string
  kind?: "speakable" | "private_hint"
}
```

`PromptSceneLayer` / `TimeBucket` 与 [01](./01-CallCard数据模型.md) 同构，便于同一套 Composer。

### 2.1 一对多组织方式

- 每个角色维护自己的 `social[]`（指向其他 `agentId`）。
- Studio 角色台：选中角色 → 编辑社交边（多选目标 + 三开关）。
- 校验：`targetAgentId` 必须存在于角色库（含 `isNarrativeOnly: true` 的角色）。

### 2.1a narrative-only

| 规则 | 说明 |
|------|------|
| `isNarrativeOnly: true` | 只出现在设定、社交边、Lore；**不可** `user_dial` |
| `dialable` | 必须为 `false` |
| `freeCardId` | 可省略；有则校验 warning |
| 解锁 | `unlocked` **不能**使 narrative-only 变为可拨（永久不可拨） |

例：剧情里提到的「小雨的邻居」可建 narrative-only 角色，无需 Free 卡。

### 2.2 dialable / unlocked

- `dialable: false`：设计默认不可拨；可被提及、可出现在卡上下文、可被 Lore 描述。
- **有效可拨**：
  ```text
  effectiveDialable =
    !def.isNarrativeOnly
    && (def.dialable === true || runtime?.unlocked === true)
  ```
- `user_dial` 看 `effectiveDialable`；未通过 → `CHARACTER_NOT_DIALABLE`。
- `agent_outbound` / `simulate_start` 不受 dialable/unlocked 限制（仍受 narrative-only：**禁止**对 narrative-only 发起外呼，除非调试 force）。
- **写入**：挂机后 Effect `set_character_unlocked`；禁止通话中改。新建 Runtime 缺省未解锁。
- narrative-only：**永不**因 unlocked 变为可拨。

## 3. 引荐 / 带话（不用 Relay 子系统）

设计期：`canIntroduce: true` = 设定上**允许**往该目标导。

运行期用 CallCard：

```text
出口 effects:
  attach_call_card(targetAgent, nextCard)   // 卡上写清要传的话
  set_redial_slot(targetAgent)              // 可选
  schedule_call_card(...)                   // 可选：对方主动打来
```

Composer 可读 SocialEdge，抑制「设定上不认识却乱提」；是否已经介绍过看 Profile 记忆/facts。

## 4. FreeCallCard 路径（钉死）

- 每个可参与通话的角色必须有 `freeCardId`（`isNarrativeOnly` 除外）。
- **磁盘定稿**：Free 卡放在角色目录旁，不进故事包：

```text
data/characters/<agentId>.json           # CharacterDef
data/characters/free-cards/<freeCardId>.s-card.json
```

- `freeCardId` 全局唯一；`cardKind` 必须为 `free`。
- 无 pending 剧情卡时 Resolver 使用该 Free 卡（见 [02](./02-运行时协议.md)）。
- Free 卡可自带 `promptScenes`；未写 opening 时回落 `CharacterDef.defaultPromptScenes`。
- Free 卡 `toolPolicy`：**默认开放** Registry 中所有 `allowedCardKinds` 含 `free` 的工具（`inherit_free` / 未写策略均按此解析）。作者可收紧为 allowlist；实机约定以全开为常态。见 [13](./13-工具与能力注册.md)。

## 5. callFlowPrompts（通话中过程话术）

与挂机后 Exit 的 `hangup_early` **不同**：这里是**通话中**静默/过长/收尾提示。

```text
壳或 Adapter 上报：
  silence_timeout | call_duration_threshold | pre_hangup_hint
    → 按 kind 取 callFlowPrompts.* 
    → 可再按 TimeBucket 筛 PromptVariant
    → 注入下一轮或播一句 speakable
```

事件名建议进 DialogueEvent 扩展（见 [50](./50-对话与模型适配.md)）；不写 Profile。

## 6. Studio 角色台（信息架构）

```text
/characters → 列表
  → 选中角色
     基本：显示名、gender、age/ageNote、dialable、号码、头像
     人设：systemPrompt、speakingStyle、样例句、音色
     默认开场：表格编辑 defaultPromptScenes（方向 × 时段）
     通话过程：longSilence / longCallNudge / preHangupFarewell
     社交：social[] 三开关
     Free 卡：跳转编辑 freeCardId 对应 .s-card.json
     （调试）当前用户下该角色 Memory 摘要
```
