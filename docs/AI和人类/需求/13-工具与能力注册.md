# 13. 工具与能力注册（ToolRegistry）

相关：[01-CallCard数据模型](./01-CallCard数据模型.md) · [15-动态出口与通话中工具事件](./15-动态出口与通话中工具事件.md) · [50-对话与模型适配](./50-对话与模型适配.md) · [19](../技术设计文档/19-引擎宿主与会话模型.md)

## 1. 问题

卡上仅有 `toolPolicy.allowedToolIds: string[]` 不够：编辑器不能只填自由字符串；校验器需知道工具是否存在、是否允许用于 story/free、是否在 `playback_only` 下禁用。

## 2. ToolRegistry 条目

```ts
interface ToolDefinition {
  toolId: string
  displayName: string
  description: string
  inputSchema: unknown          // JSON Schema / Zod 导出
  allowedCardKinds: Array<"free" | "story" | "system">
  /** 为 false 时任何 playback 阶段不可用 */
  allowedInPlayback: boolean
  runtimeGuards?: {
    requiresStoryLock?: boolean
    requiresCanIntroduce?: boolean
    maxCallsPerSession?: number
  }
  /**
   * 工具语义：
   * - register_exit: 只登记 RuntimeExitCandidate（默认推荐）
   * - session_local: 仅影响本通会话展示（极少）
   */
  behavior: "register_exit" | "session_local"
  /** 登记出口时的模板（可选）；挂机才实例化为 Effect */
  exitCandidateTemplate?: {
    exitKind?: ExitKind
    priority?: number
    effectTemplateIds?: string[]
  }
}
```

- `register_exit`：通话中只登记 candidate，挂机后 Effect。  
- `session_local`：**只读或仅本通展示**（如记忆召回）；**不写 Profile / Memory DB 业务态**（查询结果回给模型即可）。

**v1 真源 = 引擎代码内置**（`builtinRegistry` / `@airpc/rpg-engine` 门面导出）；内容包可扩展须另立项接线，当前 **未知 toolId = 校验 error**。

**与旧话机差异（定稿）：**  
doubaoSister 在通话中 FC handler **立刻写 store/调度/挂机**。本引擎一律改为：通话中只 `register_exit` → 挂机后 ExitSelector / FreeCallPostPipeline → Effect（+ Sink）。挂机类 FC **不进**引擎 Registry（见 §7.2）。  
旧仓是**功能参考**，不是实现继承；inputSchema 见 §6.1。

## 3. 与 toolPolicy

```ts
interface ToolPolicy {
  mode: "inherit_free" | "allowlist" | "deny_all"
  allowedToolIds?: string[]
}
```

解析规则：

1. `playback_only` → 强制无工具（等价 deny_all）
2. `deny_all` → 空列表
3. `allowlist` → 仅 `allowedToolIds` ∩ Registry，且满足 `allowedCardKinds`
4. `inherit_free`（**Free 卡默认**；未写 toolPolicy 的 free 卡同此）→ Registry 中全部 `allowedCardKinds` 含 `free` 的工具  
   - **实机约定**：Free 常态全开；Studio 允许作者改成 allowlist 收紧

编辑器：`allowedToolIds` **下拉自 Registry**，禁止手打未知 id（高级模式可警告）。

## 4. 校验规则（补充）

| ruleId | 级别 | 说明 |
|--------|------|------|
| `TOOL_UNKNOWN` | error | allowedToolIds 含未注册 id |
| `TOOL_KIND_MISMATCH` | error | 工具不允许该 cardKind |
| `TOOL_PLAYBACK` | error | playback_only 仍声明工具 |
| `TOOL_DIRECT_EFFECT` | error | 试图把工具标成通话中直接 effect |

完整规则总表见 [08-内容校验规则](./08-内容校验规则.md)。

## 5. Studio

- 工具台或设置页：查看内置 Registry（只读 + 包扩展）
- 卡属性：多选工具；显示 description
- 调试器：当前卡可用工具列表；可手动登记等价 candidate

## 6. 工程约束

- 壳侧 Realtime `session.update` tools 由 **Adapter** 从 Registry 投影为厂商 schema；引擎不写 DashScope/OpenAI 嵌套格式
- 工具回调结果：引擎只返回「已登记 / 拒绝」语义；`function_call_output` 续说由 Adapter 封装

### 6.1 inputSchema 真源（定稿）

```text
需求真源：本文 §7 参数要点（语义与必填字段）
实现真源：引擎内 Zod（或等价）建模 → 导出 JSON Schema 给 Adapter
旧仓 schema：仅设计时对照「缺没缺字段」（如 weekdays、notify_mode）；对得上即可
```

- **禁止**把 doubaoSister 的 JSON Schema / config 文件拷进本仓当实现  
- 需求文档保持参数摘要即可；schema 全文是实现产物，不必堆进需求文  
- Adapter 只消费引擎导出的 schema，不直读旧仓文件

## 7. v1 内置工具清单（能力对标 doubaoSister）

来源对照：旧仓 `buildAllSessionTools` 及各工具 config（**只读对标，不搬文件**）。  
**toolId 可保持旧名**，便于和实机能力清单对照；语义与执行按本引擎铁律重绑（`register_exit` → 挂机 Effect）。

### 7.1 进引擎 Registry（`behavior: register_exit`）

默认 `allowedCardKinds: ["free","story"]`（system / playback 阶段禁用）；`allowedInPlayback: false`。

| toolId | 显示名 | 功能（旧 description 摘要） | 参数要点 | 挂机后 Effect 方向 |
|--------|--------|------------------------------|----------|-------------------|
| `refer_to_expert` | 安排专家回电 | 安排目标角色稍后主动来电（引荐） | `target_agent_id`, `topic_hint`；可选 `referral_style` | `set_character_unlocked` + `schedule_call_card`（目标 Free/剧情卡）+ 可选目标卡 `attach`；**不设 Relay**；须 `social.canIntroduce`（guard） |
| `share_expert_number` | 已口播专家号码 | 用户已知晓对方号码（报号路径） | `target_agent_id`, `topic_hint` | `set_character_unlocked`；可选 `update_npc_knowledge` / `patch_memory` hints |
| `schedule_reminder_call` | 预约回电提醒 | X 分钟/小时后回电提醒某话题 | `topic_hint`；`delay_minutes` **或** `delay_hours`；可选 `confirm_phrase` | `schedule_call_card`（本通角色或约定角色；delay → minMs） |
| `schedule_recurring_call` | 登记重复外呼 | 每天/每周定时外呼（旧 corn） | `topic_hint`, `hour`(0–23), `minute`, `schedule_mode`:`daily`\|`weekly`；weekly 要 `weekdays`；可选 `job_id` | Effect `schedule_recurring_call` → `ScheduledIntent.kind: "recurring"`；壳 Sink 执行真定时 |
| `record_shared_secret` | 登记共同秘密 | 只存代号与回忆提示，不含完整秘密 | `label`, `recall_hint` | `patch_memory`（semantic / commitments）或 fact；禁止通话中写 |
| `create_research_commitment` | 研究承诺 | 挂机后查资料再告知 | `question`；可选 `notify_mode`:`next_call`\|`intent`\|`urgent` | Effect `create_research_commitment` → **唯一**写 `Profile.research.commitments`（见 [10](./10-用户与存档.md)）；联网在通话外 |
| `record_user_name` | 登记用户称呼 | 孩子自报昵称/姓名 | `nickname`；可选 `full_name` | Effect `update_user_profile`（写 `Profile.user`）；可另 `patch_memory` 本通角色「初识」备注 |

**候选模板约定：**

- 纯动态 candidate 须自带 `effects`（由 Registry 模板在登记时展开或挂机实例化）。
- `refer_to_expert` / `share_expert_number`：`runtimeGuards.requiresCanIntroduce: true`（对 `target_agent_id`）。
- 同通 `refer_to_expert` 与 `share_expert_number`：旧仓互斥路径；v1 校验 **warning**（可并存但调试器标黄）。

**Free 默认：** `inherit_free` → §7.1 业务工具 + §7.1b 记忆召回（凡 `allowedCardKinds` 含 free）。  
**Story 默认：** 建议 allowlist；记忆召回可按卡开放。

### 7.1b 记忆召回（`behavior: session_local`，只读）

**不进**旧话机业务 FC 列表；为本引擎补强（旧仓无检索、仅硬截断）。  
算法与 FTS DDL 见 [20 §5](../技术设计文档/20-记忆存储与投影.md)。

| toolId | 显示名 | 参数要点 | 行为 |
|--------|--------|----------|------|
| `search_memory` | 搜索记忆 | `text_query?` 与时间窗 `from`/`to` **至少一个**；可选 `kinds[]`；`max_results`（默认 5，硬上限 10） | `MemoryPort.search` → 短列表（id+**≤200 字**摘要+时间）；**查 SQLite FTS**；禁止全表灌 prompt |
| `get_memory_by_id` | 按 id 取一条 | `entry_id` | `MemoryPort.getById`；须同 user+agent；正文 **≤500 字** |

- 通话中可用；**不**登记 Exit、**不**写库（除审计 log）。  
- `userId`/`agentId` 由 Host 注入；v1 禁止跨角色搜。  
- 超时/过多 → 工具错误信息，不拖死通话。  
- **v1 = 字面 FTS**（可 LIKE 降级）；**不做**向量语义检索。  

**工具描述应引导模型：**

1. 「N 年前」→ 先估 `from`/`to`，再带 `text_query`  
2. 先 `search_memory`，摘要不够再 `get_memory_by_id`  
3. 空结果 → 承认没查到或换词；禁止假装记得  

详见 [12](./12-记忆模型.md)、[20 §5](../技术设计文档/20-记忆存储与投影.md)。

### 7.2 不进引擎 Registry（壳 / Adapter 专用）

旧仓挂机 FC；由话机 Realtime Adapter 直接处理，**不是**剧情 Exit：

| toolId | 功能 | 落点 |
|--------|------|------|
| `request_hangup` | 违禁/策略结束通话（`reason_code` 等） | 壳：farewell → 挂断；可上报 `phoneFlags` / Outcome |
| `end_conversation` | 双方已道别，自然结束 | 壳：自然挂机 |
| `complete_referral_handoff` | 引荐已登记后催挂机 | 壳：须已有 `refer_to_expert` 成功留下的 pending；引擎只关心引荐 candidate |

Studio Manual / 文本调试：用挂机按钮 + Outcome flags 模拟，不必实现这三支 FC。

### 7.3 旧 → 新行为对照

```text
旧：FC → handler 立刻写 store / 调度 / 挂机 → function_call_output 续说
新：FC → ToolRuntime → RuntimeExitCandidate
      → session.ended
      → Story: ExitSelector | Free: PostPipeline（Commit + 有候选则 Exit）
      → Effect (+ Sink)
```

## 8. v1 范围与非目标

- **必做：** §7.1 七支 + §7.1b 记忆召回（FTS 字面检索）；Zod 建模；挂机后 Effect 可测  
- **可后做：** research 真联网、corn UI、密钥加密、**向量召回**  
- **不做：** 通话中直接 Effect；挂机 FC 进引擎；拷贝旧 schema；全量 JSON 扫记忆灌模型
