# 项目第二步计划执行索引

本文回答：**初始阶段（P0–P6）之后，按什么顺序产品化「CallCard 剧情系统」**。  
这不是「初始阶段有缺陷要返工」，而是**下一步计划**：主机语义再闭合一层，编辑器从 JSON 投影变成作者工作台，调试台从 dump 变成剧情观测台。

产品「做什么」见 [需求/00](../需求/00-文档索引.md)；工程「怎么落」见 [技术设计/00](../技术设计文档/00-技术设计索引.md)；验什么见 [需求/06](../需求/06-验收标准.md)。  
**初始阶段勾选真源**仍是 [项目初始执行索引](./项目初始执行索引.md)（P6 = 简版画布/校验/导出已完成）。

**不写**长方案。每期只含：目标、必读、完成标准。一期做完再勾选，再开下一期。

对标 ≠ 迁移：旧话机仅能力参考；新壳另建。见需求 00 / 13。

## 总序（第二步）

```text
S0  口径修订（文档）：P6a / 目录名 / 工具真源
S1  Content 热重载 ≠ 清 Session／Profile
S2  entryMode / activation 参与 pending 选择
S3  agent_outbound + schedule/redial 推进（含 CallCard）
S4  playback_only 最小执行路径
S5  故事包 CRUD（创建 / 重命名 / 删除）
S6  结构化属性面板 + 连线出口类型
S7  schema 收紧 + 保存前 parse／validate
S8  角色台可写 + FreeCallCard 编辑
S9  调试台信息化 + 工具列表动态化
S10 画布 Start／End／泳道／撤销（体验层）
S11 Effect／世界柱增量（按需，勿阻断主线）
```

**推荐刀序：** 先 S0–S3（主机诚实）→ S5–S7（作者语义）→ S4 / S8 / S9 可交错 → S10–S11 后置。

## S1 · Content 热重载 ≠ 清运行时

**目标：** 编辑器保存 Content 不打断本通调试 Session / 已载入 Profile。

**必读：** [19](../技术设计文档/19-引擎宿主与会话模型.md) · [03 §10](../需求/03-编辑器界面需求.md)

**完成标准：**

- [x] `loadWorkspace` / Studio `reloadStudioWorkspace` **拆开**：content 缓存可刷；`sessions` / `profiles` / `activeByUser` 默认保留
- [x] 有测或可手测：编辑中保存卡后，同 user 活跃 Session 仍在
- [x] 显式「重置工作区／踢会话」若需要则单独 API，禁与普通保存绑定

---

## S2 · entryMode／activation 过滤

**目标：** 用户拨号与外呼挂卡不再「谁都可能接到第一张 pending」。

**必读：** [01](../需求/01-CallCard数据模型.md) · [02](../需求/02-运行时协议.md) · `pickPendingForUserDial`

**完成标准：**

- [x] `user_dial` 只挑 `inbound_user_dial` / `either`（及文档约定等价项）
- [x] 外呼／调度只挑 `outbound_auto` / `either`
- [x] `redialSlot` 命中后仍受 entryMode 约束；单测覆盖「错 mode 不得选中」

---

## S3 · agent_outbound＋schedule／redial 推进

**目标：** 「被动出口让角色打给用户」在 Host 可 resolve，且调度落到 CallCard。

**必读：** [19](../技术设计文档/19-引擎宿主与会话模型.md) · [02](../需求/02-运行时协议.md) · [10](../需求/10-用户与存档.md) ScheduledIntent

**完成标准：**

- [x] Resolve 支持（至少）`agent_outbound`（名以类型定稿为准）与 schedule tick／到期扫描入口
- [x] `schedule_call_card`／工具展开 **能指定或解析目标 cardId**；禁止仅 `topicHint` 充作剧情推进
- [x] redial／pending 路径与 S2 过滤一致
- [x] 单测或脚本：挂机效果写入调度 → tick → 可 begin 外呼卡（可 Manual）

## S4 · playback_only 最小执行路径

**目标：** 校验里的 playback 规则在运行时有对应相位，调试可「播完」。

**必读：** [01](../需求/01-CallCard数据模型.md) · [14](../需求/14-资源与媒体资产.md) · [04](../需求/04-调试器界面需求.md)

**完成标准：**

- [x] `interactionMode: playback_only`（或等价）→ Session `interactionPhase` 为 playback，非误标 dialogue
- [x] asset／`playbackClipId` 可解析（真播 wav 可桩）；调试器有「播放完成」→ Outcome／挂机入口
- [x] 工具在 playback 相位策略与现有 `invokeSessionLocal` 一致且可测
