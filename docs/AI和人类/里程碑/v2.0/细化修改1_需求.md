## 1. 本轮定位

### 1.1 要做什么

按刀序完成四段：

```text
0 引擎小改动（去时段桶、补 identity 全名/昵称）
1 AutoForm 升级（声明式 items[] + 自动绑 + 双逃生口）
2 角色编辑重新编排（对齐 CharacterDef + 记忆只读）
3 其它已有表单对齐 AutoForm
```

### 1.2 本轮不做

| 不做 | 说明 |
|------|------|
| 社交边结构化编辑 UI | `social[]` 可二期；本轮可不进角色详情主表单 |
| Free 卡深链编辑器 | 可保留只读/跳转占位，不做完整卡编辑 |
| 记忆写入 / 编辑 | 记忆区 **只读** 列表 + 分页 |
| 调试器整模块改版 | 除「为记忆只读选当前 userId」所需的最小引用外，不碰 `/debugger` 深化 |
| 电话壳 / Realtime 真链 | 音色枚举仅前端常量，不接真 Realtime |
| 照搬 Mithril 样式 / jquery | 只借鉴 AutoForm **编排方法** |

### 1.3 与第二步的关系

- 第二步（V2-S*/P*/C*/A*/U*/E*）已收口：mock CRUD + 初版 `FormSchemaRenderer`。
- 本轮 **升级** 表单基建（AutoForm），并按引擎真源 **重做** 角色详情字段；不是另起第三步「Host 全量写盘」。
- 角色 JSON **允许**写到 `data/characters/`（经 Next API 门面）；记忆 **只读** 查库。其它库（users/assets）本轮以 AutoForm 对齐为主，不强制本轮接真实写盘。

## 3. 已钉死口径

### 3.1 引擎：去掉时段桶，只留本地小时区间

- `PromptSceneLayer.match` **删除** `timeBuckets`；**只保留** `localHourRange: { from, to }`（半开区间：`h >= from && h < to`）。
- 删除 `TimeBucket` / `TimeBucketSchema` / `hourToTimeBucket`；`ComposeScene.localTime` 不再依赖 `bucket`。
- 过程话术变体与文档中凡引用「按时段桶筛选」的表述一并去掉。
- **旧内容含 `timeBuckets`：拒载**（校验失败）；仓内样例 JSON 本轮一次性改完。不做静默忽略。

### 3.2 引擎：`identity` 补全名 / 昵称

```ts
identity: {
  fullName?: string   // 全名
  nickname?: string   // 昵称
  gender?: string
  age?: number
  birthday?: string
  // ageNote 可保留兼容；新角色编辑 UI 不强制展示
}
```

### 3.3 Studio 编辑态：显示字段全必填

- 角色详情 **编辑态展示出来的字段** 全部必填（Formik validate + 必填星号）。
- 引擎 Zod 仍可对多数字段 optional；由 Studio 保存前保证填齐。
- 创建最小集可另定；详情页以完整表单为准。

### 3.4 数组类字段：多行列表编辑

| 字段 | 落盘形态 | UI |
|------|----------|-----|
| `persona.exampleLines` | `string[]` | 可增删的多行列表 |
| `callFlowPrompts.longSilence` 等 | `PromptVariant[]`（至少 `variantId` + `text`） | 可增删的多行列表 |
| `defaultPromptScenes` | `PromptSceneLayer[]` | 列表卡：增删、折叠展开、拖拽排序 |

场景优先级：拖拽顺序写回 `priority`（建议 `0,10,20…`），并与数组序一致。

### 3.5 音色

- 前端维护 realtime 音色枚举常量（label + value）。
- 引擎 JSON **只存** `persona.voiceId` 字符串。

### 3.6 记忆区

- **不在** `CharacterDef` 内。
- 按当前调试 `userId` + `agentId` **只读**查询 Memory；列表 + 分页。
- 禁止写入、禁止把记忆塞进角色 JSON。

### 3.7 AutoForm 编排契约

目标用法（示意）：

```tsx
<AutoForm
  formik={formik}
  mode="edit"
  enabled={true}
  items={[
    { label: "显示名", name: "displayName", comType: "TextField", required: true },
    {
      label: "系统人设",
      name: "persona.systemPrompt",
      comType: "AutoTextArea",
      required: true,
    },
    {
      label: "只读编号",
      name: "code",
      comType: "TextField",
      comProps: { /* 逃生：覆盖自动绑 */ },
    },
    {
      label: "头像",
      name: "meta.avatarAssetId",
      children: <AvatarUploadField />,
    },
  ]}
/>
```

必须具备：

1. `comType` → `ComsMap` 注册表  
2. 按 `name` **自动绑定**（含嵌套路径）  
3. **`comProps` 逃生口**：自动绑不够时局部手写 value/onChange  
4. **`children` 逃生口**：头像、列表卡、记忆表等复杂块  
5. 字段壳：label / 必填星 / 错误 / `add|edit|watch`  
6. 样式自有（MUI + scss module）；不照搬 Mithril Grid 皮肤  

旧 `FormSchemaRenderer` + `kind`：本轮升级为 AutoForm（可短暂 adapter，收口前删除双轨）。

## 4. 角色详情信息架构

### 4.1 头像

- 点击可上传（或选资源）；写入 `meta.avatarAssetId`。
- 自定义控件，经 AutoForm `children` 挂载。

### 4.2 基本信息（全必填）

| UI | 落盘 |
|----|------|
| 显示名 | `displayName` |
| 全名 | `identity.fullName` |
| 昵称 | `identity.nickname` |
| 性别（男/女/其它） | `identity.gender`（枚举与引擎对齐，UI 三档可映射） |
| 年龄（整数） | `identity.age` |
| 生日（年/月/日） | `identity.birthday` |
| 电话号码（整数串，非小数） | `meta.phoneNumber` |
| 音色（下拉） | `persona.voiceId` |
| 音色备注（自适应 textarea） | `persona.voiceNotes` |

### 4.3 栏目：提示词（全必填）

| UI | 落盘 |
|----|------|
| 系统人设长文 | `persona.systemPrompt` |
| 说话风格 | `persona.speakingStyle` |
| 样例句（列表） | `persona.exampleLines` |
| 职业 | `persona.profession` |
| 长静默话术（列表） | `callFlowPrompts.longSilence` |
| 超长通话催促（列表） | `callFlowPrompts.longCallNudge` |
| 预挂机告别（列表） | `callFlowPrompts.preHangupFarewell` |
| 场景提示词（列表卡） | `defaultPromptScenes[]` |

场景卡展开项：

| UI | 落盘 |
|----|------|
| 优先级（拖拽顺序） | `priority` |
| 呼入 / 呼出 / 呼入+呼出 | `match.callDirection`（`inbound` / `outbound` / `either`） |
| 本地小时区间 | `match.localHourRange`（**不要**时段桶） |
| 开场白提示词 | `patch.openingSpeakable` |
| 私有开场提示 | `patch.openingPrivate` |
| 情绪 | `patch.emotion` |
| 语气提示 | `patch.toneHint` |
| 追加可说 / 私有文本 | `patch.appendSpeakable` / `appendPrivate` |

### 4.4 记忆（只读）

- 列表 + 分页；字段对齐 Memory 读模型摘要即可。
- 依赖当前 `userId`；无 user 时明确空态文案。
