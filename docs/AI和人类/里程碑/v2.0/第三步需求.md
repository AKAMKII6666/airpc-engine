## 1. 本步定位

### 1.1 要做什么

```text
阶段 A · 内容真源落盘
  A1 正式五人 CharacterDef → data/characters/*.json（含 personalityCode）
  A2 第一幕故事包 wrong_number_act1 → data/storis-packages/wrong_number_act1/
     （story.conf.json + cards/*.s-card.json + canvas.layout.json）

阶段 B · 管道接通（列表 + 读写）+ 同步清 mock
  B1 角色库页 / 画布角色弹窗：增删改走 /api/characters；删角色相关 MOCK_* / 仅 mock 路径
  B2 玩家配置：验收 /api/users CRUD；删玩家列表相关 mock 残余
  B3 故事包 BFF：列包 / 读包 / 写包（整包保存）
  B4 故事列表 + 编辑器打开读盘、保存写盘；删包列表/打开路径上的 mockWorkbench、CanvasMockGraph 当真源用法

阶段 C · 编辑接线（读写通后再做）
  C1 属性浮窗 / 出口 / Effect / 归属改动进入会话图，且可经「保存」落盘
  C2 角色锚点与 participants / ownerAgentId 对齐磁盘角色
  C3 编辑器侧仅服务 mock 的种子/合并函数若已无引用则删除
```

### 1.2 本步不做

| 不做 | 说明 |
|------|------|
| Host / beginCall / Effect 真执行 | 编辑器写 Content，不假装调试闭环 |
| 调试器深化 | 不扩 `/debugger` 真会话；**调试器域 mock 可暂留**（本步清零范围不含 debugger） |
| 资源库落盘 | **资源库域 mock 可暂留**；本步不强制落盘、不强制清资源 mock |
| 导入导出正式管道 | 列表可展示真包；导入预检/导出打包可后置 |
| 引擎 schema / runtime 大改 | 角色字段沿用已有 `CharacterDef`（含已落地的 `personalityCode`）；CallCard 磁盘格式对齐现有 `golden_handoff` |
| 第二幕及以后内容 | 只落第一幕 `wrong_number_act1` |
| 自动保存 / 单卡增量 API | 第一版 **整包保存** 即可 |
| 把 `pkg_memory_bar_1` 当正式 packageId | 新包语义 id：`wrong_number_act1` |
| 「先接真源、mock 留着备用」 | **禁止**；见 §2.0 |

### 1.3 与第二步 / 细化修改的关系

| 前置 | 关系 |
|------|------|
| 第二步 | Formik / commonUi / 各库可操作原型 |
| 细化修改 4–6 | 画布节点、角色锚点、属性浮窗、放置 |
| 细化修改 7 | 出口 Effect 参数面板与效果边（本步负责能落盘） |
| docs/AI 记忆 | 第一幕拓扑与五人 agentId 为落盘蓝本；接通后回写「已非 mock」 |

**刀序铁律：** 阶段 A → B → C；**禁止**在 B 未通前深挖 C。  
**清 mock 铁律：** 某域真源接通的**同一任务内**删除该域 mock；禁止拖到收口再扫。

## 2. 已钉死口径

### 2.0 真源 + 删 mock（零残余）——全批铁律

| 项 | 口径 |
|----|------|
| 目标 | 做**真实业务接线**，不是继续用 mock 模拟业务 |
| 时机 | 对应模块 API/磁盘路径可用并完成验收点后，**立刻**删 mock |
| 删除范围（本步必须清零） | **角色库**、**玩家配置列表/CRUD**、**故事包列表**、**故事编辑器打开/保存种子** 相关 mock 源与引用 |
| 允许暂留 | **资源库** mock、**调试器** mock、设置页纯展示 mock（本步不接这些域） |
| 禁止 | 双轨（真源 + mock 开关并存）、re-export 兼容壳、`@deprecated 已迁至…` 空转发、仅改 import 留无引用文件 |
| 测试 | 改打 API / 磁盘夹具；禁止业务测继续依赖可变内存 `MOCK_*` 数组当真源 |
| 验收 | `rg`/引用检查：本步清零范围内无 `MOCK_CHARACTERS`、`listMockCharacters`、`listMockPackages`（作打开真源）、`commitUpdateCharacterMock` 等业务入口残留 |

落盘蓝本可继续对照 `docs/AI` 与历史 Canvas 拓扑描述；**蓝本文档 ≠ 运行时 mock 代码**。

### 2.4 故事包 BFF 与保存

| 项 | 口径 |
|----|------|
| 列包 | 扫描 `data/storis-packages/*/story.conf.json` |
| 读包 | conf + cards[] + layout（无 layout 时安全默认坐标） |
| 写包 | **整包保存**；禁止只改内存假装已保存 |
| 新建/删除包 | 至少支持打开已有包；新建/删包可作 B 末可选 |
| 路由默认包 | `wrong_number_act1` |
| 删 mock | 删除/停止以 `mockWorkbenchData`、`MOCK_STORY_PACKAGES`、`CanvasMockGraph` 初始数组作为**打开/列表真源**的路径；种子文件若无引用则删除或降为测试夹具（不得被 page 业务 import） |

### 2.5 编辑器两段接通

| 段 | 完成标准 |
|----|----------|
| **B 读写** | 打开 `wrong_number_act1` 与落盘一致；保存后刷新仍在；打开路径不依赖内存唯一种子 |
| **C 接线** | 浮窗/出口/Effect/归属 → 会话图 → 保存落盘；锚点来自磁盘角色 |

### 2.6 layout 最小契约（实现须遵守）

```text
canvas.layout.json
  schemaVersion
  packageId
  nodes[]: { nodeId?, cardId | kind(chapter_start|chapter_end), x, y }
  edges[]: { edgeId, edgeKind(story|role|effect), source, target, sourceHandle?, targetHandle?, label?, effect meta? }
  （可选）lanes / note
```

- **内容真源**在 cards / conf；layout **不**重复存 objective 全文。  
- chapter_start / chapter_end 可仅存 layout（或 conf 扩展，二选一须钉死）。  
- 效果边 meta 须能与 `exit.effects[].id` 对上。
