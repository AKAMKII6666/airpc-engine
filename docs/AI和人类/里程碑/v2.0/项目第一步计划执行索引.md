## 0. 已钉死口径

### 0.1 范围边界

| 做 | 不做（本步） |
|----|----------------|
| 引擎 ScheduleCard / recurring / 延迟外呼提前呼入 / end_story 章节切换 / EffectSink 确认语义 **全文** | 电话壳、Realtime、浅色主题 |
| 新建 `apps/studioV2`（workspace `@airpc/studio-v2`） | 在旧 `apps/studio` 上改成 V2、整仓迁移旧 UI |
| `08` 要求的 lint / check:comments / check:studio-structure / test:studio / quality:studio | 用旧 Studio 超标基线给 V2 豁免 |
| 按导向稿做各主界面 **静态页**（信息架构 + 暗色视觉） | 完整保存事务、Host 调试闭环、undo/redo、真实导入导出管道 |

### 0.2 产品硬共识（静态页也必须守）

- V2 不是旧 UI 重皮；首页是故事工程工作台。
- 故事编辑器 = 全屏自由画布；**不使用泳道作默认视图**。
- 角色：左侧锚点 + 顶部归属线；CallCard：左 parent / 右 exit / 顶角色。
- 出口是触点，可挂动作列表；JSON 仅高级视图。
- 内部 ID 由系统生成；主流程不手填。
- UI 默认暗色调色板（见 [色板参考.md](../../sudioV2.0版本/需求/ui/色板参考.md)）。

### 0.3 引擎硬共识（对照增强计划 §9）

- 剧情里只有卡驱动的一次性延迟外呼；禁止 topicHint 裸推进。
- 周期性外呼必须由 ScheduleCard（或明确 Free/Schedule 卡）驱动。
- StoryPackage = 章节；切换只由结束点（`end_story`）驱动；**不做**独立 `startStory` API。
- 章节结束后必须清旧 story pending；世界回到 Free 或下一章入口等待态。
- EffectSink 可等待确认；失败不得记 ledger `executed`；`critical` 可中止。

### 0.4 工程约定

- 目录：`apps/studioV2/`；npm 包名：`@airpc/studio-v2`。
- 栈：Next.js（与仓一致稳定线）+ React + TypeScript + MUI + scss module + `@xyflow/react` + Zustand；禁 Studio 内 barrel `index.ts`。
- 分层建议：`domain/` · `commands/` · `store/` · `services/` · `features/` · `app/`（见总览 §8.2）。
- 旧 `apps/studio` 保留为引擎验证台；本步不删。
- 测试：引擎测进 `packages/rpg-engine/tests/`；V2 测进 `apps/studioV2/tests/`（禁与业务文件平放）。

### 0.5 推荐刀序

```text
V1-E*  引擎全文（ScheduleCard → 延迟外呼 → 章节结束 → EffectSink → §7 测）
V1-P*  apps/studioV2 工程骨架 + 路由壳 + token
V1-Q*  质量门禁脚本（先于大面积静态页）
V1-U*  静态页按界面分批（一批验一批）
```

`group: order`：严格按表中从上到下取 ⬜；每轮最多 `batch_size` 条。

> **gbx 表解析注意：** 任务单元格勿同时出现英文字母 `id` 与「任务」，勿写 `status` / 「状态」子串（会被误判为表头而丢行）。细节写在 §4 映射或正文口径即可。
