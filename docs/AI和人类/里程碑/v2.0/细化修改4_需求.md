> **状态：** 已定稿，供实现与 Review 引用。

> **人类讨论来源：** [原始需求讨论记录](../../../人类/原始需求讨论记录)（约 764–778 行，2026-07-20 确认）。

## 1. 本轮定位

### 1.1 要做什么

```text
0 删重复角色入口：左侧角色轨 + 底栏「角色浮窗」；只保留画布角色节点
1 画布角色：新增/编辑复用 /characters 同一套 AutoForm（A1，可写 data/characters）
2 通话卡属性浮窗：加宽 + AutoForm 对齐 CallCardDefinition；去掉 debug/占位可编辑项
3 出口：按 exits[] 动态 Handle；每个出口 MUI Tooltip 显示名称+概要
4 包配置：独立 docker 浮窗；StoryPackageConf 字段只读 Label
5 资源浮窗：增删改复用 /assets 的 FormModal 与 bis
6 占位折叠区改为真实表单模块（promptScenes / exits / toolPolicy / schedule 等）
7 同步向导稿 02；标注第二步 E4/E5 被本批 supersede
```

### 1.2 本轮不做

| 不做 | 说明 |
|------|------|
| 通话卡 / 包 manifest / 画布布局写盘 | 不接 `data/storis-packages`、不写 `story.conf.json` |
| Host / EngineHost / 引擎 save API | 通话卡属性「应用到画布」仅会话内 |
| `packages/rpg-engine` 契约变更 | 字段对齐以 **只读引用** schema 为准 |
| 出口条件树完整编辑器 | `ExitCondition` 递归 UI 可只做概要 + Tooltip |
| Effect 完整编排器 | exits.effects 可列表 mock，结构对齐 `EffectSchema` |
| 调试器改版 | 不碰 DebuggerShell |
| 泳道 / 故事包地图 / Realtime 真链 | — |
| 重复实现角色/资源表单 | 必须复用已有 `bis` + `FormModal` + AutoForm items |

### 1.3 与细化修改 1–3 的关系

- **依赖** 细化修改 1 的 AutoForm、`PromptSceneListEditor`、`FormModal`。
- **复用** 细化修改 1 角色详情表单契约；细化修改 2/3 的 Modal / 列表模式。
- **不依赖** 细化修改 3 的 schedule API（故事编辑器本轮不涉及 Profile.schedule 编辑）。

### 1.4 与第二步需求 E4/E5 的关系

第二步 [第二步需求.md](./第二步需求.md) §4.1 中 **E4（左侧角色轨悬浮）**、**E5（可添加角色轨）** 已被本批 **supersede**：

- 角色列表与新增/编辑 **只在画布角色节点** 完成；
- 不再保留 `CharacterAnchorRail` 与底栏「角色浮窗」。

## 2. 已钉死口径

### 2.1 角色（画布唯一入口 · A1）

| 项 | 口径 |
|----|------|
| 删除 | `CharacterAnchorRail` 整棵 UI；`CharacterPickerFloat`；底栏 `IconPeople` |
| 保留 | 画布 `CharacterAnchorFlowNode` + 顶 role 连线 |
| 新增角色 | 复用 `/characters` 新建 `FormModal` + `createCharacterForm` / 等价 bis；落盘 `data/characters` 后同步加画布锚点节点 |
| 编辑角色 | 选中画布角色节点 → `FormModal` 或浮窗，复用 `characterDetailForm` / `CharacterDetailEditForm` 同款 items |
| 禁止 | 另起 `addEditorCharacterForm` 平行契约（可删并改 import） |

### 2.2 底栏 docker

| 保留 | 资源浮窗 icon |
| 新增 | 包配置浮窗 icon |
| 删除 | 角色浮窗 icon |
| 其它 | 新建卡/连线等 disabled 项可保留，tooltip 标明「界面预览未开放」 |

### 2.3 包配置浮窗（只读）

对齐引擎 `StoryPackageConfSchema`（`packages/rpg-engine/src/schema/callCard.ts`）：

| 字段 | UI |
|------|-----|
| `schemaVersion` | Label |
| `packageId` | Label |
| `title` | Label |
| `participants` | Label 列表（agentId） |
| `entryCardId` | Label |
| `assetRefs` | Label 列表 |
| `cards` | 只读摘要（cardId 数量或列表） |

说明：**包级元数据只读**；编辑单卡请点画布 CallCard。数据来自 mock / 现有 `findMockPackage` 投影即可。

### 2.4 资源浮窗

- 复用 `createAssetForm` / `assetDetailForm` 与 `/assets` 页相同 bis。
- 支持新建、编辑、删除（mock 或已有 assets API，与资源库一致）。
- 禁止在 `storyEditor/library` 内复制一套 asset 字段定义。

### 2.5 通话卡属性浮窗

**宽度：** 由现约 `300px` 增至 **380–420px**（或等价 `maxWidth`）。

**字段纪律：**

- **不得** 出现用户可编辑的 `validation` / `校验态`。
- **不得** 手填 `exitCount`；由 `exits.length` 推导，节点 footer 只读展示。
- **不得** 手填 `characterName` 作为真源；展示 `ownerAgentId` + 显示名，归属靠顶口连线。
- **不得** 使用 `entryLabel` 自由文本；改用 `entryMode` Select（中文标签）。
- **不得** 使用 `goalOneLiner`；对齐 **`context.objective`**（本轮目标）。
- **不得** 在单卡属性内嵌「包配置」折叠区。

**应对齐的 CallCard 字段（会话 mock 投影）：**

| 分组 | 字段 |
|------|------|
| 只读 | `cardId`、`ownerAgentId`、角色显示名、`cardKind` |
| 基本 | `title`、`entryMode`、`interactionMode` |
| context | `privateBrief`、`speakableBrief`、`background`、`premise`、`emotion`、`objective`、`forbidden[]`、`promptScenes`、`playbackClipId` |
| objectives | `requiredBeats[]` |
| toolPolicy | `mode`、`allowedToolIds` |
| exits[] | `exitId`、`title`、`exitKind`、`priority`、`condition`（概要）、`effects`（列表 mock） |
| schedule | 仅 `cardKind === schedule` 时展示 `ScheduleMeta` 字段 |

**校验态展示：** 仅节点 footer **computed badge**（前端 mock validator），不进 Formik。

**枚举中文：** 新建 `typeFiles/story/callCardLabels.ts`（或等价）统一 `entryMode` / `interactionMode` / `exitKind` / `cardKind` 标签。

### 2.6 画布 CallCard 节点与出口

- 出口 Handle **按 `exits[]` 动态渲染**，禁止长期写死 `exit_a` / `exit_b`。
- 每个出口 Handle 外包 **MUI `Tooltip`**：悬停显示 **出口名称（title 或 exitId）+ 概要**（可从 `exitKind` + `condition.op` 等人话拼接）。
- 节点卡片 footer 可只读展示出口数量与校验 badge。

### 2.7 章节节点

- `chapter_start` / `chapter_end` **不与** CallCard 共用同一套可编辑 mock 字段。
- 第一版：章节节点属性窗可保持轻量或独立小节；**禁止** 把 `validation` / `exitCount` 等 debug 字段塞进章节表单。

### 2.8 类型投影命名

- 逐步以 **`EditorCallCardProjection`**（或等价命名）替代 `MockCallCardNodeData` 语义；字段名贴近 `CallCardDefinition`。
- mock 图数据 `CanvasMockGraph` 随投影字段更新。
