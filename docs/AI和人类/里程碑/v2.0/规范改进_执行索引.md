本文回答：**Studio V2 主模块按一域一 store + shell/feature bis 收口**按什么顺序做、做到哪算完。

---

## 0. 已钉死口径

### 0.1 范围边界

| 做 | 不做（本轮） |
|----|----------------|
| 一域一 Zustand + shell/feature bis | 引擎 schema / 语音留言运行时（另索引） |
| 编辑器双层 flush + 保存真源 | 全站禁 useState |
| 清 layering baseline 目标文件 | 旧 `apps/studio` 强制同迁 |
| Provider 挂多域 store | 上帝 store 扩业务切片 |

### 0.2 推荐刀序

```text
V2-LY-1    骨架：stores/<domain> 目录约定 + AppProviders 可挂多 store；shellBis 空目录说明
V2-LY-2    编辑器 store：会话/conf/dirty/validation 切片 + 结果型 actions
V2-LY-3    编辑器 shell bis：打开包灌 store；听 refresh；替代 page hook 真源
V2-LY-4    编辑器双层：画布 throttle+flush；保存先 flush；feature bis 拆分；出 baseline 编辑器项
V2-LY-5    角色库 store + characters.shell.bis
V2-LY-6    角色库 feature bis（create/save/delete/memory/schedule）+ UI 去直引；出 baseline
V2-LY-7    用户库同构收口
V2-LY-8    资源库同构收口
V2-LY-9    包列表 / 导入导出收口
V2-LY-10   调试器 store + shell；DebuggerShell 去 mock 直引
V2-LY-11   工作台 / 设置扫尾（直引 ajax/mock）
V2-LY-12   baseline 重扫趋零 + 需求 §5 勾选 + quality:studio 绿
```

`group: order`：严格按表取 ⬜。

> **gbx 表解析注意：** 任务单元格勿写英文 `status`、勿写「状态」子串。
