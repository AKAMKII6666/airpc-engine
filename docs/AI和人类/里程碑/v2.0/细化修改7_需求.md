> **状态：** 已定稿，供实现与 Review 引用。  
> **性质：** 在细化修改 4/5/6 已收口基础上，把出口 **Effect 列表从「枚举下拉 + 自由摘要 mock」升级为「每种 effect 有专属参数面板」**；纯 Studio 前端会话 mock，字段对齐引擎 `EffectSchema` 意图（只读镜像），**不写盘、不改引擎**。

## 1. 本轮定位

### 1.1 要做什么

```text
0 Effect 契约升级：EditorExitEffectProjection 从 {id, effect, summary} → 带按 effect 判别的 params
1 每种 effect 一套专属参数面板：目标下拉 / 表单字段 / 无参说明，替代当前「死」的单摘要框
2 attach_call_card / unmount_call_card：modal 列表（可配多条）+ 与画布连线双向同步（增删改）
3 set_redial_slot 等目标型 effect：角色 / 卡下拉选择器（禁自由文本手填 id）
4 每个 effect 枚举选项挂 tooltip：一句话解释「这条 effect 干什么、写到哪」
5 summary 由 params 自动派生人话预览（仍可覆盖），供节点 Tooltip / 列表预览
```

### 1.2 本轮不做

| 不做 | 说明 |
|------|------|
| 写盘 storis-packages / Host save | 保持会话 mock；params 仅内存投影，为将来序列化「预备」而非落库 |
| 引擎 schema / runtime 改动 | 不碰 `packages/rpg-engine`；字段名对齐 `EffectSchema` 但只读镜像，不 import 引擎值 |
| 完整 ExitCondition 条件树编辑器 | 出口条件仍用 `conditionSummary` 文本（refine 后续） |
| Effect 真执行 / 预览 | 不在 Studio 跑 `executeEffects`；面板只编辑参数 |
| 校验参数是否指向真实 Profile 状态 | 只做「必填 / 枚举 / id 来自画布」级前端 mock 校验，不查引擎运行时 |
| 新增 effect 种类 | 仅覆盖引擎现有 `KNOWN_EFFECT_NAMES` 14 种 |

### 1.3 与 refine4/5/6 的关系

- **依赖** refine4 画布 CallCard / 角色锚点、属性浮窗、资源与包配置浮窗。
- **依赖** refine5 出口列表编辑器（`ExitListEditor` / `ExitListRow` / `ExitEffectsMockList`）与枚举下拉、`nextPackageId` / `nextEntryCardId` 的包/卡下拉数据源。
- **依赖** refine6 放置创建与画布节点（角色锚点、CallCard 节点作为下拉选项来源）。
- **前置微调（已在会话内直接完成，非本批任务）：** 框选多卡只挪位置不弹属性面板；`toolPolicy?.mode` 空值兜底。
- **复用** 已有的删除确认 Modal 交互（CallCard 角标 × → 确认弹窗）作为 attach/unmount 行删除样式参考。

## 2. 已钉死口径

### 2.1 Effect 面板三分类

| 类别 | 含义 | 面板形态 |
|------|------|----------|
| **A 目标选择型** | 需要指向画布上的角色 / 卡 | 角色下拉 + 卡下拉 + 少量标量；下拉选项来自画布节点 |
| **B 表单参数型** | 纯标量 / 枚举字段，不指画布连线 | AutoForm 字段面板（TextField / Select / IntegerInput / Switch） |
| **C 无参数型** | 语义固定、作用于当前会话卡 | 只显示说明文案，无输入 |
| **连线同步子型** | A 类中 attach/unmount：与画布边双向同步 | modal 列表 + attach/unmount 边（见 §2.3） |

### 2.2 十四种 effect 的参数与面板（真源：引擎 `effectExecutor.ts`）

> 字段名与「必填」严格对齐引擎运行时读取；本轮为会话 mock，缺失必填项仅前端角标提示，不阻断保存。  
> **id 类字段一律下拉，禁止自由文本手填**（角色 ← 画布锚点；卡 ← 画布 CallCard 节点；包 ← 包配置源；片段 ← 资源浮窗）。

| effect | 中文 | 类别 | 参数（必=必填） | 面板控件 |
|--------|------|------|-----------------|----------|
| `set_character_unlocked` | 解锁角色 | A | `agentId`(必) · `unlocked`(默 true) | 角色下拉 + 布尔开关 |
| `attach_call_card` | 挂载通话卡 | 连线 | `agentId`(必) · `cardId`(必) · `packageId?` · `activation?`(entryMode) | modal 列表 + attach 边（§2.3） |
| `set_redial_slot` | 设置重拨槽 | A | `agentId`(必) · `cardId?` | 角色下拉 + 卡下拉（可空） |
| `unmount_call_card` | 卸载通话卡 | 连线 | `agentId?`(默当前会话角色) · `cardId?`(默当前会话卡) | modal 列表 + unmount 边（§2.3） |
| `keep_card_pending` | 保持卡待处理 | C | 无 | 说明文案「作用于当前卡，无需配置」 |
| `schedule_call_card` | 调度通话卡 | B | `agentId`(必) · `packageId`(必) · `cardId`(必) · `delayMinutes?`(默 5) · `topicHint?` | 角色/包/卡下拉 + 整数 + 文本 |
| `schedule_recurring_call` | 登记重复外呼 | B | `agentId?`(默当前) · 卡引用（`scheduleCardId` 或 `cardId`+`packageId`，二选一必） · `scheduleMode`(daily/weekly，默 daily) · `hour`(0–23，默 9) · `minute`(0–59，默 0) · `weekdays?`(weekly 时多选 0–6) · `topicHint?` | 下拉 + 整数 + 周几多选 |
| `create_research_commitment` | 创建研究承诺 | B | `question`(必) · `notifyMode?`(默 `next_call`) | 文本域 + Select |
| `update_user_profile` | 更新用户档案 | B | `nickname`(必) · `fullName?` | 两个 TextField |
| `patch_memory` | 补丁记忆 | A | `agentId?`(默当前) · `layer?`(默 `semantic`) · `kind?`(默 `semantic`) · `text`(必) | 角色下拉 + 两个 Select + 文本域 |
| `set_world_fact` | 写入世界事实 | B | `factId`(必) · `value?`(默 true) · `visibility?`(默 `global`) | TextField + 值输入 + Select |
| `update_npc_knowledge` | 更新 NPC 知识 | A | `agentId`(必) · `factId`(必) · `known?`(默 true) | 角色下拉 + factId 输入 + 布尔开关 |
| `end_story` | 结束故事 | B（复合） | `reason?` · `cleanup?`{`clearStoryCards?`(all/none，默 all) · `preserveFreeCards?`(默 true)} · `next?`{`packageId`(必) · `agentId`(必) · `cardId`(必) · `activation`(immediate/delay/wait_user_dial，默 wait_user_dial) · `delayMinutes?` · `entryMode?` · `activationHint?` · `acquireLockEarly?`} | 文本 + 清场折叠 + 「下一章入口」折叠子面板 |
| `create_voicemail` | 创建语音信箱 | B | `agentId?`(默当前) · `cardId?` · `topicHint?` | 角色/卡下拉 + 文本 |
| `play_system_prompt` | 播放系统提示 | B | `clipId`(必) | 资源片段下拉 |

**枚举取值范围（本轮 mock 采用，标注默认值；与引擎默认一致，未知值不校验拒绝）：**

- `activation` / `entryMode`：`inbound_user_dial` · `outbound_auto` · `either` · `agent_outbound` · `playback`（复用 refine 现有 entryMode 选项）。
- `end_story.next.activation`：`immediate` · `delay` · `wait_user_dial`。
- `notifyMode`：至少含 `next_call`；其余以引擎为准，本轮下拉可仅列 `next_call`。
- `patch_memory.layer` / `kind`：默认 `semantic`；下拉候选本轮可仅列 `semantic`（预留扩展 `episodic` 等）。
- `set_world_fact.visibility`：默认 `global`；下拉候选本轮可仅列 `global`。

### 2.3 attach_call_card / unmount_call_card：modal 列表 + 连线双向同步

**数据模型（会话 mock）：**

- Effect 仍归属**出口**（`exit.effects[]`）；一条 `attach_call_card` / `unmount_call_card` 行 ↔ 画布上一条**效果边**。
- 效果边 `data.kind`：`attach` / `unmount`（视觉区别于故事流转边，如虚线 + 「挂载」/「卸载」标签、不同色）。
- 效果边 `source` = 承载该 exit 的 CallCard 节点，`sourceHandle` = `exitId`；`target` = 被挂载/卸载的 CallCard 节点（= `cardId`）。
- `agentId`：默认取 target 卡的归属角色（其 role 边 / `ownerAgentId`）；modal 中可覆盖。target 卡无归属时 `agentId` 必填于 modal。

**modal 列表交互：**

1. effect 面板显示「挂载列表」：每行 = 目标卡（title）+ 角色（显示名）+ 可选包/激活标签 + 删除 ×。
2. 点「＋ 添加挂载」→ 打开 modal：选 **目标卡**（画布 CallCard 下拉）、选 **角色**（画布锚点下拉，默认 target 卡归属）、可选 packageId / activation。
3. 确认 → 追加一行 `attach_call_card` 到该 exit.effects[]，**并在画布生成一条 attach 效果边**。
4. 删除某行 → 移除该 effect **且移除对应效果边**。
5. 编辑某行 → 复用 modal 回填；改 target 卡即改边 target。

**画布 → 列表反向同步：**

- 从卡的 exit handle 拖出效果边到另一张卡（本轮可用「按住修饰键拖 = 效果边」或专用 handle，实现方式由技术设计定，但**必须与故事流转边可区分**）→ 在该 exit.effects[] 追加对应 `attach_call_card` 行，`agentId` 取 target 卡归属。
- 画布删边 → 移除对应 effect 行。

**unmount 同构：** 同一套边/列表机制，`data.kind=unmount`；`cardId`/`agentId` 缺省语义 = 当前会话卡/角色（modal 中允许「留空 = 当前卡」）。

**降级边界（若拖拽建边工程量过大）：** 至少保证「modal 列表 → 建/删效果边」这条正向同步与列表 CRUD 完整；反向（画布拖边 → 写 effect 行）可作为本子型的收尾项，但**不得**只做列表而完全不画边（用户明确要「与连线同步」）。

### 2.4 目标选择型（A 类）下拉数据源

| 下拉 | 数据源 | value / label |
|------|--------|---------------|
| 角色 `agentId` | 画布角色锚点（`listCharacterAnchors` 查询） | value=agentId · label=显示名 |
| 卡 `cardId` | 画布 CallCard 节点 | value=cardId · label=title |
| 包 `packageId` | 包配置 / refine5 章节结束所用包源 | value=packageId · label=包名 |
| 片段 `clipId` | 资源浮窗资源列表 | value=资源 id · label=资源名 |

- 下拉均提供「（未设）」空选项（仅当该字段可空时）。
- 选项为空时显示 helperText「画布上暂无可选角色/卡，请先创建」。

### 2.5 每个 effect 选项 tooltip 文案（挂在 Effect 下拉 MenuItem）

| effect | tooltip |
|--------|---------|
| `set_character_unlocked` | 把指定角色标记为已解锁/可拨，写入用户 Profile 的角色解锁位 |
| `attach_call_card` | 向指定角色的待办板挂一张待处理卡，使其后续可被拨打或外呼 |
| `set_redial_slot` | 设定用户「重拨」时默认接通的角色（及可选卡） |
| `unmount_call_card` | 从指定角色待办板移除一张待处理卡；缺省作用于当前通话卡 |
| `keep_card_pending` | 让当前卡保持待处理，不因本次通话结束而被消费；无需参数 |
| `schedule_call_card` | 登记一次性定时外呼意图（角色+包+卡+延迟），到点触发 |
| `schedule_recurring_call` | 登记每日/每周循环外呼意图（时/分/可选周几） |
| `create_research_commitment` | 记录一个待研究问题，在下次通话或指定时机回访 |
| `update_user_profile` | 写入用户昵称/全名到用户档案 |
| `patch_memory` | 向指定角色的记忆层写入一条记忆文本 |
| `set_world_fact` | 写入或更新一条世界事实（键+值+可见范围） |
| `update_npc_knowledge` | 让指定角色「知道/忘记」某条世界事实 |
| `end_story` | 结束当前故事包，可清场并安排下一章入口卡 |
| `create_voicemail` | 生成一条语音信箱桩，真正播放由电话壳执行 |
| `play_system_prompt` | 记录一条系统提示播放桩（片段 id），引擎不直接播放音频 |

实现方式：在 `EFFECT_NAME_OPTIONS` 每项补 `description`，Effect 下拉 MenuItem 外包 MUI `Tooltip`（或 `TextField select` 的 renderValue/helperText 组合），鼠标悬停展示。

### 2.6 投影类型扩展

在 `typeFiles/story/editor/callCard/editorCallCardProjection.ts`（或新增 `editorEffectParams.ts`）：

```text
EditorExitEffectProjection = {
  id: string
  effect: KnownEffectName        // 收紧为枚举联合，替代现 string
  summary?: string               // 人话预览，缺省由 params 自动派生，可覆盖
  params?: EditorEffectParams    // 按 effect 判别的参数投影
}

EditorEffectParams = 判别式联合（discriminated union by effect）
  每个变体字段严格对齐 §2.2 表；缺省等同未配置；禁止 catchall 任意透传
```

- 判别键用 `effect`；每变体只列该 effect 的合法字段（编译期防写错字段）。
- 复杂变体（`end_story`）内嵌 `cleanup?` / `next?` 子对象，字段级注释说明单位与默认。
- 保留 `coerceKnownEffectName`；`effect` 收紧后 `ExitEffectsMockList` 的 value 走枚举。

### 2.7 summary 自动派生

- 新增派生函数（bis 层）：`summarizeEffect(effect, params)` → 人话，如「挂载『专家回电卡』给 医生」「每日 09:00 外呼 客服」。
- 面板改 params 时若 summary 未被用户手动编辑，则自动刷新；用户手改后不再覆盖（标记 dirty）。
- summary 供节点 Tooltip / 出口列表预览；不进引擎，不写盘。

### 2.8 落点约束

| 内容 | 落点 |
|------|------|
| 契约类型 | `typeFiles/story/editor/callCard/`（扩展或 `editorEffectParams.ts`） |

| 参数字段编排 / summary 派生 | `src/bis/pageBis/storyEditor/form/exitList/effects/` |
| 效果边 ↔ effects 同步 | `src/bis/pageBis/storyEditor/canvas/`（新增 effect 边同步模块） |
| 每 effect 面板组件 | `src/pageComponents/storyEditor/com/panel/ExitListEditor/com/effects/`（单页私有；目录单元；禁 barrel） |
| attach/unmount modal | 同上 effects 目录内私有组件 |

- **禁止** 把 effect 参数硬塞进 `EditorCallCardProjection` 顶层或引擎 schema。
- **禁止** 在 `ExitEffectsMockList` 内堆全部 14 种面板；按 effect 分发到子组件。
- 组件遵守 studio-v2 门禁：tab 缩进、props 入口解构 + 注释、引用注释、无 barrel、目录单元。

## 6. 给批执行 Agent 的硬停语境

命中以下应 `BLOCKED`：

- 改 `packages/rpg-engine` schema / runtime，或 import 引擎运行时值当参数真源
- 写盘 `data/storis-packages`、`story.conf.json`、接 Host save
- 让用户自由文本手填 `agentId` / `cardId` / `packageId` 作为创建主路径
- 把 effect 参数塞进 `EditorCallCardProjection` 顶层或引擎 `CallCardDefinition`
- attach/unmount 只做列表、完全不画/不同步效果边
- 效果边与故事流转边无法区分
- 在 `ExitEffectsMockList` 单文件堆 14 种面板（不拆子组件）
- 用 `eslint-disable` / 关 strict / 门禁 allowlist 掩盖结构问题
- 恢复 refine4 已 supersede 的角色轨 / 底栏角色浮窗
