## 1. 本轮定位

### 1.1 要做什么

```text
阶段 A · 引擎契约（路径 B）
  A1 story.conf.participants 可选/派生；collectReferencedAgentIds
  A2 validatePackage：按「包内被引用 agentId」校验；PARTICIPANT_* 降级
  A3 磁盘包迁移 + 需求 01/08 口径更新

  B1 底栏/画布新建卡 → conf.cards[] + cards/<id>.s-card.json
  B2 画布删卡 → conf 移除 + 删磁盘 s-card（策略钉死）
  B3 entryCardId 可编；effect.critical roundtrip + 面板开关

阶段 C · 出口条件与节拍（P0）
  C1 ExitCondition 编辑器 v1（写回 exit.condition，非 conditionSummary 假字段）
  C2 requiredBeats ↔ beat_completed/beat_missing 联动
  C3 exitKind / priority 与引擎 enum roundtrip 验收

阶段 D · 角色模型与编辑器锚点
  D1 锚点 = 角色库全量；挂卡数由 cards 统计；未用灰显
  D2 去 participants 白名单；lanes 派生；effect 下拉全库

阶段 E · 资源真源（P1）
  E1 资源库落盘 data/assets + /api/assets CRUD；删 MOCK_ASSETS 业务真源
  E2 playbackClipId / play_system_prompt.clipId / assetRefs 接真源
  E3 头像直传（可与 E1 共用上传基础设施）

阶段 F · 包级元数据与 Schedule 约定（P1）
  F1 assetRefs / worldFacts / meta.imports|exports 最小可编（表单或受控 JSON 块二选一）
  F2 Schedule 卡落盘分工：故事包内 schedule vs characters/schedule-cards

阶段 G · UX 债 + 校验定位 + 收口
  G1 删遗留角色 + golden_handoff/demo-user 迁移；变体 id；时段 Select；包列表搜索+新建
  G2 保存前 validatePackage + error 路径定位（package/card/exit）；阻断写盘
  G3 docs/AI 同步 + 全批验收勾选
```

### 1.2 本轮不做

| 不做 | 说明 |
|------|------|
| Host / beginCall / Effect **真执行** | 编辑器仍只写 Content；不 pretend 调试闭环 |
| 通话中 RuntimeExitCandidate / 工具运行时 UI | 动态出口 `exitKind: dynamic` 仍禁止内容 JSON 手写 |
| 调试器 **真会话**（beginCall 驱动） | 可选尾任务仅读盘展示 validate，不 Host |
| 导入导出正式 ZIP 管道 | 列表导入保持现状 |
| 第二幕内容包 | 不扩叙事 |
| 单卡增量 API | 仍整包保存 |
| ExitCondition **完整** and/or/not 嵌套树 UI | v1 仅常用叶子 + always；复杂嵌套另批或 JSON 逃生 |
| Free 卡主编辑入口迁到故事编辑器 | Free 卡真源仍在 `characters/free-cards/` + 角色库 |

## 2. 已钉死口径

### 2.0 角色全局化（路径 B 铁律）

| 项 | 口径 |
|----|------|
| 角色真源 | **仅** `data/characters/<agentId>.json` + `/api/characters` |
| 故事包不私有角色 | **禁止** `participants` 当白名单；作者不手维护 |
| 「本包用到谁」 | `cards[].ownerAgentId` + effects 内 `agentId` + attach 目标归属 **派生** |
| 编辑器锚点 | **全库**显示；挂卡数来自本包 cards；未用可灰显 |
| 「添加角色」 | **仅**角色库新建；不是加入包 participants |
| participants | 新包可不写；保存推荐不写；校验现场派生 |
| layout.lanes | Studio 按库序或「有卡者优先」派生；引擎忽略 |

### 2.1 引擎 validatePackage 改造

1. **`collectReferencedAgentIds(bundle)`**：扫描 ownerAgentId、effects.agentId、end_story.next.agentId、attach 目标归属等（与 validate 一致）。
2. **`REFERENCED_AGENT_UNKNOWN`**（error）：派生 id 不在角色库。
3. **`validateParticipantsCharacters`** 对**派生集合**执行。
4. **`PARTICIPANT_UNKNOWN`**：非空遗留 participants 未知 → **warning**「遗留字段将忽略」；空数组不报错。
5. **ownerAgentId 不在旧 participants** 但在角色库 → 不得 error。

### 2.2 包内卡生命周期（P0）

| 操作 | 必须行为 |
|------|----------|
| 底栏放置新 CallCard | 生成 cardId；追加 `conf.cards[]`；新建 `cards/<cardId>.s-card.json`；画布节点与 conf 一致 |
| 画布删除 CallCard | 从 `conf.cards[]` 移除；**删除**对应 s-card 文件（若 entryCardId 指向该卡则阻止或强制改入口） |
| 保存整包 | `editorGraphToBundle` 写回 **conf.cards 与磁盘卡文件并集**（以画布为准），禁止只改 layout 不改 conf |
| orphan 磁盘卡 | 未在 conf 索引的 s-card → validate `CARD_FILE_ORPHAN` warning；保存时可清理或保留策略写清 |

**禁止：** 仅会话内新增节点，保存后 conf 仍只有打开时的卡列表。

### 2.3 entryCardId 与 effect.critical

| 项 | 口径 |
|----|------|
| entryCardId | 包配置或章节浮窗可 Select 本包 cardId；新建包默认指向首张 story 卡或占位 |
| critical | 读盘保留；投影不丢；Effect 面板 boolean「失败则中止后续」；写回 effect.critical |

### 2.4 ExitCondition 编辑器 v1（P0）

**问题：** 现 `conditionSummary` 仅 UI 人话；写盘用 `base.condition ?? DEFAULT_CONDITION`，改 summary **不改**引擎 condition。

**v1 必须：**

| op | 面板 |
|----|------|
| `always` | 单选「始终」 |
| `outcome_flag` | flag Select（对齐 OutcomeFlagSchema）+ equals 布尔 |
| `beat_completed` | beatId Select（来自本卡 objectives.requiredBeats + 手填逃生） |
| `beat_missing` | 同上 |
| `all_required_beats_completed` | 单选 |

- 写回 **`exit.condition`** JSON；`conditionSummary` 改为 **派生预览**（可覆盖显示，但不替代 condition）。
- **禁止** 保存时一律 `DEFAULT_CONDITION` 覆盖作者已选 condition。
- and/or/not 嵌套：**本轮不做**完整树；复杂条件允许 JSON 逃生块或保留磁盘原 nested 不覆盖。

### 2.5 Studio 编辑器锚点

| 行为 | 口径 |
|------|------|
| 打开包 | `fetchCharacterDefs()` 全量锚点 |
| statusLabel | 有卡 → `本包 · N 卡`；无 → `本章未挂卡` |
| Effect 下拉 | 全库锚点 + 本包卡列表 |

### 2.6 资源真源（P1）

| 项 | 口径 |
|----|------|
| 落盘 | `data/assets/`（或仓内约定目录）+ 元数据 JSON |
| API | `/api/assets` 列表/读/写/删；Studio 资源库页与 AssetPickerFloat 读 API |
| 删除 mock | 接通后删 `MOCK_ASSETS` / `listMockAssets` 作业务真源（零残余） |
| 引用 | `playbackClipId`、`play_system_prompt.clipId` 下拉来自 API；`conf.assetRefs` 可编辑（多选或列表） |
| validate | 保存 validate 时 `ASSET_UNKNOWN` / `ASSET_URI_MISSING` 可真实拦错 |

头像：`POST /api/characters/:id/avatar` 或复用 assets 管道；**去掉** assetId 手填主路径。

### 2.7 包级元数据与 Schedule（P1）

| 项 | 口径 |
|----|------|
| assetRefs | 包配置可编；与全局 assets 下拉联动 |
| worldFacts / meta | 最小表单或受控 JSON 块（实现二选一）；不要求可视化编辑器 |
| Schedule 卡 | **钉死分工**（写入需求 01/19）：故事包内 `cardKind=schedule` 仅剧情调度节点；角色日常 `schedule-cards/` 由角色库/专用路径维护；Studio 创建 schedule 卡时落盘路径与 `schedule_recurring_call` 校验一致 |

### 2.8 故事包列表与新建

| 项 | 口径 |
|----|------|
| 搜索 | filter title / packageId |
| 新建 | `POST /api/stories` 最小 conf（可无 participants）+ layout + 可选默认 entry 卡 |
| 删除 mock | `commitCreatePackageMock` throw 清零 |
| characterCount | 派生「本包引用角色数」 |

### 2.9 角色库 / 玩家 UX

- **C1** 删 doubao-sister/xiaoyu；迁移 golden_handoff、demo-user 后再删。
- **C2** 变体 id UUID 自动生成，UI 隐藏。
- **C3** LocalHourRangeField → 双 Select。

### 2.10 保存 validate + 定位

| 项 | 口径 |
|----|------|
| 时机 | 顶栏保存前 |
| 实现 | BFF 调 `validatePackage` |
| 失败 | 阻断 PUT；展示 errors[]（ruleId + path + message） |
| 定位 v1 | 至少：按 path 解析 packageId/cardId/exitId；点击 error → 选中对应画布节点或打开浮窗（最小可行） |
| warning | 可展示但不阻断（可配置；默认 warning 不阻断写盘） |

## 6. 给批执行 Agent 的硬停语境

命中以下应 `BLOCKED`：

- 锚点仍仅 conf.participants 子集 / 双轨白名单
- 新建卡不写 conf.cards 或不建 s-card.json
- 删卡不删磁盘或 conf 不同步
- conditionSummary 写盘替代 exit.condition 或保存一律 DEFAULT_CONDITION
- critical 保存丢失
- MOCK_ASSETS 仍作资源业务真源
- validate 失败仍 PUT
- 未迁移 golden_handoff 即删 doubao-sister 致 engine 红
- Host beginCall 真跑 / 调试器真会话当作本批必达
- `eslint-disable` / 门禁 allowlist 掩盖问题
- 未授权改 `docs/人类/`
