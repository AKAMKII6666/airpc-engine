## 2. 已钉死口径

### 2.1 章节结束点配置（chapter_end）

**触发：** 在画布选中 `chapter_end` 节点后，属性浮窗/章节属性区需要提供可配置字段。  
**范围：** 仅覆盖 `chapter_end`；`chapter_start` 维持当前轻量投影口径。

**字段：**
- `nextPackageId`：下一故事包选择（从现有 mock 包列表得到）
- `nextEntryCardId`：下一章节起点卡选择（依据 `nextPackageId` 可选卡集合）

**交互约束（第一版）：**
- `nextPackageId` 变更时，`nextEntryCardId` 若不在新包集合内则置空或回退到该包的默认起点卡。
- 列表控件使用 **Select（下拉）**；禁止把包/卡 ID 以自由文本方式交给用户。
- 属性浮窗/章节属性区仍保持“不写盘”，仅更新会话内节点 data 投影。

**渲染口径：**
- chapter 节点标题与摘要仍保留当前用途；
- 新增字段进入 UI 后，摘要可作为只读提示或保持轻量可编辑文本（由实现侧决定，但需与向导稿一致：清理 pending + 安排下一章的含义仍可被看见）。

> 向导稿参考：`02-新版故事编辑器界面UI设计向导稿.md` 中 “章节结束点的第一版配置方式”（下一包 + 起点卡）。

### 2.2 出口 Effect 列表：枚举强约束 + 中文名

**现状问题：** `keep_card_pending` 这类 Effect 本质是枚举；用户不应在 UI 中自由输入任意字符串。

**口径：**
- exits.effects 的每一行 `effect` 字段改为 **Select 下拉**；
- 下拉选项来自引擎已知枚举集合（与 `KNOWN_EFFECT_NAMES` 对齐）；
- 展示中文名 + value 写入枚举字符串（例如 `keep_card_pending`）。

**summary：**
- summary 可保留为 TextField（人话辅助/备注），但不作为枚举判别真源；
- 若实现允许，summary 可在 effect 改变时自动填入默认说明（可选，非必须）。

### 2.3 工具策略：允许的工具 ID 改为多选内置工具列表

**触发：** 当 `toolPolicy.mode === "allowlist"` 时，展示 `allowedToolIds` 的多选控件。  
**否则：** 非 allowlist 模式下 `allowedToolIds` 需隐藏或禁用（并给出 helperText 提示“留空即可”）。

**口径：**
- 控件展示内置工具列表（与 `BUILTIN_TOOL_DEFINITIONS` 的 `toolId`/`displayName` 对齐）。
- value 写入：`allowedToolIds` 为 `toolId[]`（字符串数组）
- UI 展示：中文名（`displayName`）并允许多选（Checkbox/多选下拉均可，但必须是“选项列表”，禁止自由文本输入 toolId）。

### 2.4 枚举审计（故事编辑器 / 角色管理 / 玩家管理）

对 UI 中所有“看起来应为枚举”的字段做一次审计：
- 若字段对应引擎/类型中的枚举：必须使用 Select/Radio/Checkbox（或等价枚举控件）
- 若字段目前仍为 TextField/StringListEditor 且写入的是枚举值：需要整改为枚举控件

审计重点包括（优先级从高到低）：
1) 故事编辑器：`exits.effects[].effect`、`toolPolicy.allowedToolIds`  
2) 角色管理：角色性别/音色等（应已为 Select）  
3) 玩家管理：玩家性别等（应已为 Select）
