## 3. 已钉死口径

### 3.1 页面定位与文案

| 位置 | 新文案 |
|------|--------|
| 侧栏导航（`studioNavItems`） | **玩家配置**（原「调试用户」） |
| `/users` 页头标题 | **玩家配置** |
| 新建按钮 | **新建玩家**（或等价；勿再写「新建用户档案」调试腔） |
| 角色库等快捷链到 `/users` | 同步改为「玩家配置」 |

- **删除**「打开调试器」按钮及本页导向 `/debugger` 的主流程入口。
- 说明文案不得再强调「供调试器模拟来电」；定位为编辑引擎 `User` / 薄 Profile 中的玩家身份信息。

### 3.2 表单字段（编辑态展示项全必填）

**基本信息**

| UI | 落盘 | 控件 |
|----|------|------|
| 昵称 | `user.nickname` | 文本框 |
| 全名 | `user.fullName` | 文本框 |
| 性别 | `user.gender` | 下拉：男 / 女 → `male` / `female` |
| 生日 | `user.birthday` | 日期选择（年/月/日） |
| 年龄 | `user.age` | 整数输入 |

- 年龄与生日：**各填各的，不做交叉校验**。

**地理位置**（仅为分组标题，**不是**自由文本字段）

| UI | 落盘 | 控件 |
|----|------|------|
| 国家 | `user.location.country` | 文本框 |
| 省/州 | `user.location.province` | 文本框 |
| 城市 | `user.location.city` | 文本框 |
| 区/县 | `user.location.district` | 文本框 |

**只读**

| UI | 落盘 |
|----|------|
| 创建时间 | `user.createdAt`（label） |
| 更新时间 | `user.updatedAt`（label） |

`userId`：高级区只读展示即可；主流程不手填。

### 3.3 引擎 `User` 增量

当前引擎已有：`userId` / `nickname` / `fullName?` / `location?` / `preferences?` / `createdAt` / `updatedAt`。

本轮必须补齐：

```ts
interface User {
  userId: string
  nickname: string
  fullName?: string
  gender?: "male" | "female"
  birthday?: string   // YYYY-MM-DD
  age?: number        // 整数
  location?: {
    country: string
    province: string
    city: string
    district?: string
  }
  preferences?: Record<string, unknown>
  createdAt: string
  updatedAt: string
}
```

- Zod：`UserSchema` 同步；Studio 编辑态全必填，磁盘 schema 仍可 optional。
- 需求 [10](../../需求/10-用户与存档.md) §2 同步。
- 样例 `data/users/**/profile.save.json` 可补齐新字段（至少 demo-user 可读）。

### 3.4 本页明确不展示

以下引擎 / 旧 Studio 投影 **不进** 玩家配置表单：

- `PlayerProfile` 的 `characters` / `stories` / `callCards` / `telephony` / `world` / `schedule` / `research`
- 旧 mock：常用度、备注、默认呼入角色、默认故事包、默认重置、最近调试记录等

`preferences` 可继续存在于引擎，本轮 UI **不编辑**（除非后续另立调试偏好里程碑）。

### 3.5 新建 = 同一套表单

- 新建弹窗 / 侧栏与详情编辑使用 **同一字段集**（同一 `items[]` 或共享配置模块）。
- `userId` 系统生成；`createdAt` / `updatedAt` 由系统写入。
- 禁止继续使用第二步那种「仅昵称级」简陋创建表。

### 3.6 持久化

- 目标：经 Next API 门面读写 `data/users/<userId>/profile.save.json` 中的 `user`（及必要的 Profile 外壳）。
- 若细化修改 1 尚未接通 users 写盘：本轮须接通 **User 段** 读写；不得假装已保存却只改 session mock。
- 重置整份存档、导入导出 SaveGame：本轮可不做（需求 10 §5 能力可后续）。
