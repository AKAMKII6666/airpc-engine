# Studio V2.0 · 语音留言改造需求

> **状态：** 已定稿（会话拍板 2026-07-23），供实现与 Review / gbx 引用。  
> **性质：** 将「语音留言」从独立 Effect 桩（`create_voicemail`）收束为专用 **CallCard 种类**（`cardKind=voicemail`），复用挂卡 / 延迟调度 / 出口 Effect，并明确引擎外 LLM 物化与壳侧未读同步。  
> **执行索引：** [语音留言改造_执行索引.md](./语音留言改造_执行索引.md)

## 1. 为什么要改

### 1.1 现状问题

| 现象 | 后果 |
|------|------|
| **没有** `cardKind=voicemail`；样例「语音留言」卡实为普通 `story` + `realtime_dialogue` | 作者与调试者误以为「挂这张卡 = 听留言」 |
| 留言能力落在 Effect `create_voicemail`：只写 `telephony.voicemails[]` stub，真播归壳 | 编辑器只有参数面板；**卡种下拉无「留言卡」**；引擎无专用会话语义 |
| 样例出口仅 `create_voicemail`（带 `cardId`），**不** `attach_call_card` | 画布边指向留言卡，但运行时**不会**把该卡挂进 Board；拓扑与执行脱节 |
| 听完留言后要设重播、再挂剧情卡、触发其它事件 | 无法自然复用该卡的 **exits / Effect plan**；调试器若按 CallCard 类型分支会走错 |

### 1.2 要解决这个痛点，正确形态应是什么样

痛点收口后的正确形态 = 作者在 Studio 编留言卡 → 挂机 Effect 入信箱 → 壳/调试可见未读 → 用户听留言 → 听完走该卡出口 → 可再挂卡/重播。端到端语义清晰、可测。

```text
作者：画布上放 cardKind=voicemail 卡，填 context（留言合同）
     出口用 attach_call_card / schedule_call_card 指向该卡
       （Effect 下拉不再提供 create_voicemail）

挂机：ExitSelector → Effect plan
  · attach 目标为 voicemail → 不进 Board.pending，入「留言生成栈」
  · schedule 目标为 voicemail → 延迟到点后再入栈（≠ 外呼响铃）
  · 其它 attach/schedule 照旧

plan 终态之后：VoicemailMaterializePipeline（显式 Host 阶段）
  · 引擎拼装生成输入（含 context / 角色 / 卡 id）
  · 回调引擎外 LLM / TTS 端口 → 得到文本（调试）或音频引用/buffer（真机）
  · 写入 Profile.telephony.voicemails[]；更新未读真源
  · 通知壳 onVoicemailUnreadChanged → LED / 角标

听留言：打开信箱 → beginCall（entryMode=mailbox_open，playback_only）
  · 播槽内已物化内容
  · 播完 / 跳过 → Outcome（voicemail_listened / playback_completed…）
  · ExitSelector → 该卡 exits（set_redial_slot、attach 下一剧情卡等）
```

### 1.3 本轮目标（一句话）

**留言是一种 CallCard；进信箱靠挂卡/延迟分流；正文由引擎外生成；听完复用同一套出口机制。**

## 2. 本轮定位

### 2.1 要做什么

```text
A 契约：CardKind 增加 voicemail；entryMode 增加 mailbox_open；废弃 create_voicemail
B 运行时：attach / schedule 对 voicemail 分流（永不进 Board.pending）
C 物化：显式 VoicemailMaterializePipeline + 外置生成端口 + 未读通知端口
D Studio：卡种/锁定字段 UI；Effect 下拉去掉 create_voicemail；挂卡可指向留言卡
E 样例：wrong_number_act1 迁到 attach + cardKind=voicemail
F 调试：信箱队列、未读、模拟听完跑出口（最小可用）
G 文档：回写需求 01/02/04、技术设计 07/19；校验规则补充
```

### 2.2 本轮不做

| 不做 | 说明 |
|------|------|
| 引擎内置 LLM / 持 Key | 生成一律外置端口；Studio/壳注入 |
| 真机 WAV 编解码进 `rpg-engine` | 引擎只存文本 / asset 引用 / stub；buffer 可经端口回传由壳落盘 |
| 完整电话壳 LED 硬件驱动 | 只定端口契约；壳实现可 mock |
| 隐式扫 transcript 拼留言 | 禁止（对齐 Free 铁律） |
| 把留言当普通拨号 Board 待接通 | 禁止 |
| 第二幕叙事扩写 | 仅改第一幕样例拓扑所需最小集 |

### 2.3 与前置的关系

| 前置 | 关系 |
|------|------|
| 细化修改 7 | Effect 面板已有 `create_voicemail` → 本批删除该选项与面板 |
| 细化修改 8 | Content 契约 / validate 已通 → 本批扩展 CardKind 与校验 |
| 技术设计 19 §6 | Executor vs Sink → 物化编排属 Host；媒介通知属 Sink/壳端口 |

## 3. 已钉死口径

### 3.1 cardKind = `voicemail`

| 项 | 口径 |
|----|------|
| Schema | `CardKind` ∪ `"voicemail"` |
| `interactionMode` | **仅** `playback_only`；Studio **disable** 其它选项；校验 error 若不符 |
| `entryMode` | **仅** `mailbox_open`（界面文案「信箱打开」）；Studio disable；校验 error 若不符 |
| `toolPolicy` | 必须等价 `deny_all`（与 playback_only 一致） |
| Board | **永不**进入 `callCards.board.byAgent.*.pending` |
| 可拨 | 不可经普通 `user_dial` / `agent_outbound` 解析为待接通剧情卡 |

### 3.2 context（留言合同）

`context` 仍是 CallCard 上的剧情/生成合同字段，对 voicemail 的消费者变为「物化端口 + 播放」，而非实时对话 Composer：

| 字段 | 留言场景用途 |
|------|----------------|
| `speakableBrief` | 留言稿骨架 / 给外置 LLM 的可说内容 |
| `privateBrief` | 生成约束；不得原样播给用户 |
| `objective` / `emotion` / `premise` / `background` / `forbidden` | 生成目标与禁区 |
| `promptScenes` | 可选；留言场景默认少用 |
| `playbackClipId` | 若已有成品片，物化可跳过 LLM，直接引用 |

### 3.3 挂载与延迟（Effect 分流）

| Effect | 目标为普通卡 | 目标为 `cardKind=voicemail` |
|--------|--------------|------------------------------|
| `attach_call_card` | 写 Board.pending（现状） | **不写 Board**；push **VoicemailGenStack**（立即） |
| `schedule_call_card` | Board.pending + once → 到点 `agent_outbound`（现状） | **不写 Board**；写 once intent，到点 **入栈/物化进信箱**（**不是**外呼响铃） |
| `create_voicemail` | — | **废弃**：Schema 拒识或 validate error；Studio 下拉删除 |

延迟语义（钉死）：「N 分钟后信箱多一条留言」，不是「N 分钟后电话响铃播留言卡」。到点时若用户正在通话：**仍写入信箱槽**（壳角标即可），不打断当前 CallSession。

### 3.4 物化阶段（VoicemailMaterializePipeline）

**时机：** Effect plan **终态之后**（含 `completed_with_errors` / `aborted` 的约定：已入栈条目仍应尝试物化，除非产品另定「aborted 丢弃栈」——默认 **不丢弃已入栈**）。

**职责：**

1. 出栈；加载卡定义；确认 `cardKind=voicemail`  
2. 拼装生成输入（结构化；可含已拼好的 prompt 字符串 + context 零件）  
3. 调用注入端口 `generateVoicemail(input)`（引擎外 LLM/TTS）  
4. 写 `Profile.telephony.voicemails[]`（正文 text 与/或 audioRef）  
5. 更新未读真源；调用 `onVoicemailUnreadChanged(hasUnread)`  

**禁止：**

- 在 `effectExecutor` 的 attach case 内直接调 LLM  
- 挂机后隐式扫描 transcript 拼留言  

**端口（建议形状，实现时可微调命名）：**

```ts
// 生成：可能慢、可能失败；Studio Manual 可注入假实现
generateVoicemail?(input: {
  card: CallCardDefinition
  agentId: string
  packageId: string
  instanceId: string
  assembledPrompt?: string
}): Promise<{ text?: string; audioRef?: string }>

// 未读推送：壳同步 LED；真源仍在 Profile
onVoicemailUnreadChanged?(hasUnread: boolean): void
```

失败：该条可标 `generate_failed` / 保持 pending；**不回滚**其它已成功 Effect（对齐 plan「失败继续」精神）。

### 3.5 听留言 = 特殊 beginCall

| 项 | 口径 |
|----|------|
| 入口 | 用户/调试打开信箱条目 → `beginCall`，`entryMode=mailbox_open` |
| Session | `frozenCard` = 该 voicemail 卡；`interactionMode=playback_only` |
| 播放 | 播槽内已物化 text（TTS）或 audioRef；引擎不解码 WAV |
| 结束 | Outcome 含 `voicemail_listened` / `playback_completed` / `playback_skipped` 等 |
| 出口 | 正常 ExitSelector → 该卡 `exits`（重播槽、挂下一剧情卡等） |

**不推荐：** 壳自播完只打 flag、Host 另写无 Session 的 exits 捷径。

### 3.6 未读真源

| 项 | 口径 |
|----|------|
| 真源 | `PlayerProfile.telephony`（由 `voicemails[]` 推导 `hasUnread`，或显式布尔字段；二选一须在实现时写进 schema 并保持单一推导） |
| 壳 | 仅订阅 `onVoicemailUnreadChanged` / 读 Profile；**禁止**壳私自改 Board/Memory/未读真源 |
| 重启 | 以 Profile 为准重新推导并再通知一次 |

### 3.7 Studio UI

| 项 | 口径 |
|----|------|
| 卡种下拉 | 增加「语音留言」→ `voicemail` |
| 选中 voicemail | `interactionMode` / `entryMode` 锁定并 disable |
| Effect 下拉 | **删除** `create_voicemail` 及专属面板 |
| 挂卡 / 延迟 | `attach_call_card` / `schedule_call_card` 可选本包 voicemail 卡；摘要/badge 标明「进信箱」而非「挂待接通」 |
| 底栏（可选） | 「新建留言卡」快捷 placement；非本批硬依赖可用属性窗改 kind |

## 4. 技术构想与方向

### 4.1 模块落点（建议）

| 能力 | 建议落点 |
|------|----------|
| Schema | `packages/rpg-engine/src/schema/callCard.ts`（CardKind / EntryMode） |
| Effect 分流 | `effectExecutor.ts`：attach/schedule 查目标卡 kind（需 `lookupCard`） |
| 延迟到点 | `scheduleTick.ts`：once 目标为 voicemail → 入栈/物化，不 `agent_outbound` |
| 物化管线 | 新模块如 `runtime/voicemailMaterialize.ts`；Host 在 plan 终态后调用 |
| 端口 | Host `createEngine({ generateVoicemail?, onVoicemailUnreadChanged? })`；与 EffectSink 并列或 Sink 扩展，**生成与 LED 分端口** |
| 废弃 effect | `outcome.ts` / Sink 媒介表去掉 `create_voicemail`；旧包 validate 报错并指引改 attach |
| Studio | `callCardLabels`、节点属性表单项、Effect 面板注册表、样例包 |
| 调试 | 信箱列表 + 模拟听完（最小）；完整调试台可后续批 |

### 4.2 时序（定稿草图）

```text
hangup / Free pipeline
  → ExitSelector
  → executeEffectPlan
       attach(voicemail) → GenStack.push
       schedule(voicemail) → intent（fire → GenStack）
       其它逻辑 effect → Profile
  → plan 终态
  → VoicemailMaterializePipeline（出栈 × N）
       generateVoicemail → write slot → unread notify
  → save Profile
  → story.card.completed（若适用）
```

### 4.3 与 EffectSink 的关系

- **逻辑 / 槽 / 未读真源：** Host / Materialize 写 Profile  
- **媒介通知（LED、真机提示音）：** 壳端口 / Sink  
- **历史 `create_voicemail` Sink 分支：** 删除或改为「仅兼容读旧 stub，不新写」——本批推荐 **直接废弃**，样例迁移后无旧路径

### 4.4 风险与原则

| 风险 | 原则 |
|------|------|
| 把生成塞进 Executor | 禁止；保持 effect 同步可测 |
| schedule(voicemail) 仍外呼 | 禁止；语义已改 |
| 调试按 story 卡 beginCall 留言卡 | 错；必须 mailbox_open / 信箱入口 |
| 扩大 diff 顺手重构 Host | 禁止；按执行索引刀序 |

## 5. 验收标准（产品向）

- [x] Schema / 校验：`voicemail` + `mailbox_open` + `playback_only` 强制；非法组合 error  
- [x] `attach_call_card` → voicemail：**不**出现在 Board.pending；信箱槽在物化后有条目  
- [x] `schedule_call_card` → voicemail：快进时钟后进信箱，**无** `agent_outbound` 响铃语义  
- [x] Studio：无 `create_voicemail` 选项；voicemail 卡字段锁定  
- [x] 样例 `wrong_number_act1`：留言路径为 attach + `lanxing_voicemail`（`cardKind=voicemail`）  
- [x] 生成端口可注入；未注入时有明确失败/跳过策略且不炸 Host  
- [x] 未读变化可回调；Profile 为真源  
- [x] 听完可跑 exits（至少单测或调试最小路径证明 `set_redial_slot` / attach 可用）  
- [x] `quality:engine`（及触及 Studio 时 `quality:studio`）绿
