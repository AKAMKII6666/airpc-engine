## 1. 本轮定位

### 1.1 要做什么

```text
0 引擎 User 增加 outboundWindow；scheduleTick 窗外 defer（保持 pending）
1 玩家配置页增加「NPC 可外呼时段」并经现有 user API 落盘
```

### 1.2 本轮不做

| 不做 | 说明 |
|------|------|
| 把 intents 写入 CharacterDef / 角色 JSON | 真源仍是玩家 `Profile.schedule.intents` |
| 玩家配置页展示完整 schedule 台 | 定时列表入口在**角色页** |
| 窗外 skip/consumed | 只 defer，下次窗内仍可火 |
| Realtime / 电话壳真链 | 仍只挂 pending / 改意图 |
| 调试器改版 | 不碰 DebuggerShell |

### 1.3 与细化修改 2 的关系

- 复用 AutoForm、`LocalHourRangeField`、用户 API（`user` 段）。
- 新建 schedule API：**只读写** `profile.schedule`，不替换 `user` 段。

## 2. 已钉死口径

### 2.1 `User.outboundWindow`

```ts
outboundWindow?: { from: number; to: number }
// from 0–23，to 0–24；半开区间：localHour >= from && localHour < to
```

- 磁盘 optional；Studio 编辑态必填（默认建议 `9–22`，即 09:00 ≤ h < 22:00）。
- 与角色 `promptScenes.localHourRange` **无关**（后者只切提示词）。
- 时刻源：`tickScheduleOnce` 的 `nowIso` 解析出的**本地小时**（v1：Host/Studio 进程本地；壳侧日后可传玩家时区 ISO）。

### 2.2 门闩行为

- 插入点：`tickScheduleOnce` 在 `ensureOutboundPending` **之前**。
- 窗外：**不**挂 outbound pending，intent **保持 `pending`**（defer）。
- 无 `outboundWindow`：行为与改前一致（不闸）。

### 2.3 定时外呼列表（角色页）

| 项 | 口径 |
|----|------|
| 挂载 | 角色详情表单内，**「场景提示词」字段下方**（仍不进 CharacterDef 保存） |
| 玩家选择 | 复用磁盘用户列表（同记忆区） |
| 过滤 | `intent.agentId === 当前角色 agentId` |
| 真源 | `data/users/<userId>/profile.save.json` → `schedule.intents` |
| **列表** | 标准行：类型/时间摘要、状态 chip、话题、编辑/删除（每日可暂停）；无数据时显示空态块 |
| **新增/编辑** | `FormModal`；类型为 **单选**（一次性 / 每日）；条件字段切换；真源说明放弹层 description |
| **删除** | `DeleteConfirmModal` |
| 校验 | 写盘前 `ScheduledIntentSchema` |
