# Studio V2.0 · 规范改进需求（客户端分层收口）

> **状态：** 已定稿（会话拍板 2026-07-23），供实现与 Review / gbx 引用。  
> **性质：** 将 Studio V2 **主要业务页**从「page/hook 自管业务态」收口为 **一域一 store + shell bis 灌账 + feature bis 编排**；故事编辑器采用 **画布高频自管 + flush→store** 双层。规则与硬门禁已先落地，本批是**存量模块对齐**。  
> **执行索引：** [规范改进_执行索引.md](./规范改进_执行索引.md)  
> **真源：** [技术设计 21](../../技术设计文档/21-Studio客户端分层.md) · [studio-v2-client-layering.mdc](../../../.cursor/rules/studio-v2-client-layering.mdc) · [目录 09](../../sudioV2.0版本/需求/ui/09-Studio-V2前端目录结构与落点规则.md) · [门禁 08](../../sudioV2.0版本/需求/ui/08-Studio-V2前端工程规则与质量门禁需求.md)（STRUCT-021～024）

---

## 1. 为什么要改

### 1.1 痛点

| 现象 | 后果 |
|------|------|
| 多数页用 `useXxxPage` / 局部 hook 持列表、选中、loading | 与技术设计 21 / SlimVID 三角不一致；难测、难共享 |
| UI 直引 `ajaxProxy`、偶发直读 store | 分层门禁只能靠 baseline 冻住；新改一碰就红 |
| 故事编辑器状态堆在 `useStoryEditorShellController` + 画布 ref | 「shell」名不副实；保存真源不清 |
| 规则已写、模块未迁 | 规范形同虚设，后续功能（如语音留言 UI）继续按旧编排写 |

### 1.2 正确形态（规范收口后）

```text
读：domain store → feature bis → UI
写：UI → feature bis → domain store
请求：shell / feature bis → ajaxProxy → API

一域一 store（characters / storyEditor / users / assets / packages / debugger …）
全局 studioV2Store 仅极薄壳偏好，不承载业务会话

故事编辑器：
  画布自管 nodes/edges/viewport（高频）
  → 节流 + 松手/提交/切选中 flush → storyEditor store
  → 保存：先 flush，再以 store 投影组 bundle
```

### 1.3 本轮目标（一句话）

**主模块按域迁到 store+shell+bis；清掉 layering baseline；编辑器双层可保存可门禁。**

## 2. 本轮定位

### 2.1 要做什么

```text
A 基础设施：domain store 约定、Provider 挂载、shellBis 目录落地
B 故事编辑器：双层状态 + shell/feature 拆分 + 保存 flush
C 角色库：列表/选中/CRUD 经 characters store + shell
D 用户库 / 资源库 / 包列表（同构库页）
E 调试器：会话快照等进 debugger store + shell
F 工作台 / 设置：扫薄 mock 直引，能迁则迁
G baseline 清零 + quality:studio 绿 + 文档勾选
```

### 2.2 本轮不做

| 不做 | 说明 |
|------|------|
| 改引擎运行时 / CallCard schema | 另见语音留言改造等 |
| 发明第二套状态框架（Redux/XState 整站） | 仅 Zustand + bis |
| 一天清零所有 page `useState` | 纯 UI 瞬时态仍允许 |
| 把 Memory/Profile 全文塞进 store | 铁律不变 |
| 旧 Studio（`apps/studio`）强制同迁 | 仅 V2 |

### 2.3 与前置关系

| 前置 | 关系 |
|------|------|
| 技术设计 21 + STRUCT-021～024 + layering baseline | **已落地**；本批消费规则、消 baseline |
| 细化修改 8 / 语音留言改造 | 可并行，但**编辑器分层与留言功能改同一文件时须同一切口**，禁止两线互踩 |
| SlimVID `app/` 分层 | 行为参照；目录映射到 `apps/studioV2` |

## 3. 已钉死口径

### 3.1 一域一 store

| 域 | store 落点（建议） | shell（建议） |
|----|-------------------|---------------|
| 壳偏好 | `stores/shell/studioV2Store`（或保留现文件，禁止扩业务切片） | layout shell（可选） |
| 故事编辑器 | `stores/storyEditor/` | `shellBis/storyEditor/storyEditor.shell.bis.ts` |
| 角色库 | `stores/characters/` | `shellBis/characters/characters.shell.bis.ts` |
| 用户库 | `stores/users/` | `shellBis/users/users.shell.bis.ts` |
| 资源库 | `stores/assets/` | `shellBis/assets/assets.shell.bis.ts` |
| 包列表 | `stores/packages/` | `shellBis/packages/packages.shell.bis.ts` |
| 调试器 | `stores/debugger/` | `shellBis/debugger/debugger.shell.bis.ts` |

- **禁止**多域业务态塞进同一个上帝 store。  
- store 只结果型 action；禁网络、禁 `next/navigation`、禁 import bis。

### 3.2 分层与门禁

| 层 | 必须 | 禁止 |
|----|------|------|
| UI | 调 bis；纯 UI `useState` | 直读 store、直引 ajaxProxy |
| feature bis | 读/写本域 store、ajaxProxy、toast | `next/navigation`；value import UI |
| shell bis | 灌本域 store；听 `*RefreshStamp` | 一类页挂多次；代替每个按钮 |
| *Flow.ts | 导航等路由副作用 | 命名成 `.bis.ts` |

硬门禁：`STUDIO-STRUCT-021`～`024`。本批每完成一域，须从 `studio-v2-layering-baseline.json` **移除已合规文件**（或整表重扫后只剩未迁域）。

### 3.3 故事编辑器双层

| 项 | 口径 |
|----|------|
| 画布 | React Flow 自管高频 nodes/edges/transform |
| 同步 | throttle + 松手 / 属性提交 / 切选中 **flush**；禁仅靠裸 interval |
| store | conf、卡投影、exits、浮窗业务态、save/validate、dirty |
| 保存 | **先 flush** → 以 store 投影 `editorGraphToBundle`；禁止有时读 canvasApi 有时读 store |
| 属性面板 | bis 写 store → patch 画布（或订 store 结构切片） |

### 3.4 库页（角色 / 用户 / 资源 / 包）同构模板

```text
shell：打开页 → GET 列表 → applyLoadResult → store
feature：create / save / delete → ajax → bumpRefreshStamp 或 patch 列表
UI：列表+详情+Modal 只绑 bis；选中 id 在 store
```

删除确认、FormModal 开合可为组件瞬时态。

### 3.5 调试器

- Session 快照、Board 摘要、mailbox 未读等进 `debugger` store。  
- mock 数据经 bis/ajaxProxy，**禁止** `DebuggerShell` 直引 mock 模块当长期真源。  
- 不在本批强制 Host 全真会话（与现调试范围一致）；结构先对齐。

## 4. 刀序与依赖

```text
V2-LY-0   约定 Provider / 空 shellBis 骨架（可与 B 合并）
V2-LY-1～4  故事编辑器（双层 + 拆 shell/feature + 保存 flush + 出 baseline）
V2-LY-5～6  角色库
V2-LY-7～9  用户 / 资源 / 包列表
V2-LY-10   调试器
V2-LY-11   工作台 / 设置扫尾
V2-LY-12   baseline 清零或仅剩声明豁免 + 验收
```

**禁止跳步：** 未完成编辑器 flush 契约就宣称「编辑器已分层」。  
库页可在编辑器中段并行，但 gbx `group: order` 默认串行；若拆并行须人工保证不共改同一 hook 树。

---

## 5. 验收标准

- [x] 各主域存在独立 `stores/<domain>/`，无跨域业务切片混存  
- [x] 各主域有且仅有一处页级 `*.shell.bis.ts` 灌数  
- [x] 目标页 UI 无 `ajaxProxy` / `stores` 直引（STRUCT-021）  
- [x] 故事编辑器：flush 后保存与打开 roundtrip；dirty 含未同步画布变更  
- [x] `studio-v2-layering-baseline.json` 本批目标文件已移除（理想：`files` 为空）  
- [x] `npm run quality:studio` 绿  
- [x] 技术设计 21 §10/§11 勾选推进；无 `@deprecated` 迁移壳残留

## 6. 风险

| 风险 | 缓解 |
|------|------|
| 与语音留言改造抢改编辑器 | 同索引合并任务或错峰；禁止两线各改属性面板 |
| flush 丢最后一笔 | 保存路径强制 sync flush；单测覆盖 |
| baseline 误删未合规文件 | 先跑 structure 再写 baseline；CI 红则回退 |
| 过度拆文件 | 异责聚类；helpers 保留；禁止为拆而拆 |

---

## 7. 非目标回顾

不把 React 引进引擎；不以 client store 为 Profile 真源；不扩大 diff 到无关美化。
