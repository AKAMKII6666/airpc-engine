# 07. 壳嵌入与导出契约

> **不做旧 doubaoSister 迁移**（对标 ≠ 迁移）。  
> 旧仓只作能力参考；电话壳将**另建新底座**并直接嵌入本引擎（硬件 / Realtime / EffectSink），不经旧仓。

相关：[00](../需求/00-文档索引.md) · [10](../需求/10-用户与存档.md) · [01](../需求/01-CallCard数据模型.md) · [13](../需求/13-工具与能力注册.md) · [19](./19-引擎宿主与会话模型.md) · [18](./18-部署拓扑与BS架构.md)

## 1. 职责边界

| 层 | 职责 |
|----|------|
| **rpg-engine / Host** | 五柱、CallSession、Profile/Content、Exit/FreePipeline、Executor |
| **Studio Next 门面** | 无密码 userId、HTTP/错误码、调 Host、`data/` 落盘 |
| **Studio 浏览器** | 故事列表/画布/调试 UI、UserGate、MUI；不持 Key |
| **电话壳** | 硬件、Realtime、call.*、**EffectSink**；`beginCall` 上报本地时间；直接链引擎 |

## 2. 两种导出物

### 2.1 内容包（Content）

工作区可导出（编辑真源形态）：

- `characters/`（角色唯一真源；无包内 agents[]）
- `storis-packages/<id>/story.conf.json` + `cards/*.s-card.json`（**可不含** `canvas.layout.json`）
- `tools/`、`assets/`（全局 meta+files）、Lore fallback
- 包可选 `assetRefs` 声明用到的 assetId，导出时可子集打包

不含用户进度。导出给壳时也可打成 zip；**仍建议保持分文件**，避免单文件巨包。

### 2.2 SaveGame

`PlayerProfile` JSON（含 telephony.redialSlot、schedule 等）。CallSession **默认不进** SaveGame；调试可另导快照。日志见 `data/logs/`。

## 4. Effect 与壳

| Effect / 端口 | 壳侧 |
|---------------|------|
| schedule / 外呼类（目标非 voicemail） | 真响铃 / CallIntent（经 EffectSink） |
| `attach_call_card` / `schedule_call_card` → **voicemail** | **不**经 Sink 写信箱；入 GenStack → plan 终态后 `VoicemailMaterializePipeline` |
| `generateVoicemail`（Host 注入端口） | 引擎外 LLM/TTS；返回 text / audioRef；未注入则该条失败/跳过策略，不炸 Host |
| `onVoicemailUnreadChanged`（Host 注入端口） | 壳 LED / 角标；**真源**仍在 `Profile.telephony.voicemails[]`（`deriveVoicemailHasUnread`） |
| ~~`create_voicemail`~~ | **已废弃**：校验 `VOICEMAIL_CREATE_EFFECT`；Executor / Sink **无**新写路径 |
| `set_redial_slot` | 重播键（引擎已写 Profile） |
| `set_character_unlocked` | **仅引擎写 Profile**；壳无媒介 |
| `play_system_prompt` | 系统音（EffectSink） |
| attach / fact / knowledge / memory / variable（普通卡） | **仅引擎写 Profile**；壳在通话前后调 Host |

时序：Executor 逻辑写盘意图 → Sink 媒介（若有）→ plan 终态 → **VoicemailMaterialize**（若栈非空）→ save。详见 [19 §6](./19-引擎宿主与会话模型.md)。

包配置与卡文件 **禁止**出现壳内部 API 名、绝对文件路径、handler 函数名。

## 5. 通话中过程事件（DialogueEvent 扩展）

挂机后的 `hangup_early` 出口 ≠ 通话中过程话术。过程事件**不改 Profile**：

| 事件 | 选用 |
|------|------|
| `silence_timeout` | `CharacterDef.callFlowPrompts.longSilence` |
| `call_duration_threshold` | `longCallNudge` |
| `pre_hangup_hint` | `preHangupFarewell` |

**壳 / DialogueAdapter 负责计时并上报**；引擎只选文案并注入下一轮或请 Adapter 播一句。默认阈值与 Studio 模拟按钮见 [50 §5.4](../需求/50-对话与模型适配.md)；角色可覆盖 `callFlowPolicy`（[11](../需求/11-角色与社交.md)）。

## 6. 版本

- 内容包与 SaveGame 均带 `schemaVersion`
- 破坏性变更 +1；壳对未知版本应拒绝并提示
