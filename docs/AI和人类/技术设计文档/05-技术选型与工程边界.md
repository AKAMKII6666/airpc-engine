## 1. 选型总表

| 层 | 选型 | 说明 |
|----|------|------|
| 拓扑 | **B/S** | 浏览器 + Next 门面 + 引擎（见 [18](./18-部署拓扑与BS架构.md)） |
| 应用 | **Next.js 最新稳定线**（当前 16.x）+ React + TypeScript | `apps/studio`：client UI + server 门面 |
| 画布 | `@xyflow/react` | 仅浏览器；不算第二套组件主栈 |
| UI | **仅 MUI** | 不加 Ant/Chakra/shadcn 主栈 |
| 样式 | **CSS Modules + SCSS**（`*.module.scss`） | 可用现代 CSS；MUI Emotion 仅服务 MUI 组件 |
| 表单 | MUI 控件 + Zod（RHF 可选） | 草稿在浏览器；提交经门面 |
| Schema | Zod（引擎内） | Content + SaveGame + CallSession 快照 |
| 客户端状态 | **Zustand + bis + shell**（定稿，见 [21](./21-Studio客户端分层.md)） | **不得**当 Profile/Memory 真源 |
| 持久化 | **Content / 薄 Profile = JSON**；**Memory = SQLite**（MemoryPort） | 见 [18](./18-部署拓扑与BS架构.md)、[20](./20-记忆存储与投影.md) |
| 单测 | Vitest | 引擎纯测 |

不选：Rete、tldraw、GoJS、自研 canvas、shadcn 主栈、第二套 CSS-in-JS 框架。  
**Memory = SQLite**；不把全量记忆塞进 JSON Profile。  
客户端编排哲学：**Store 记账 · Shell 灌数 · bis 办事 · UI 傻渲染**（多系统验证，本仓沿用）。

## 1.1 包边界 · barrel · 目录 · 测试

| 规则 | 定稿 |
|------|------|
| 引擎对外 | **准 npm 包**；Studio/壳只 `import from "@airpc/rpg-engine"`；**允许且鼓励** `src/index.ts` 统一门面倒出 |
| 引擎内部 | 可按域分子目录；调用方**不要**深挖内部路径 |
| Studio / 其它应用代码 | **禁止** `index.ts` barrel 聚合 re-export；直引具体文件（防过度工程化） |
| 目录职责聚类 | 同目录 ≥2 组异责且可分类文件 **≥4** → **必须**分子目录；禁止长前缀代目录；不为空壳「未来域」先建单文件空目录 |
| 测试 | 一律独立 `tests/`（引擎：`packages/rpg-engine/tests/`；Studio 日后同级约定）；**禁止**与 `src`/`app` 业务文件平放 `*.test.ts` |

详见 [21](./21-Studio客户端分层.md)（Studio）、本包 README、`.cursor/rules`。

## 2. 三层与仓库结构

```text
Browser                    apps/studio：MUI + scss module + Zustand/bis/shell
  ↓ HTTP / SSE
Next API 门面              route handlers；UserGate 选定的 userId
  ↓ 进程内 import
packages/rpg-engine        EngineHost 单例；领域 + CallSession
data/                      工作区布局见 [19 §3](./19-引擎宿主与会话模型.md)
docs/AI和人类/需求/         # 产品需求
docs/AI和人类/技术设计文档/ # 工程定稿
```

- 引擎 = 后端的后端；**Host 单例**；对外仅经包门面（见 §1.1）。
- Next = 门面：DTO、错误码映射、何时 save；不实现剧情规则；**不深耦**引擎内部文件。
- React = 请求与展示（经 bis/ajaxHelper）；不持 LLM Key。
- CI：禁引擎 import react/next/@xyflow；禁 client 打包引擎写口；Studio 分层门禁见 [21 §8](./21-Studio客户端分层.md)。

## 3. 真源分离

```text
Content（JSON 文件）
  + 薄 Profile（JSON：Board / stories / research / schedule…）
  + Memory（SQLite，经 MemoryPort）
  + logs（jsonl）
        ↑ Persistence / MemoryPort
 rpg-engine / EngineHost
        ↑（仅快照 DTO）
 Studio Zustand（非真源）
```

**Content JSON ≠ Memory DB。** 存得下 ≠ 查得起、注得起。

## 4. 端口落点

| Port | 定义 | 实现位置 |
|------|------|----------|
| DialogueAdapter | 引擎接口 | Manual/Text → **Next server**；Realtime → 壳 |
| OutcomeEvaluator | 引擎 Manual | Text 建议可放 server |
| MemoryPort / LoreBootstrap | 引擎 | Memory：**SqliteMemoryPort**；LLM Commit 在 server |
| PersistencePort | 引擎接口 | Studio：Content/薄 Profile 用 `fs`；Memory 用 sqlite |
| EffectSink | 媒介 | Studio mock / 壳真执行 |
| FreeCallPostPipeline | 引擎可注入 | 默认实现见 [19](./19-引擎宿主与会话模型.md) |

## 5. 其它

- UI 简体中文；稳定 id 英文
- 单包舒适 ≤ 80 卡节点
- 条件为结构化谓词，禁止 `eval`
- 本地单机 = `localhost` B/S
- **不做**旧话机代码 / 配置 / 数据迁移；旧仓仅能力对标；新壳另建并嵌入本引擎
