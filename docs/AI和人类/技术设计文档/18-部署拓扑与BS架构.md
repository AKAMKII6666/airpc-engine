# 18. 部署拓扑 · B/S 与持久化

相关：[05](./05-技术选型与工程边界.md) · [19](./19-引擎宿主与会话模型.md) · [20](./20-记忆存储与投影.md) · [**23-引擎存取Port与IO适配**](./23-引擎存取Port与IO适配.md) · [21](./21-Studio客户端分层.md) · [50](../需求/50-对话与模型适配.md) · [10](../需求/10-用户与存档.md)

## 1. 产品形态

```text
Browser (React + MUI + scss module + Zustand/bis/shell)
  · 故事列表 / 画布投影 / 调试 UI / UserGate
        ↓ HTTP / SSE
Next.js — API 门面
  · 开发态 userId（无密码）
  · DTO、错误码（见 19 §15）
  · 调用 EngineHost 单例；决定何时 save
        ↓
packages/rpg-engine — EngineHost
  · CallSession / Resolver / Exit / Effect / FreePipeline
  · 经 **存取 Port** 访问 Content / 薄 Profile / Memory / 日志（协议见 [23](./23-引擎存取Port与IO适配.md)）
  · 不内置 fs/sqlite 实现；本机实现见 apps/studioV2/engineIOModule
```

Studio 客户端分层见 [21](./21-Studio客户端分层.md)。

电话壳以后：**不经 Next**，直接链同一引擎，并自行注入 IO Port。

## 2. 三层职责

| 层 | 负责 | 不负责 |
|----|------|--------|
| 浏览器 | 意图、快照、UserGate UI、流式展示 | Profile/Memory 真源、Exit/Effect、LLM Key、本地引擎 IO |
| Next 门面 | 注入 userId、校验、调 Host、HTTP 映射；**装配 engineIOModule → Host** | 剧情规则 |
| rpg-engine | 领域、Session、**Port 契约** | req/res、JSX、具体落盘实现 |
| engineIOModule（Studio 本机） | JSON+fs、SQLite、jsonl 等 Port 实现 | 剧情规则、Client 引用 |

**铁律**：client bundle 禁止引擎写口；引擎只在 server/壳。

## 3. 持久化与工作区

完整目录见 [19 §3](./19-引擎宿主与会话模型.md)。摘要：

| 产物 | 路径 | 谁读写（定稿） |
|------|------|----------------|
| 故事包 / 角色 / 资产 | JSON（`storis-packages/`、`characters/`、`assets/`） | ContentPort → 本机 `engineIOModule` |
| 薄 SaveGame | `data/users/<userId>/profile.save.json` | ProfilePort → 同上 |
| **Memory DB** | `data/memory/memory.sqlite`（见 [20](./20-记忆存储与投影.md)） | MemoryPort → 同上 |
| 日志 | `data/logs/engine-YYYYMMDD.jsonl` | EngineLogPort → 同上 |
| 黄金样例 | `data/storis-packages/golden_handoff/` | 经 ContentPort |

存取协议与注入方式见 **[23](./23-引擎存取Port与IO适配.md)**。

- CallSession 默认只在内存。
- **Content JSON ≠ Memory SQLite**；v1 不做云同步。
- 记忆查询 / 热层投影见需求 [12](../需求/12-记忆模型.md)。
- 纯浏览器宿主：不本地 IO；经 XHR 到后端再走 Port。

## 4. 文本 LLM（Studio）

```text
browser 聊天 UI → /api/debug/chat (SSE，E8 已落地)
  → Next: LLM + TextDialogueAdapter → Host/Session
  → event-stream 回传（message / done）；browser 只渲染；refreshStamp 可兜底
挂机 → /api/debug/endCall → Outcome → Exit|FreePipeline → save
```

## 4.1 Studio API 草表（v1）

错误体统一 `{ ok, code, message, details? }`（[19 §15](./19-引擎宿主与会话模型.md)）。写成功路径由门面调 Host（经 ProfilePort / Content 编辑 BFF）；**禁止**在 API 层绕过 Port 直写引擎以为真源的经历态。

| 方法 | 路径 | UserGate | Host / 行为 |
|------|------|----------|-------------|
| GET | `/api/users` | 否 | 列用户 |
| POST | `/api/users` | 否 | 建用户；有 location 则 lore bootstrap |
| POST | `/api/session/selectUser` | — | 门面记住当前 `userId` |
| GET | `/api/stories` | 建议 | 列 `storis-packages` |
| POST | `/api/stories` | 是 | 新建包目录 + 空 conf |
| GET/PUT | `/api/stories/[packageId]` | 是 | 读/写 `story.conf.json` |
| GET/PUT | `/api/stories/[packageId]/cards/[cardId]` | 是 | 单卡 `.s-card.json` |
| GET/PUT | `/api/stories/[packageId]/layout` | 是 | `canvas.layout.json`（引擎忽略） |
| POST | `/api/stories/[packageId]/validate` | 是 | `validatePackage`（规则见 [08](../需求/08-内容校验规则.md)；读包经 ContentPort，见 [23 §7](./23-引擎存取Port与IO适配.md)） |
| GET/PUT | `/api/characters/[agentId]` | 是 | CharacterDef |
| GET/PUT | `/api/characters/free-cards/[freeCardId]` | 是 | Free 卡 |
| GET | `/api/tools/registry` | 否 | 内置 + 包扩展 Registry |
| POST | `/api/debug/beginCall` | 是 | resolve + beginCall（带 localNowIso） |
| POST/SSE | `/api/debug/chat` | 是 | TextDialogueAdapter 一轮 |
| POST | `/api/debug/endCall` | 是 | endCall / FreePipeline |
| POST | `/api/debug/simEvent` | 是 | 过程话术事件 |
| POST | `/api/debug/advanceClock` | 是 | Tick：`deltaMs`／`toClockMs`／`toNextIntent` |
| GET | `/api/debug/snapshot` | 是 | Session / Board / Composer 快照 |
| GET | `/api/logs` | 是 | jsonl 切片 |

壳：**不经**上述 Next 路由，直接链 Host。

## 5. 本地单机

`npm run dev` → localhost；门面内嵌 Host 单例 + `data/`。

## 6. 验收要点

- [ ] client 无引擎写口、无 engineIOModule；LLM Key 仅 server
- [ ] UserGate 选定 userId；Profile 隔离
- [ ] 工作区经 Port：Content JSON + 薄 Profile JSON + **Memory SQLite**；layout 旁车不进运行（实现见 engineIOModule）
- [ ] Host 单例；同 user 通话互斥
- [ ] API 草表路径可通；validate / debug 错误码映射正确
- [ ] Memory 投影有预算；search 走库不扫 JSON 全文件
- [ ] 存取协议符合 [23](./23-引擎存取Port与IO适配.md)
