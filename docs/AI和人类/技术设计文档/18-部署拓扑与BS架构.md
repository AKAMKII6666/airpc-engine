# 18. 部署拓扑 · B/S 与持久化

相关：[05](./05-技术选型与工程边界.md) · [19](./19-引擎宿主与会话模型.md) · [21](./21-Studio客户端分层.md) · [50](../需求/50-对话与模型适配.md) · [10](../需求/10-用户与存档.md)

## 1. 产品形态

```text
Browser (React + MUI + scss module + Zustand/bis/shell)
  · 故事列表 / 画布投影 / 调试 UI / UserGate
        ↓ HTTP / SSE
Next.js — API 门面
  · 开发态 userId（无密码）
  · DTO、错误码（见 19 §15）
  · 调用 EngineHost 单例；决定何时 save
        ↓
packages/rpg-engine — EngineHost
  · CallSession / Resolver / Exit / Effect / FreePipeline
  · Content JSON + 薄 Profile JSON + Memory SQLite 语义
```

Studio 客户端分层见 [21](./21-Studio客户端分层.md)。

电话壳以后：**不经 Next**，直接链同一引擎。

## 2. 三层职责

| 层 | 负责 | 不负责 |
|----|------|--------|
| 浏览器 | 意图、快照、UserGate UI、流式展示 | Profile/Memory 真源、Exit/Effect、LLM Key |
| Next 门面 | 注入 userId、校验、调 Host、HTTP 映射 | 剧情规则、散落改业务 JSON/SQL |
| rpg-engine | 领域、Session、存档与 MemoryPort 语义 | req/res、JSX |

**铁律**：client bundle 禁止引擎写口；引擎只在 server/壳。

## 3. 持久化与工作区

完整目录见 [19 §3](./19-引擎宿主与会话模型.md)。摘要：

| 产物 | 路径 |
|------|------|
| 故事包 / 角色 / 资产 | JSON（`storis-packages/`、`characters/`、`assets/`） |
| 薄 SaveGame | `data/users/<userId>/profile.save.json`（**不含**万级 episodic） |
| **Memory DB** | `data/memory/memory.sqlite`（或 per-user sqlite；见 [20](./20-记忆存储与投影.md)） |
| 日志 | `data/logs/engine-YYYYMMDD.jsonl` |
| 黄金样例 | `data/storis-packages/golden_handoff/` |

- CallSession 默认只在内存。
- **Content JSON ≠ Memory SQLite**；v1 不做云同步。
- 记忆查询 / 热层投影见需求 [12](../需求/12-记忆模型.md)。

## 4. 文本 LLM（Studio）

```text
browser 聊天 UI → /api/debug/chat (SSE)
  → Next: LLM + TextDialogueAdapter → Host/Session
  → 流式回传；browser 只渲染
挂机 → /api/debug/endCall → Outcome → Exit|FreePipeline → save
```

## 4.1 Studio API 草表（v1）

错误体统一 `{ ok, code, message, details? }`（[19 §15](./19-引擎宿主与会话模型.md)）。写成功路径由门面调 `saveProfile` / 落盘 Content。

| 方法 | 路径 | UserGate | Host / 行为 |
|------|------|----------|-------------|
| GET | `/api/users` | 否 | 列用户 |
| POST | `/api/users` | 否 | 建用户；有 location 则 lore bootstrap |
| POST | `/api/session/selectUser` | — | 门面记住当前 `userId` |
| GET | `/api/stories` | 建议 | 列 `storis-packages` |
| POST | `/api/stories` | 是 | 新建包目录 + 空 conf |
| GET/PUT | `/api/stories/[packageId]` | 是 | 读/写 `story.conf.json` |
| GET/PUT | `/api/stories/[packageId]/cards/[cardId]` | 是 | 单卡 `.s-card.json` |
| GET/PUT | `/api/stories/[packageId]/layout` | 是 | `canvas.layout.json`（引擎忽略） |
| POST | `/api/stories/[packageId]/validate` | 是 | `validatePackage`（规则见 [08](../需求/08-内容校验规则.md)） |
| GET/PUT | `/api/characters/[agentId]` | 是 | CharacterDef |
| GET/PUT | `/api/characters/free-cards/[freeCardId]` | 是 | Free 卡 |
| GET | `/api/tools/registry` | 否 | 内置 + 包扩展 Registry |
| POST | `/api/debug/beginCall` | 是 | resolve + beginCall（带 localNowIso） |
| POST/SSE | `/api/debug/chat` | 是 | TextDialogueAdapter 一轮 |
| POST | `/api/debug/endCall` | 是 | endCall / FreePipeline |
| POST | `/api/debug/simEvent` | 是 | 过程话术事件 |
| POST | `/api/debug/advanceClock` | 是 | `advanceClock` |
| GET | `/api/debug/snapshot` | 是 | Session / Board / Composer 快照 |
| GET | `/api/logs` | 是 | jsonl 切片 |

壳：**不经**上述 Next 路由，直接链 Host。

## 5. 本地单机

`npm run dev` → localhost；门面内嵌 Host 单例 + `data/`。

## 6. 验收要点

- [ ] client 无引擎写口；LLM Key 仅 server
- [ ] UserGate 选定 userId；Profile 隔离
- [ ] 工作区：Content JSON + 薄 Profile JSON + **Memory SQLite**；layout 旁车不进运行
- [ ] Host 单例；同 user 通话互斥
- [ ] API 草表路径可通；validate / debug 错误码映射正确
- [ ] Memory 投影有预算；search 走库不扫 JSON 全文件
