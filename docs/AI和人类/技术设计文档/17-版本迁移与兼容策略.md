# 17. 版本迁移与兼容策略

相关：[07-壳嵌入与导出契约](./07-壳嵌入与导出契约.md) · [10-用户与存档](../需求/10-用户与存档.md)

## 1. 版本字段

内容包与 SaveGame 均含整数 `schemaVersion`（v1 起为 `1`）。

未来若拆 major/minor，约定：

```text
schemaVersion = major          // v1 仅整数
或 schemaVersion: { major, minor }  // 若升级格式，另文定义
```

本文 v1 按 **单一整数 major** 描述策略；语义对齐如下。

## 2. 加载策略

| 情况 | 行为 |
|------|------|
| 版本 == 引擎支持版本 | 正常打开 |
| 版本 < 支持，且存在 migration | 纯函数迁移到当前版本后打开 |
| 版本 < 支持，且 **无** migration | **只读**打开（或拒绝编辑/导出）；提示升级路径 |
| 版本 > 引擎支持（未知更新） | **拒绝**加载，提示升级 Studio/引擎 |

```text
unknown newer version → reject
older version + migration → migrate (pure function)
older version + no migration → readonly (or soft-fail with message)
migrations are pure: (oldDoc) => newDoc，无 I/O 副作用
```

## 3. Migration 实现要求

- 放在 `packages/rpg-engine` 内，按版本链 `1→2→…→current`
- 单测：夹具旧包迁移后通过当前 Zod schema
- 迁移不得静默丢关键字段；无法映射则写入 `migrationWarnings[]`

## 4. 内容包 vs SaveGame

- 两者可独立升版本，但尽量同号
- SaveGame 迁移须保留 `userId` 与进度语义

## 5. Studio

- `file.schemaVersion > SUPPORTED` → **拒载**；Dialog：「此内容由更新版引擎编写，请升级 Studio」
- `file < SUPPORTED` 且无 migration → 拒载或只读横幅（v1 仅版本 1 时非 1 一律 reject）
- 列表项禁用；**禁止**静默损坏数据
- 导出前可「另存为当前版本」（有 migration 时）

## 6. v1 当下

- 仅 `schemaVersion: 1`；`SUPPORTED_SCHEMA = 1`
- 无 migration 表时加载非 1 一律 `SCHEMA_UNSUPPORTED`（见 [19 §15–16](./19-引擎宿主与会话模型.md)）
