## 1. EngineHost（进程单例）

### 1.1 形态

```text
Next server 启动
  → createEngineIOPorts(dataRoot)     # Studio：engineIOModule
  → createEngineHost({ ...ports })    # 进程单例
  → loadWorkspace（经 ContentPort）
  → 内存：profiles 缓存、sessionsByUserId、log ring buffer
请求
  → 门面注入 userId（来自 UserGate 选定后的开发态 cookie/header）
  → host.xxx(...)
endCall / 显式 save
  → ProfilePort / MemoryPort 持久化（非引擎内直写 fs）
```
- **v1 不做**每请求 new Engine、不做多 Host 实例。
- 未来 Web 多登录可按登录身份建实例；本阶段不预留复杂工厂。
- 同 `userId` **同时最多 1 个**活跃 CallSession（`in_call`…`executing_effects`）；第二通 `beginCall` → `CONFLICT_ACTIVE_CALL`。
- **Content 热重载 ≠ 清运行时**：编辑保存后 `loadWorkspace` / Studio `reloadStudioWorkspace` 只刷包缓存；`sessions` / `profiles` / `activeByUser` 默认保留。本通仍用 `frozenCard`（需求 03 §10）。踢会话走 `resetRuntime`（或 `loadWorkspace(..., { resetRuntime: true })`），禁与普通保存绑定。

### 1.2 对外 API（实现级收敛）

```ts
interface EngineHost {
  /** 默认 resetRuntime=false：只刷 Content；显式 true 才清 Session/Profile */
  loadWorkspace(rootDir: string, opts?: { resetRuntime?: boolean }): Promise<void>
  /** 踢会话 + 清 Profile 缓存；禁与 Content 保存绑定 */
  resetRuntime(): void
  /** 规则在引擎；读包经 ContentPort.loadPackageForValidate（见 23 §7） */
  validatePackage(packageId: string): ValidationReport

  listUsers(): UserSummary[]
  createUser(input: CreateUserInput): Promise<User>
  deleteUser(userId: string): Promise<void>
  /** 门禁选定后，后续 API 带此上下文或显式传 userId；落盘经 ProfilePort */
  ensureProfile(userId: string): Promise<PlayerProfile>
  saveProfile(userId: string, reason: SaveReason): Promise<void>

  resolve(userId: string, intent: CallIntent): ResolveResult | EngineError
  beginCall(userId: string, result: ResolveResult, opts: BeginCallOpts): CallSession
  registerExitCandidate(sessionId: string, candidate: RuntimeExitCandidate): void
  endCall(sessionId: string, outcome: Outcome): EndCallResult

  getSnapshot(userId: string): EngineSnapshot
  advanceClock(userId: string, ms: number): void
  getRecentLogs(opts?: { userId?: string; limit?: number }): LogRecord[]
}

type SaveReason = "after_effect" | "after_free_pipeline" | "manual" | "autosave"

/** beginCall 时由壳 / Studio 注入；缺省走 User 时区回退（见需求 50） */
interface BeginCallOpts {
  channel: "manual" | "text_turn" | "realtime_audio"
  dialogue?: DialogueAdapter
  /** 用户设备本地墙钟 ISO（含 offset） */
  localNowIso?: string
  timeZone?: string
  /** 调试覆盖 ComposeScene */
  sceneOverride?: Partial<ComposeScene>
}
```

`createEngine({ freeCallPostPipeline?, effectSink?, generateVoicemail?, onVoicemailUnreadChanged? })` 由 Host 持有；默认实现见 §5、§6。  
Composer / `ComposeScene` / `promptScenes` 见 [50](../需求/50-对话与模型适配.md)、[01](../需求/01-CallCard数据模型.md)。

## 2. CallSession schema（v1）

**存活**：server 内存（Host）。调试可「导出快照」序列化。**默认不写入** SaveGame 主文件。

```ts
type CallSessionStatus =
  | "resolving"
  | "composing"
  | "in_call"
  | "evaluating"
  | "selecting_exit"
  | "executing_effects"
  | "running_free_pipeline"
  | "committing_memory"
  | "completed"
  | "aborted"

interface CallSession {
  schemaVersion: 1
  sessionId: string
  userId: string
  /**
   * 故事包 id；纯 Free 通话固定为哨兵 `FREE_PACKAGE_ID`（`"__free__"`）
   * 禁止把哨兵当成真实 storis-packages 目录去 load
   */
  packageId: string

  status: CallSessionStatus
  startedAt: string
  endedAt?: string

  resolve: {
    source: "story_pending" | "free" | "redial" | "simulate"
    instanceId: string
    cardId: string
    agentId: string
    intent: CallIntent
  }

  /** beginCall 时从已保存 Content 冻结；本通不受编辑器未保存草稿影响 */
  frozenCard: CallCardDefinition
  /** 本通场景：方向 + 用户本地时间；供 Composer / 调试展示 */
  composeScene: ComposeScene
  /** Composer 输出；含 opening 与 matchedLayerIds */
  renderedPrompt?: RenderedPrompt
  matchedLayerIds?: string[]

  channel: "manual" | "text_turn" | "realtime_audio"
  interactionPhase: "playback" | "dialogue" | "done"

  phoneFlags: Partial<Record<OutcomeFlag, boolean>>
  completedBeats: string[]
  optionalBeatsCompleted?: string[]

  toolTrace: ToolTraceEntry[]
  exitCandidates: RuntimeExitCandidate[]

  /** 本通 effect 幂等账本（权威） */
  effectLedger: Record<
    string,
    { status: "executed" | "skipped"; at: string }
  >

  outcome?: Outcome
  selectedExit?: {
    exitId?: string
    candidateId?: string
    source: "static" | "dynamic"
    priority: number
  }
  effectPlanResult?: EffectPlanResult

  debug?: { note?: string }
}

interface ToolTraceEntry {
  toolCallId: string
  toolId: string
  args: unknown
  result: "ok" | "denied" | "error"
  registeredCandidateId?: string
  at: string
}
```

### 2.1 状态机

```text
Idle
  --CallIntent--> Resolving → Composing → InCall
       │  tool → registerExitCandidate only
  --session.ended--> Evaluating
  --Outcome--> SelectingExit | RunningFreePipeline
  --Exit--> ExecutingEffects
  --plan终态 / free pipeline--> CommittingMemory?
  --> Completed →（可丢弃或保留最近 N 条供时间线）
```

- Story 卡：`Evaluating → SelectingExit → ExecutingEffects`
- Free 卡（`resolve.source === "free"` 或 `frozenCard.cardKind === "free"`）：走 `FreeCallPostPipeline`（§5）；`packageId === FREE_PACKAGE_ID`

### 2.2 与编辑器隔离

- 调试只读**已保存** Content；`beginCall` 写入 `frozenCard` + `composeScene`（本地时间 / 方向）→ Composer → `renderedPrompt`。
- 发现问题：退出调试 → 回画布修改并保存 → 再开调试。调试器内**不提供**改卡 UI。

## 3. Content 工作区磁盘布局

> **布局真源在本文；谁读谁写见 [23](./23-引擎存取Port与IO适配.md)。** Host / 引擎只调 Port，**不**拼接下列路径。本机路径解释属 `apps/studioV2/engineIOModule/`。

创建故事包 = 在硬盘上新建目录 `data/storis-packages/<packageId>/`。  
**大故事按文件拆开**：包配置一份、每张卡一个文件；禁止把全部卡内联进单一巨型 JSON 作为编辑真源。

```text
data/
  workspace.json
  characters/
    <agentId>.json
  tools/
    registry.json                 # 非真源／预留导出；ToolRegistry v1 = 代码 builtinRegistry（见需求 13）
  assets/
    meta/<assetId>.json           # 全局资产元数据真源（见需求 14）
    files/...
  storis-packages/                    # 故事包根目录正式名（拼写定稿；全仓改名另立项）
    <packageId>/                      # 例：golden_handoff/
      story.conf.json                 # 包级唯一配置（轻量；cards 只登记 cardId；可选 assetRefs）
      canvas.layout.json              # 画布坐标/泳道旁车；引擎忽略
      cards/
        <cardId>.s-card.json          # 单卡真源；由 cardId 拼文件名载入
  # 角色真源仅 characters/；包内禁止 agents[] 第二份定义
  users/
    index.json
    <userId>/
      profile.save.json           # 薄存档；不含万级 episodic
  memory/
    memory.sqlite                 # Memory 真源（见技术设计 20）
  logs/
    engine-YYYYMMDD.jsonl
```

### 3.1 `story.conf.json`（每包唯一）

```json
{
  "schemaVersion": 1,
  "packageId": "golden_handoff",
  "title": "黄金样例：转介",
  "participants": ["doubao-sister", "xiaoyu"],
  "entryCardId": "doubao_intro_outbound",
  "cards": [
    { "cardId": "doubao_intro_outbound" },
    { "cardId": "xiaoyu_waiting_user" }
  ]
}
```

- `cards[]` **只含索引**（可选再加列表预览用 `title`），**不内嵌**卡 body。
- 载入规则：`cardId` → `cards/<cardId>.s-card.json`。
- conf 有 id 但文件缺失 → 校验 **error**；磁盘有多余 `.s-card.json` 未登记 → **warning**（可忽略或提示登记）。

### 3.2 `*.s-card.json`（单卡）

完整 `CallCardDefinition`（见需求 [01](../需求/01-CallCard数据模型.md)）。编辑/保存单卡只读写对应文件；运行时可按需 load，再写入 `CallSession.frozenCard`。

逻辑上的「整包 StoryPackage」由 Host **组装视图**（conf + 已加载卡）；磁盘真源永远是分文件。

### 3.3 Free 卡路径（与角色库）

```text
data/characters/<agentId>.json
data/characters/free-cards/<freeCardId>.s-card.json
```

- Free 卡**不进** `storis-packages/`；由 `CharacterDef.freeCardId` 引用。
- 纯 Free 通话 Session.`packageId = FREE_PACKAGE_ID`（`"__free__"`）；**禁止**创建或 load `storis-packages/__free__/`。
- 详见 [11 §4](../需求/11-角色与社交.md)。

### 3.4 Schedule 卡路径（与故事包剧情节点分工）

```text
data/characters/schedule-cards/<scheduleCardId>.s-card.json   # 角色日常 ScheduleCard 真源
data/storis-packages/<packageId>/cards/<cardId>.s-card.json   # 可含 cardKind=schedule 的剧情调度节点
```

| 种类 | 磁盘 | Host `packageId` | `schedule_recurring_call` |
|------|------|------------------|---------------------------|
| 角色日常 ScheduleCard | `characters/schedule-cards/` | `SCHEDULE_PACKAGE_ID`（`__schedule__`） | **唯一**合法主路径（`scheduleCardId` 或 `cardId`+`__schedule__`） |
| 包内剧情调度节点 | `storis-packages/<pkg>/cards/` 且 `cardKind=schedule` | 真实包 id | **禁止**作 recurring 目标（`SCHEDULE_CARD_KIND`） |

- Workspace 加载：`loadWorkspace` 经 ContentPort 装入 `scheduleCards`（本机 IO 扫 `characters/schedule-cards/*`）；**禁止**创建或 load `storis-packages/__schedule__/`。
- Studio：日常卡经 `/api/schedule-cards` 落盘；故事编辑器改 `cardKind=schedule` 只写本包 cards（剧情节点）。Effect 面板 `scheduleCardId` 下拉真源 = schedule-cards，不得用画布包内卡冒充。
- 产品口径见 [01 §8.1](../需求/01-CallCard数据模型.md)；校验见 [08](../需求/08-内容校验规则.md) SCHEDULE_*。

`workspace.json` 示例字段：`schemaVersion`、`engineMinVersion`、`title`、**`startupPackageId`（首故事；必填且须对应已存在故事包）**。

Studio 主路径：

| 路由 | 职责 |
|------|------|
| `/stories` | 故事列表；创建 = 新建 `storis-packages/<id>/` + 空 conf + 空 cards/ |
| `/stories/[packageId]` | 画布；进门先 UserGate |
| `/characters` | 全局角色库 |
| `/debugger` | 先 UserGate，再调试台 |

## 4. Studio 用户门禁（无密码）

- **无密码**；本地开发态选定 `userId` 即可。
- 可复用组件 **`UserGate`**（非独立业务页）：列表 / 新增 / 选择 / 删除；可塞进 MUI `Dialog`。
- **触发点**：
  1. `/debugger` 进入时（必选）
  2. `/stories` 点击某故事或「创建故事」时（保险门禁）
- 故事**编辑逻辑**不依赖用户（改的是 Content）；门禁只为后续调试/预览/校验需要 Profile 时建立上下文。
- 顶栏可显示「当前用户」；未选时部分调试 API 返回 `USER_REQUIRED`。

## 5. FreeCallPostPipeline（自由通话挂机总函数）

自由通话**不要求**作者堆静态 exits。引擎挂单一管线。

```ts
/** 纯 Free 会话的 packageId 哨兵；禁止对应 data/storis-packages/__free__/ */
export const FREE_PACKAGE_ID = "__free__"

type FreeCallPostPipeline = (ctx: {
  session: CallSession
  profile: PlayerProfile
  transcript: unknown
}) => Promise<void>
```

**v1 默认步骤（定稿）：**

```text
1. 有效聊天门闩（实质轮次 ≥ N；N 默认 2，可配置）
2. 有效 → MemoryCommit（契约见 [12](../需求/12-记忆模型.md)：至少 call_summary + 可选 vignette；**条目必带时间戳**；文本 LLM 仅 server；可配置关闭）
3. 若 session.exitCandidates 非空
     → 与 Story 相同规则跑 ExitSelector
       （静态 exits 可为空；纯动态须自带 effects）
     → EffectExecutor + EffectSink + ledger
   若为空 → 跳过 Exit（纯闲聊只记记忆）
4. 经 EngineLogPort 写 jsonl + 经 ProfilePort save Profile
```

**禁止：**

- 从 transcript 隐式扫描「约定 / 介绍 / 回拨」再写 Profile
- 对 `FREE_PACKAGE_ID` 调用 `loadPackage` / 创建故事目录

**工具：** Free 默认全开 Registry 中 `free` 可用工具（旧话机业务 FC 抽来的七支，见 [13 §7](../需求/13-工具与能力注册.md)）；通话中仍只登记 candidate。

注入：`createEngine({ freeCallPostPipeline })`。Story 卡仍走 Outcome → ExitSelector。存取注入见 [23 §5](./23-引擎存取Port与IO适配.md)。

## 6. EffectExecutor vs EffectSink

| 组件 | 职责 |
|------|------|
| **EffectExecutor** | 引擎内：改 Profile（挂卡、fact、memory、variable、schedule 队列、**voicemail GenStack**…）+ WET |
| **EffectSink** | 壳/Studio：媒介副作用（播 WAV、真响铃等）；**不含**已废弃的 `create_voicemail` |
| **VoicemailMaterializePipeline** | Host 在 plan **终态之后**出栈；调外置 `generateVoicemail`；写 `telephony.voicemails[]`；`onVoicemailUnreadChanged` |

时序：

```text
Exit 选中（或 Free pipeline 发出等价 effect）
  → Executor 逐条：
       · 逻辑类：改 Profile + WET
       · attach/schedule 目标 voicemail → GenStack / once→入栈（永不 Board.pending；到点≠ agent_outbound）
       · 媒介类：先 WET（intent）→ 调用 EffectSink（如 play_system_prompt）
  → plan / pipeline 终态
  → VoicemailMaterializePipeline（已入栈条目默认不因 aborted 丢弃）
       generateVoicemail → 写槽 → unread notify
  → completed → save Profile
```

壳**不得**自己改 Board/Memory/未读真源；只执行 Sink、注入生成/LED 端口，并上报 `call.*`。

### 6.1 scheduleTick 与 voicemail

- `schedule_call_card` 目标为普通卡：once 到点 → 既有 `agent_outbound` / Board 语义。
- 目标为 `cardKind=voicemail`：到点 **入栈/物化进信箱**（「N 分钟后信箱多一条」）；用户正在通话时仍写槽，不打断当前 CallSession。

## 7. 动态出口 vs Outcome（求值定稿）

- **Outcome**：挂机后结构化结果（flags、beats…）。
- **静态 exits**：作者预置 `condition + priority + effects`。
- **动态 candidates**：通话中工具仅登记；挂机后与静态合并进 ExitSelector。

规则：

1. 候选池 = `frozenCard.exits` ∪ `session.exitCandidates`
2. 动态带 `exitId` → 使用该静态出口的 condition + effects（candidate 可提高 priority / 提供 hints）
3. 纯动态必须自带 `effects`；隐式条件 =「candidate 仍有效」AND 基础通话 flags（如 Registry 模板声明须 `answered`）
4. 同 priority：**静态声明顺序优先于动态**

## 8. Resolver 补充定稿

### 8.1 可拨判定（`effectiveDialable`）

```text
effectiveDialable(def, runtime) =
  !def.isNarrativeOnly
  && (def.dialable === true || runtime?.unlocked === true)
```

- `user_dial` 且 `!effectiveDialable` → `CHARACTER_NOT_DIALABLE`（UI：「不可拨打」；可提示未解锁 / 仅设定角色）
- `agent_outbound` / `simulate_start`：普通不可拨角色仍允许；**`isNarrativeOnly` 拒绝**（调试可 force）
- `unlocked` 仅经 Effect `set_character_unlocked` 或调试写口修改（挂机后；须 WET）；**不能**解锁 narrative-only

### 8.2 `schedule_call_card` / Clock Tick

- 写入 `profile.schedule.intents`，shape = ScheduledIntent.`once`（见需求 10）
- **必须**带 `cardId` + `packageId`；禁止仅 `topicHint` 当作剧情推进
- v1 **确定触发点** = `clockMs + minMs`（不用窗口随机）
- `EngineHost.advanceClock(userId, deltaMs)` / `setClockMs` / `advanceClockToNextIntent`：推进逻辑时钟 → **本仓调试语义**：
  - `once` 到期 → 挂 `entryMode=outbound_auto` 的 pending
  - `recurring` 跨过逻辑日 `hour:minute`（`SCHEDULE_DAY_MS=24h`）→ 生成可观测 `once` 实例（`sourcedFromRecurringId`）；有 `cardId+packageId` 时再点火；真壁钟仍归壳 Sink
- 工具展开（`refer_to_expert` / `schedule_reminder_call`）同要求 `card_id` + `package_id`；`schedule_recurring_call` 可选带卡以便 Tick 点火
- Studio：`POST /api/debug/advanceClock` 支持 `deltaMs`｜`toClockMs`｜`toNextIntent`；调试台／世界台入口可用

### 8.3 `redialSlot`

- 存在 **PlayerProfile**（建议 `profile.telephony.redialSlot`）
- 跨通话保留；`set_redial_slot` 写 Profile；Session 只读

### 8.4 ActiveStoryLock 决策表

| 场景 | soft | hard |
|------|-------|------|
| 拨 `allowedAgentIds` 内 | 允许 | 允许 |
| 拨锁外 + `reject_call` | 警告但允许 Free | **拒绝** `STORY_LOCKED` |
| 拨锁外 + `force_free_suppressed` | 允许接通，强制 Free、忽略其 pending 剧情卡 | 同左 |
| 拨锁外 + `allow_with_warning` | 允许 + 日志（调试向） | 同左 |
| 外呼目标在 allowed | 允许 | 允许 |
| 外呼目标不在 allowed | soft：允许并打标 | hard：拒绝 |
| `simulate_start` | 调试可 force 绕过 | 同左 |

v1：同一 Profile **至多一个** hard lock。

| 新建用户且有 `location` | 自动 bootstrap |
| 用户改 `location` | 提示是否重新生成（默认不自动覆盖已有） |
| 世界台 / 调试手动按钮 | 强制重新生成 |

**端口：** Host 经 `CreateEngineHostOptions.loreBootstrap` 注入 `LoreBootstrapPort`（Studio server 用环境变量装配 OpenAI 兼容 LLM；Key 仅 server）。成功写入 `source: "llm"`；无 port／无 Key／调用抛错 → `buildFallbackLore`（`source: "fallback"`）。单测用 mock port。

失败／降级：仍写入 Profile.world.lore（fallback 文档）；日志 `world.lore_bootstrapped`（含 `usedFallback`／`errorMessage`）；UI 黄条。不阻塞建用户。

Studio env（可选）：`AIRPC_LORE_LLM_API_KEY`（或 `OPENAI_API_KEY`）、`AIRPC_LORE_LLM_BASE_URL`、`AIRPC_LORE_LLM_MODEL`、`AIRPC_LORE_LLM_ENABLED`。

## 14. 日志

| 通道 | 内容 |
|------|------|
| Host 内存 ring | 调试时间线／WET 查询热源 |
| `data/logs/engine-YYYYMMDD.jsonl` | 事件 + error；**旁路**，不整份塞进 SaveGame；**持久化只经 EngineLogPort**（[23 §4.4](./23-引擎存取Port与IO适配.md)） |

**WET 与 Profile：** 逻辑事件只走 ring + EngineLogPort→jsonl；**禁止**把整份 WET 写入薄 Profile／SaveGame。幂等权威在通话内 `CallSession.effectLedger`。Studio 受控追加仅 `wet.annotation`／`wet.compensation`（标注／补偿说明），**只追加不改写**既有行，禁止借追加篡改 effect 历史。

排障依赖 jsonl + 调试时间线；无 log 视为验收失败。

## 16. 包版本 × Studio 拒载

- `workspace` / package / SaveGame 均带整数 `schemaVersion`
- 引擎 `SUPPORTED_SCHEMA = 1`（当前）
- `file > SUPPORTED` → **拒载**；文案：「此内容由更新版引擎编写，请升级 Studio」
- `file < SUPPORTED` 且无 migration → 拒载或只读（见 [17](./17-版本迁移与兼容策略.md)）；v1 仅版本 1 时非 1 一律 reject
- UI：列表项禁用 + Dialog 说明；**禁止**静默损坏数据
