## 2. 投影与表单贯通

```text
CharacterSummary (typeFiles/.../characterSummary.ts)
  ↔ toCharacterDetailFormValues / applyCharacterDetailForm
  ↔ CharacterDetailFormValues (Formik)
  ↔ characterDefToSummary / mergeDetailFormIntoCharacterDef  (落盘 CharacterDef)
```

关键文件：

- Items：`bis/.../characterDetailFormItems.ts`  
  - 基本信息：`CHARACTER_BASIC_ITEMS`（含音色 Select）  
  - 提示词：`CHARACTER_PROMPT_ITEMS`（含 **人格类型** `persona.personalityCode`）  
- 校验：展示字段编辑态**全必填**（含 personalityCode）。  
- MBTI 选项：`typeFiles/library/characters/persona/personalityCodeOptions.ts`（16 型）。  
- 音色选项：`typeFiles/library/characters/realtime/realtimeVoiceOptions.ts`（引擎只存 `voiceId` 字符串）。

## 3. 哪里能编角色

| 入口 | 行为 |
|------|------|
| 角色库详情页 | `CharacterDetailEditForm` + BASIC/PROMPT items；保存走 API |
| 故事编辑器选中锚点 | `StoryEditorCharacterModals` 复用同一套 items；打开前 `fetchCharacterDef` |
| 画布「添加角色」 | 先走精简 `CREATE_CHARACTER_*` → `commitCreateCharacter`，再可编辑详情槽 |

画布角色锚点 **打开时**由 `/api/characters` 全库 + 本包挂卡数派生（路径 B；不依赖 conf.participants 白名单）；`pendingCardCount` 由当前包内卡归属统计。**不是**独立 mock 数组；完整人设仍按 agentId 从 API 取。

## 4. persona.personalityCode 业务理解

- Studio：Select 写入投影字段；保存时 trim 后进 CharacterDef。  
- 引擎：见 [05-引擎Composer与personalityCode.md](./05-引擎Composer与personalityCode.md)。  
- UI helperText 已写明会进 Composer systemHard（演示口径；Host 真跑另议）。

## 5. 已知小不一致（勿当 bug 乱修扩大范围）

- 部分磁盘 `voiceId`（如 `zh_child_boy`、`zh_male_elder`）**不在** `REALTIME_VOICE_OPTIONS` 枚举内；下拉可能显示空/异常，属音色枚举未扩全，与 MBTI 任务无关。  
- `createEmptyCharacterDetailSlots` 默认 personalityCode = 选项表第一项（ISTJ），新建后再改。
