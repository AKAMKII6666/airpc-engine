# Studio V2 · 分层与目录落点（代码事实）

> 对照：`apps/studioV2/` · 需求真源见 [09-Studio-V2前端目录结构与落点规则](../../AI和人类/sudioV2.0版本/需求/ui/09-Studio-V2前端目录结构与落点规则.md)。  
> 固化：2026-07-22（V2-T3-8：第三步真源接通收口）。

## 1. 只认的业务顶层

```text
apps/studioV2/
  app/           # Next 路由入口；page.tsx 不写完整业务
  src/
    pageComponents/<route>/   # 页面主组件与页私有 com/hooks
    bis/pageBis/              # 页面编排、表单契约、纯函数命令
    stores/                   # Zustand（若有）
    utils/                    # 工具；mock 多在 ajaxProxy/**/mock/
    commonUiComponents/       # ≥2 页复用的展示组件（目录单元，禁 barrel）
  typeFiles/                  # 类型、DTO、UI 投影、枚举选项
  tests/                      # Vitest；禁与 src 平放 *.test.ts
```

禁止再引入旧式顶层：`features/`、`domain/`、`services/`、`commands/`、`store/`（旧 Studio 那套）。

## 2. 职责怎么分（对照代码习惯）

| 层 | 放什么 | 不放什么 |
|----|--------|----------|
| `pageComponents` | JSX、壳编排、画布节点 UI | 领域判断、保存事务、磁盘 Content 真源 |
| `bis/pageBis` | Formik items/validate/apply、投影纯函数、整包读写 | React 组件（少数例外已在 page） |
| `typeFiles` | 投影类型、Select options、标签表 | mock 数据数组本身 |
| `utils/.../mock` | **仅**资源库/调试器/设置等允许暂留的展示 mock | 角色/玩家/故事包 Content 真源 |

## 3. 真源与剩余 mock（重要）

| 域 | 落点 | 说明 |
|----|------|------|
| **角色库列表/详情** | `data/characters` ↔ `/api/characters` | **已接通**；角色业务 mock **已删** |
| **玩家配置列表/详情** | `data/users` ↔ `/api/users` | **已接通**；玩家列表 mock **已删** |
| **故事包列表** | `listStoryPackages_bis` → `GET /api/stories` 扫磁盘 | **已接通**；`MOCK_STORY_PACKAGES` **已删** |
| **编辑器打开/保存** | `loadStoryPackage_bis` / `saveStoryPackage_bis` + `graph/diskBundleGraph.ts` | **已接通**；`CanvasMockGraph` **已删** |
| 画布角色锚点 | 打开时由 `bundle.participants` + `/api/characters` displayName 生成 | 非独立 mock 数组 |
| 包配置浮窗 | `package/conf/packageConfProjection.ts` | 读当前 bundle conf，禁止内存种子 |
| 资源库列表 | `utils/ajaxProxy/library/mock/mockLibraryData.ts` | **仅**资源会话 mock（本步允许暂留） |
| 工作台侧栏 | `mockWorkbenchData.ts` | 工程状态/最近调试 **展示 mock**（非故事包列表真源） |
| 调试器 / 设置 | `mockDebuggerData` / `mockSettingsData` | 展示 mock（本步允许暂留） |

**统一口径：** 角色 / 玩家 / 故事包 Content **已落盘**；上表「允许暂留」项不得被业务误当成 Content 真源。

## 4. 质量门禁提醒（写代码时）

- V2 统一入口：`npm run quality:studio`（lint / typecheck / comments / structure / tests）。  
- **触碰历史空格缩进文件**须整文件改 tab，否则 `STUDIO-STRUCT-017`。  
- 大文件告警线约 350：`diskBundleGraph.ts` 等拆分或控增长。  
- `apps/studioV2` **禁止**业务 `index.ts` barrel。

## 5. 与旧 Studio / 引擎边界

- Studio **client 不 import 引擎写口**；类型可从 `@airpc/rpg-engine` 门面读。  
- 引擎纯度：`packages/rpg-engine` 无 React / DOM / Next。  
- 旧 `apps/studio` 仅参考，不得降低 V2 门禁。
