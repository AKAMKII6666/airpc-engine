## 1. 壳层结构（认知地图）

```text
StoryEditorShell
  ├ StoryEditorTopBar          # chapterTitle 等；保存 → saveStoryPackage_bis
  ├ StoryEditorCanvasLayer     # React Flow 舞台
  │    └ StoryCanvasStage / useStoryCanvasGraph
  ├ FloatingPanelShell         # 属性浮窗（可拖、粘性头尾）
  │    ├ NodePropertyForm | ChapterPropertyForm
  │    └ ExitListEditor → ExitEffectsList → ExitEffectPanel
  ├ BottomToolBar              # placement：CallCard / chapter_end …
  └ library floats             # 包配置 / 资源挑选等
```

会话态：节点/边在 `useStoryCanvasGraph` 的 React state；**顶栏保存**经 `editorGraphToBundle` 写回 `data/storis-packages`。

## 2. 节点种类

| React Flow type | data 投影 | 作用 |
|-----------------|-----------|------|
| `callCard` | `EditorCallCardProjection` | 通话合同卡 |
| `chapter` | `EditorChapterNodeData` | `chapter_start` / `chapter_end` |
| `characterAnchor` | `CharacterAnchorNodeData` | 左侧角色锚点；role handle |

另有轻量 dock 节点（action / commentGroup）：可放置，但属性浮窗策略与 CallCard 分离（见 dock 工厂）。

## 3. 边种类

| edgeKind | 视觉意图 | 真源 |
|----------|----------|------|
| `story` | 章节/出口流转（实线） | layout.edges + 会话内连线 |
| `role` | 卡 → 锚点归属（灰点线） | `ROLE_EDGE_STYLE` |
| `effect` | attach/unmount（绿虚线等） | `effectEdgeSync.ts` 与 `exit.effects[]` 双向 |

效果边 id：`effect_${sourceNodeId}_${exitId}_${effectId}`，须与 layout 种子边一致以便 reconcile 幂等。

## 4. 选中与属性浮窗（铁律级行为）

- **仅单选**节点打开属性浮窗：`storyCanvasSelection.ts` 中 `params.nodes.length === 1`。  
- **框选多节点不得开浮窗**（曾出岔子，见 [06](./06-过程岔子与纠正记录.md)）。  
- `selectionKind`：`callCard` | `chapter`；锚点走角色 FormModal，不是 CallCard 浮窗。  
- 防御：`toolPolicy?.mode ?? ""` 等，避免缺字段炸表单。

## 5. 出口与 Effect 面板

- 出口列表：`ExitListEditor` / `exitListForm`。  
- Effect 参数联合：`typeFiles/.../editorEffectParams.ts`（按 effect 判别，禁 catchall）。  
- 面板注册：`ExitEffectPanel.tsx` → 各 `*EffectPanel`。  
- 摘要文案：`summarizeEffect.ts`。  
- 挂载目标：Mount 弹层只选**目标卡**，`agentId` 由卡归属推导（曾简化，见 06）。

第一幕磁盘包常用 effect：`attach_call_card`、`schedule_call_card`、`end_story`。  
留言进信箱：目标卡 `cardKind=voicemail` 时用 `attach_call_card` / `schedule_call_card`（**不再**使用已废弃的 `create_voicemail`）。

## 6. 浮窗拖拽

- `hooks/panel/useFloatingPanelDrag.ts`。  
- 坐标相对 **`offsetParent`**，不是 viewport 裸 client（曾偏位，见 06）。

## 7. 打开 / 保存管道

- **打开：** `loadStoryPackageForEditor` → `GET /api/stories/:id` → `bundleToEditorGraph` → 初始 nodes/edges。  
- **保存：** 顶栏 → `saveStoryPackageToDisk` → `editorGraphToBundle` → `PUT /api/stories/:id`。  
- **保存闸门（V2-S8-18）：** BFF `writeValidatedDiskStoryPackage` 写盘后调引擎 `validatePackage`；**error 回滚写前快照并返回 `PACKAGE_VALIDATION_FAILED`（details.report）**；warning 不阻断。  
- **只读校验（V2-S8-19）：** `GET /api/stories/:packageId/validate` → 同引擎 `validatePackage`；调试器面板展示；不写盘、不 Host。  
- **错误定位 v1：** 条幅列出 `ruleId / path / message`；可解析 `cards/<cardId>.s-card.json#exits.<exitId>…` → `selectCallCardByCardId` 选中画布卡并开属性浮窗。  
- **默认包：** 路由 `wrong_number_act1`；缺 layout 时 BFF 安全默认坐标。  
- 业务叙事：见 [04](./04-第一幕打错电话-mock拓扑.md)。
